# Модуль 04 — Линейная регрессия с нуля

Большая идея модуля: до сих пор мы только *считали* — формы тензоров, производные,
градиенты. Здесь мы впервые **обучаем модель**. Линейная регрессия — самый простой
обучаемый объект на свете, и именно поэтому он идеален: в нём уже есть ВСЕ части,
из которых собрана любая нейросеть — модель, функция потерь, градиент и шаг
оптимизации, — но без единого скрытого слоя, который мог бы запутать. Если ты понял
этот модуль до конца, ты понял скелет всего глубокого обучения. Всё, что будет
дальше (нейросети, backprop, оптимизаторы, внимание) — это надстройки над тем же
самым циклом: **предсказать → измерить ошибку → шагнуть в сторону меньшей ошибки**.

В этом модуле всё пишется руками на numpy, без PyTorch.

---

## Идея 1. Модель `y = X·w + b`

Линейная модель отвечает на вопрос: «как предсказать число `y` по набору признаков
объекта?». Ответ — взвешенная сумма признаков плюс сдвиг.

Для одного объекта с признаками `x = (x₁, x₂, …, x_D)`:

```
y_pred = w₁·x₁ + w₂·x₂ + … + w_D·x_D + b
```

`w` (веса) говорят, **насколько важен** каждый признак и в какую сторону он тянет
ответ; `b` (bias, сдвиг) — это «базовый уровень», ответ когда все признаки равны нулю.

**Формы тензоров — держи их в голове, это половина дела.** Обычно мы предсказываем
не для одного объекта, а сразу для целой пачки:

- `X` имеет форму **(N, D)**: `N` объектов в строках, `D` признаков в столбцах.
- `w` имеет форму **(D,)**: по одному весу на признак.
- `b` — скаляр (одно число).
- результат `y_pred` — форма **(N,)**: по одному предсказанию на объект.

Магия в том, что всю пачку считает одно матрично-векторное умножение:

```
y_pred = X @ w + b        # (N, D) @ (D,) -> (N,), затем + скаляр ко всем
```

**ПОЧЕМУ это работает?** `X @ w` берёт каждую строку `X` (один объект) и скалярно
умножает её на `w` — то есть считает ту самую взвешенную сумму, но сразу для всех
`N` объектов. Никаких python-циклов: numpy делает это одной операцией. Прибавление
скаляра `b` (broadcasting) добавляет сдвиг к каждому из `N` предсказаний.

**Маленький численный пример.** Пусть
`X = [[1, 2], [3, 4]]` (2 объекта, 2 признака), `w = [1, -1]`, `b = 0.5`.

```
объект 0:  1·1 + 2·(-1) + 0.5 = 1 - 2 + 0.5 = -0.5
объект 1:  3·1 + 4·(-1) + 0.5 = 3 - 4 + 0.5 = -0.5
y_pred = [-0.5, -0.5]
```

## Идея 2. Функция потерь MSE

Модель что-то предсказала — но **насколько хорошо**? Нужно одно число, измеряющее
ошибку. Для регрессии стандарт — **средняя квадратичная ошибка** (Mean Squared Error):

```
MSE = mean( (y_pred − y)² )
```

То есть: для каждого объекта берём разность «предсказание минус правда», возводим в
квадрат и усредняем по всем `N` объектам. Форма: на входе два вектора `(N,)`, на
выходе — один **скаляр** (`float`).

**ПОЧЕМУ квадрат, а не модуль?** Две причины. Во-первых, квадрат сильно штрафует
большие промахи: ошибка 10 даёт вклад 100, а ошибка 2 — всего 4. Модель в первую
очередь «лечит» грубые ошибки. Во-вторых — и это важнее для нас — квадрат **гладкий**:
у него есть аккуратная производная везде, тогда как у `|·|` в нуле излом. А
производная нам сейчас понадобится для обучения.

**Маленький численный пример.** `y_pred = [2, 0, 2, 0]`, `y = [0, 0, 0, 0]`:

```
ошибки:   2, 0, 2, 0
квадраты: 4, 0, 4, 0
MSE = (4 + 0 + 4 + 0) / 4 = 2.0
```

`MSE ≥ 0` всегда, и равна `0` только при идеальном попадании. Цель обучения — сделать
её как можно меньше.

## Идея 3. Градиентный спуск и шаг `lr`

Теперь главное. У нас есть «гора» — функция потерь `MSE(w, b)`, зависящая от
параметров. Мы стоим в какой-то точке (текущие `w, b`) и хотим спуститься вниз, в
долину малой ошибки. Куда шагать?

Градиент `∇loss` указывает направление **самого крутого подъёма**. Значит, чтобы
спускаться, надо идти в **противоположную** сторону — отсюда знак минус:

```
w ← w − lr · dw
b ← b − lr · db
```

`lr` (learning rate, скорость обучения) — это **длина шага**. Большой `lr` — спускаемся
быстро, но рискуем «перелететь» долину и улететь вверх (расходимся). Маленький `lr` —
надёжно, но мучительно медленно. Это первый гиперпараметр, который ты будешь
подбирать руками.

**Откуда берутся формулы для `dw` и `db`?** Распишем честно, шаг за шагом. Обозначим
ошибку `e = y_pred − y` (форма `(N,)`). Loss:

```
L = (1/N) · Σᵢ eᵢ²,   где  eᵢ = (Xᵢ·w + b) − yᵢ
```

Берём производную по одному весу `w_j` цепным правилом (`dL/de · de/dw_j`):

```
∂L/∂w_j = (1/N) · Σᵢ 2·eᵢ · ∂eᵢ/∂w_j = (2/N) · Σᵢ eᵢ · X_{i,j}
```

`Σᵢ eᵢ · X_{i,j}` по всем `j` — это в точности `X.T @ e`. Поэтому в векторном виде:

```
dw = (2 / N) · (X.T @ e)      # форма (D,)  — по одному числу на вес
db = (2 / N) · sum(e)         # скаляр      — потому что ∂eᵢ/∂b = 1
```

Проверь формы: `X.T` имеет форму `(D, N)`, `e` — `(N,)`, значит `X.T @ e` — `(D,)`,
ровно как `w`. Так и должно быть: градиент всегда той же формы, что и параметр.

**Численная мини-проверка.** `X = [[1, 2], [3, 4]]`, `w = [0, 0]`, `b = 0`,
`y = [1, 1]`. Тогда `y_pred = [0, 0]`, `e = [−1, −1]`, `N = 2`:

```
X.T @ e = [1·(−1) + 3·(−1),  2·(−1) + 4·(−1)] = [−4, −6]
dw = (2/2)·[−4, −6] = [−4, −6]
db = (2/2)·(−1 + −1) = −2
```

`dw` отрицательный → шаг `w − lr·dw` **увеличит** веса. Логично: предсказания (0)
меньше правды (1), значит веса надо поднять. Знаки сходятся — формула верна.

## Идея 4. Полный цикл обучения

Все четыре кирпичика складываются в один цикл. Это и есть «обучение модели»:

```python
import numpy as np

# --- данные с известным линейным законом + чуть шума ---
np.random.seed(0)
N, D = 64, 3
X = np.random.randn(N, D)
w_true = np.array([2.0, -3.0, 0.5])
b_true = 1.0
y = X @ w_true + b_true + 0.01 * np.random.randn(N)

# --- стартуем «с нуля»: модель ничего не знает ---
w = np.zeros(D)
b = 0.0
lr = 0.1

print("loss до:", mse_loss(predict(X, w, b), y))

for step in range(200):
    y_pred   = predict(X, w, b)        # 1) предсказать
    dw, db   = gradients(X, y, w, b)   # 2) посчитать градиент loss
    w, b     = gd_step(w, b, dw, db, lr)  # 3) шагнуть вниз по loss

print("loss после:", mse_loss(predict(X, w, b), y))
print("найденные w:", w, " (истинные:", w_true, ")")
```

После цикла `loss` должен резко упасть, а найденные `w, b` — почти совпасть с
`w_true, b_true`. **Вот и всё обучение.** Запомни эту четвёрку строк в цикле — почти
весь курс дальше про то, как делать каждый шаг для всё более сложных моделей.

## Задания

Открой `linreg.py` и реализуй четыре функции. Тела сейчас бросают
`NotImplementedError` — замени их своим кодом. Сигнатуры менять нельзя (по ним
зовут тесты).

1. `predict(X, w, b) -> np.ndarray`
   Предсказания линейной модели. `X: (N, D)`, `w: (D,)`, `b: float` → результат `(N,)`.
   Считается как `X @ w + b`.

2. `mse_loss(y_pred, y) -> float`
   Средняя квадратичная ошибка `mean((y_pred − y)²)`. На входе два вектора `(N,)`,
   на выходе один `float`.

3. `gradients(X, y, w, b) -> (dw, db)`
   Градиенты MSE по параметрам. Вернуть `dw` формы `(D,)` и `db` типа `float`
   по формулам из «Идеи 3». Переиспользуй свой `predict`.

4. `gd_step(w, b, dw, db, lr) -> (w, b)`
   Один шаг градиентного спуска: `w − lr·dw`, `b − lr·db`. Не забудь **минус**.

Запуск тестов:

```bash
./deep-learning/run.sh 04
```

Цель — превратить все красные тесты в зелёные. Главный из них собирает полный цикл
обучения и проверяет, что итоговый `loss` стал меньше половины начального.

### Подсказки

<details><summary>Задание 1 — predict, формы не сходятся</summary>

`X @ w` — это `(N, D) @ (D,)`, numpy возвращает `(N,)`. Прибавление скаляра `b` ничего
не ломает: broadcasting добавит его к каждому элементу. Если получаешь ошибку про
формы — проверь, что `w` именно `(D,)`, а не `(D, 1)`.
</details>

<details><summary>Задание 2 — mse_loss, вернулся массив вместо float</summary>

`np.mean(...)` на 1-мерном массиве даёт 0-мерный numpy-скаляр. Оберни результат в
`float(...)`, чтобы вернуть честный python-`float` — этого ждёт тест `isinstance`.
Сама формула: `(y_pred - y) ** 2`, затем `.mean()`.
</details>

<details><summary>Задание 3 — gradients, с чего начать</summary>

Сначала ошибка: `e = predict(X, w, b) - y` (форма `(N,)`). Дальше прямо по формулам:
`dw = (2 / N) * (X.T @ e)`, `db = (2 / N) * np.sum(e)`, где `N = X.shape[0]`.
`db` оберни в `float(...)`.
</details>

<details><summary>Задание 3 — а почему именно X.T @ e?</summary>

Нам нужен вектор из `D` чисел — по производной на каждый вес. Производная по `w_j`
равна `(2/N)·Σᵢ eᵢ·X_{i,j}` — это скалярное произведение `j`-го столбца `X` на вектор
`e`. Собрать такие произведения по всем столбцам сразу — это и есть `X.T @ e`
(форма `(D, N) @ (N,) = (D,)`). Если перепутать на `X @ e`, форма не сойдётся.
</details>

<details><summary>Задание 4 — gd_step, тест на «движение вниз» падает</summary>

Самая частая ошибка — знак. Должно быть `w - lr * dw` (минус!), а не плюс. Градиент
смотрит вверх по loss; мы вычитаем его, чтобы идти вниз. `b` верни как `float(...)`.
</details>

<details><summary>Весь цикл «учится», но loss не падает или расходится</summary>

Если loss растёт — почти наверняка перепутан знак в `gd_step` или слишком большой
`lr`. Если стоит на месте — проверь множитель `2/N` в градиентах и что `gradients`
реально зовёт твой `predict`. Сверься с численным градиентом (тест
`test_gradients_match_numerical` делает это за тебя).
</details>
