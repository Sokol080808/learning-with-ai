# Модуль 02 — Линейная алгебра для DL

Большая идея модуля: нейросеть — это, по сути, **много скалярных произведений подряд**.
Один нейрон берёт входной вектор, поэлементно перемножает его с вектором весов, складывает —
и получает одно число. Целый слой — это пачка таких нейронов, то есть **матричное умножение**.
А чтобы сравнивать векторы (насколько похожи два слова, две картинки, два эмбеддинга), нужны
**норма** и **косинусная близость**. Освоив эти четыре кирпичика, ты поймёшь, что происходит
внутри `Linear`-слоя, внутри внимания и внутри почти любой формулы из следующих модулей.

В этом модуле мы реализуем всё **руками на numpy**, без `np.dot` и `np.matmul`. Не потому, что
numpy плох (он быстрее в сотни раз!), а потому, что один раз написав цикл сам, ты навсегда
перестанешь бояться формул вида `y = Wx + b`.

Договоримся про **формы** (shape). Форму тензора я буду проговаривать словами:
- вектор длины `n` — это форма `(n,)` (одна ось, `n` чисел в ряд);
- матрица — форма `(rows, cols)`, например `(n, k)` это `n` строк и `k` столбцов;
- в DL «батч из `N` примеров по `D` признаков» — это матрица формы `(N, D)`.

---

## Идея 1. Скалярное произведение и его смысл

**Интуиция.** Возьми два вектора одинаковой длины. Скалярное произведение (dot product) —
это «перемножь поэлементно и сложи»:

```
a = [a0, a1, a2]
b = [b0, b1, b2]
dot(a, b) = a0*b0 + a1*b1 + a2*b2   # одно число (скаляр)
```

Маленький численный пример:

```
a = [1, 2, 3]
b = [4, 5, 6]
dot = 1*4 + 2*5 + 3*6 = 4 + 10 + 18 = 32
```

**ПОЧЕМУ это важно.** Один нейрон делает ровно это: `выход = dot(вход, веса) + сдвиг`. Большое
скалярное произведение означает, что вход «смотрит в ту же сторону», что и веса — нейрон
сильно «зажигается». Скалярное произведение — это мера согласованности двух векторов. Оно же
прячется за геометрической формулой:

```
dot(a, b) = |a| * |b| * cos(угол между a и b)
```

То есть знак подсказывает направление: `dot > 0` — векторы смотрят примерно в одну сторону,
`dot < 0` — в противоположные, `dot = 0` — перпендикулярны (ортогональны).

**Форма.** Входы — два вектора формы `(n,)`. Выход — обычное число `float`, формы нет.

**Подводный камень.** Векторы должны быть одной длины, иначе сложить пары не получится — в нашей
реализации это будет ошибкой.

## Идея 2. Матричное умножение `matmul` и согласование форм

**Интуиция.** Матричное умножение — это «скалярное произведение каждой строки левой матрицы с
каждым столбцом правой». Если `A` имеет форму `(n, k)`, а `B` — форму `(k, m)`, то результат `C`
имеет форму `(n, m)`, и

```
C[i, j] = dot( i-я строка A , j-й столбец B )
```

Маленький численный пример `(2,2) @ (2,2)`:

```
A = [[1, 2],      B = [[5, 6],
     [3, 4]]           [7, 8]]

C[0,0] = 1*5 + 2*7 = 19
C[0,1] = 1*6 + 2*8 = 22
C[1,0] = 3*5 + 4*7 = 43
C[1,1] = 3*6 + 4*8 = 50

C = [[19, 22],
     [43, 50]]
```

**ПОЧЕМУ это важно.** Целый линейный слой сети — это `Y = X @ W + b`. Если `X` — батч формы
`(N, D)` (N примеров, D признаков), а `W` — веса формы `(D, H)` (D входов, H нейронов), то `X @ W`
имеет форму `(N, H)`: те же N примеров, но теперь по H выходов. Всё обучение — это перемалывание
таких произведений.

**Про сдвиг `b` (bias).** Одного `X @ W` мало: матричное умножение всегда тащит линию (плоскость,
гиперплоскость) через начало координат. Вектор сдвига `b` формы `(H,)` — по одному числу на каждый
из H нейронов — прибавляется к результату и **двигает** гиперплоскость от нуля. Геометрически: без
`b` граница решения обязана проходить через `0`; с `b` её можно поставить где угодно. Формула по
элементам: `Y[i, j] = dot(X[i, :], W[:, j]) + b[j]`.

Тонкость с формами — **broadcasting**. `X @ W` имеет форму `(N, H)`, а `b` — всего `(H,)`. numpy
автоматически «растягивает» `b` по оси батча: один и тот же вектор сдвига прибавляется к каждой из
N строк. Это не цикл по примерам — numpy раздаёт `b[j]` в столбец `j` для всех строк сразу:

```
(N, H) + (H,)  ->  (N, H)     # b прибавляется к КАЖДОЙ строке
[[. . .],         [b0 b1 b2]      [[.+b0 .+b1 .+b2],
 [. . .],     +                ->  [.+b0 .+b1 .+b2],
 [. . .]]                          [.+b0 .+b1 .+b2]]
```

Именно это и делает `torch.nn.Linear`: хранит `W` и `b`, на forward считает `X @ W + b`. В задании
5 ты соберёшь этот слой сам.

**Согласование форм (золотое правило).** Внутренние размеры должны совпасть:

```
(n, k) @ (k, m) -> (n, m)
        ^^^  ^^^
        эти два k обязаны быть равны; они «схлопываются»
```

Запомни мнемонику: «внутренние сокращаются, внешние остаются». `(N, D) @ (D, H) -> (N, H)`.
Если внутренние размеры не равны — умножение невозможно, и это тоже будет ошибкой в нашей
реализации.

**Подводный камень.** Порядок важен: `A @ B` в общем случае **не равно** `B @ A` (даже формы
могут не сойтись). Линейная алгебра некоммутативна — это нормально.

## Идея 3. Норма L2

**Интуиция.** Норма — это «длина» вектора. Самая частая, L2 (евклидова), — это привычная длина по
теореме Пифагора, обобщённая на любое число измерений:

```
l2_norm(v) = sqrt( v0^2 + v1^2 + ... + v_{n-1}^2 ) = sqrt( dot(v, v) )
```

Маленький численный пример:

```
v = [3, 4]
l2_norm = sqrt(3^2 + 4^2) = sqrt(9 + 16) = sqrt(25) = 5
```

**ПОЧЕМУ это важно.** Норма отвечает на вопрос «насколько большой вектор». В DL она везде:
- величина градиента `|grad|` — сигнал «насколько резко меняем веса» (отсюда *gradient
  clipping* — обрезаем норму, чтобы шаги не взрывались);
- регуляризация (weight decay) штрафует за большую норму весов, чтобы сеть не переобучалась;
- чтобы получить **направление** вектора, его делят на собственную норму — выходит вектор
  длины 1 (см. следующую идею).

Заметь связь с идеей 1: `l2_norm(v) = sqrt(dot(v, v))`. Скалярное произведение вектора с самим
собой — это сумма квадратов, а корень из неё — длина.

**Форма.** Вход — вектор `(n,)`, выход — число `float >= 0`.

**Подводный камень.** Норма нулевого вектора `[0, 0, ...]` равна `0`. На него потом нельзя делить —
держи это в голове для идеи 4.

## Идея 4. Косинусная близость

**Интуиция.** Иногда нам не важна длина векторов, важно только **направление** — насколько они
«смотрят в одну сторону». Для этого берём угол между ними, точнее косинус этого угла:

```
cosine_similarity(a, b) = dot(a, b) / ( l2_norm(a) * l2_norm(b) )
```

Это число всегда лежит в отрезке `[-1, 1]`:
- `+1` — векторы сонаправлены (один — положительно масштабированная копия другого);
- `0` — перпендикулярны, «не связаны»;
- `-1` — направлены строго противоположно.

Маленький численный пример:

```
a = [1, 0],  b = [1, 1]
dot(a, b) = 1*1 + 0*1 = 1
l2_norm(a) = 1,  l2_norm(b) = sqrt(2)
cos = 1 / (1 * sqrt(2)) = 0.7071...   # угол 45°, как и ждали
```

**ПОЧЕМУ это важно.** В мире эмбеддингов (слова, картинки, пользователи превращаются в векторы)
именно косинусная близость отвечает на вопрос «насколько похожи два объекта». Поисковики,
рекомендации, RAG, кластеризация — все сравнивают эмбеддинги косинусом, потому что важно
направление смысла, а не длина вектора. Заметь: косинус — это скалярное произведение **единичных**
векторов (длины 1). Мы как бы сначала нормируем оба вектора, а потом считаем dot.

**Форма.** Два вектора `(n,)` -> одно число `float` в `[-1, 1]`.

**Подводный камень.** Если хоть один вектор нулевой, его норма равна нулю, и деление невозможно.
В нашем контракте это случай-ошибка (см. ниже).

## Идея 5. Транспонирование и линейный слой целиком

**Интуиция.** Транспонировать матрицу — значит **отразить её относительно главной диагонали**:
строки становятся столбцами, столбцы — строками. Обозначается `A^T` (в numpy — `A.T`). Элемент на
позиции `[i, j]` уходит на позицию `[j, i]`:

```
A^T[i, j] = A[j, i]
```

Форма при этом **переворачивается**: `(n, k) -> (k, n)`. Был `n` строк и `k` столбцов — стало `k`
строк и `n` столбцов.

Маленький численный пример `2x3 -> 3x2`:

```
A =  [[1, 2, 3],          A^T = [[1, 4],
      [4, 5, 6]]                 [2, 5],
                                 [3, 6]]
форма (2, 3)              форма (3, 2)
```

Проверь по определению: `A[0,1] = 2` оказался в `A^T[1,0] = 2`; `A[1,2] = 6` — в `A^T[2,1] = 6`.
Транспонировать дважды — вернуться к исходному: `(A^T)^T = A`.

**Правило для произведения (важнейшее).** Транспонирование переворачивает порядок в матричном
умножении:

```
(A @ B)^T = B^T @ A^T          # порядок множителей МЕНЯЕТСЯ местами
```

Почему именно так (набросок доказательства). По определению левая часть в позиции `[i, j]` — это
элемент `(A @ B)` на позиции `[j, i]`, то есть скалярное произведение `j`-й строки `A` на `i`-й
столбец `B`: `sum_p A[j, p] * B[p, i]`. Правая часть в позиции `[i, j]` — это `dot(i`-я строка
`B^T, j`-й столбец `A^T)`, а `i`-я строка `B^T` это `i`-й столбец `B`, и `j`-й столбец `A^T` это
`j`-я строка `A`. Получаем ту же сумму `sum_p B[p, i] * A[j, p]`. Суммы совпадают поэлементно →
матрицы равны. (Заодно понятно, почему порядок переворачивается: только так сходятся формы —
`(m, k) @ (k, n)`.)

**ПОЧЕМУ это важно для DL.** Транспонирование — не украшение, оно неизбежно всплывает в обратном
проходе (backprop). Если forward слоя это `Y = X @ W` (формы `(N, D) @ (D, H) -> (N, H)`), то
градиент функции потерь по весам считается так:

```
dL/dW = X^T @ dL/dY          # формы (D, N) @ (N, H) -> (D, H)
```

Заметь: чтобы формы сошлись с `W` (которая `(D, H)`), `X` приходится транспонировать в `(D, N)`.
Именно `X^T` «выравнивает» оси так, чтобы сумма набралась по батчу (по N примерам). Ты ещё вернёшься
к этой формуле в модуле про backprop (07) — но корень её здесь: транспонирование меняет, по какой
оси «схлопывается» скалярное произведение.

**Форма.** Вход — матрица `(n, k)`, выход — матрица `(k, n)`. Данные те же, переставлены оси.

**Подводный камень.** `A.T` в numpy — это **view** (вид на те же данные, без копии), а не новая
матрица. Менять элементы транспонированного вида опасно — изменится и оригинал. Для самостоятельной
копии берут `A.T.copy()`. Для чтения (как в наших формулах) view идеален и бесплатен.

1. `dot(a: np.ndarray, b: np.ndarray) -> float`

2. `matmul(A: np.ndarray, B: np.ndarray) -> np.ndarray`
   Матричное умножение: `A` формы `(n, k)`, `B` формы `(k, m)` -> результат формы `(n, m)`.
   `C[i, j]` = скалярное произведение i-й строки `A` и j-го столбца `B`. Если внутренние размеры

   `np.ndarray` типа `float`.

3. `l2_norm(v: np.ndarray) -> float`
   Евклидова длина вектора `(n,)`: корень из суммы квадратов. Верни `float >= 0`.

4. `cosine_similarity(a: np.ndarray, b: np.ndarray) -> float`
   Косинусная близость: `dot(a, b) / (l2_norm(a) * l2_norm(b))`. Результат в `[-1, 1]`. Если длины
   векторов не совпадают — `ValueError`; если хотя бы один вектор нулевой (норма 0) — тоже
   `ValueError` (делить на ноль нельзя).

5. `linear_forward(X: np.ndarray, W: np.ndarray, b: np.ndarray) -> np.ndarray`
   Прямой проход линейного слоя: `Y = X @ W + b`. `X` формы `(N, D)`, `W` формы `(D, H)`,
   `b` формы `(H,)` -> результат формы `(N, H)`, тип `float`, где
   `Y[i, j] = dot(X[i, :], W[:, j]) + b[j]`. Переиспользуй свой `matmul` (без `@`), а `b` прибавь

### Подсказки

<details><summary>dot — с чего начать</summary>
Проверь, что <code>a.shape == b.shape</code> (или хотя бы что длины равны); если нет — <code>raise
ValueError</code>. Затем заведи аккумулятор <code>s = 0.0</code> и в цикле по индексам прибавляй
<code>a[i] * b[i]</code>. В конце верни <code>float(s)</code>. Векторизованный вариант
(<code>float(np.sum(a * b))</code>) тоже допустим — поэлементное умножение и сумма это не «жульни­
чество», запрещён именно <code>np.dot</code>.
</details>

<details><summary>matmul — три цикла и согласование форм</summary>
Размеры: <code>n, k = A.shape</code> и <code>k2, m = B.shape</code>. Если <code>k != k2</code> —
<code>raise ValueError</code>. Заведи результат нужной формы: <code>C = np.zeros((n, m))</code>.
Тройной цикл: для каждой строки <code>i</code> (0..n-1), каждого столбца <code>j</code> (0..m-1)
накапливай <code>C[i, j] += A[i, p] * B[p, j]</code> по <code>p</code> (0..k-1). Это и есть
«скалярное произведение i-й строки на j-й столбец».
</details>

<details><summary>matmul — а можно покороче?</summary>
Да: внутренний цикл по <code>p</code> — это ровно <code>dot(A[i, :], B[:, j])</code>. Если уже
написал <code>dot</code>, тело сведётся к двум циклам: <code>C[i, j] = dot(A[i], B[:, j])</code>.
Помни: <code>B[:, j]</code> — это j-й столбец (срез по всем строкам, фиксированный столбец).
</details>

<details><summary>l2_norm — связь со скалярным произведением</summary>
Длина — это <code>sqrt(сумма квадратов)</code>. А сумма квадратов это <code>dot(v, v)</code>.
Значит <code>return float(np.sqrt(dot(v, v)))</code>. Можно и без <code>dot</code>:
<code>float(np.sqrt(np.sum(v * v)))</code>. <code>np.linalg.norm</code> использовать нельзя.
</details>

<details><summary>cosine_similarity — порядок проверок</summary>
Сначала проверь длины (<code>ValueError</code> при несовпадении). Посчитай <code>na = l2_norm(a)</code>
и <code>nb = l2_norm(b)</code>. Если <code>na == 0</code> или <code>nb == 0</code> — <code>raise
ValueError</code> (иначе деление на ноль). Иначе верни <code>dot(a, b) / (na * nb)</code>, обёрнутое
в <code>float(...)</code>.
</details>

<details><summary>linear_forward — с чего начать (слабая подсказка)</summary>
Разбери задачу на два шага, которые ты уже умеешь. Шаг 1 — «сырые» выходы нейронов: это ровно
матричное умножение <code>X @ W</code>, то есть твой <code>matmul(X, W)</code>. Какую форму он
вернёт, если <code>X</code> это <code>(N, D)</code>, а <code>W</code> это <code>(D, H)</code>?
Шаг 2 — добавить сдвиг <code>b</code>. Подумай: к чему именно прибавляется <code>b</code> и какой он
формы по сравнению с результатом шага 1.
</details>

<details><summary>linear_forward — про формы и проверки (сильнее)</summary>
Достань размеры: <code>D = X.shape[1]</code>, а <code>D2, H = W.shape</code>. Согласование как в
<code>matmul</code>: если <code>D != D2</code> — <code>raise ValueError</code>. Дополнительно
проверь bias: его форма обязана быть <code>(H,)</code> (по одному числу на выходной нейрон); если
<code>b.shape != (H,)</code> — тоже <code>ValueError</code>. Само сложение
<code>matmul(X, W) + b</code> работает «само»: numpy растянет <code>(H,)</code> по всем N строкам
(broadcast). Не пиши цикл по примерам — это лишнее. Возвращать вне <code>float(...)</code>: результат
и так <code>np.ndarray</code> типа float.
</details>

Когда станет зелёным — покажи мне свою `matmul`, и мы обсудим, почему numpy делает то же самое в
сотни раз быстрее (спойлер: ваш тройной цикл на Python медленный, но логику ты теперь знаешь
насквозь — а это и есть цель).
