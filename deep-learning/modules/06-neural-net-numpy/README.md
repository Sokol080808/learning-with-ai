# Модуль 06 — Нейросеть с нуля: forward pass

В прошлых модулях ты складывал и перемножал массивы numpy, считал градиенты руками на
скалярах, разбирался с broadcasting. Теперь мы собираем из этих кирпичиков **первую
настоящую нейросеть** — пока только её «прямой ход» (forward pass): от входных чисел к
предсказанию. Обучение (backprop) будет в следующем модуле; сначала надо твёрдо понять,
что именно сеть **вычисляет**.

Большая идея модуля: *нейросеть — это просто цепочка из линейных преобразований и
нелинейностей*. Линейное преобразование умеет смешивать признаки и масштабировать их;
нелинейность ломает «прямизну» и даёт сети возможность выражать сложные зависимости. Всё
вместе — это обычная функция `f(x)`, которую можно записать в пять строк numpy. Никакой
магии: матрицы, `max(0, ·)` и нормировка вероятностей.

Дальше по тексту я постоянно проговариваю **формы тензоров** словами — это самый частый
источник ошибок. Привыкай держать формы в голове, как держишь размерности в физике.

## Идея 1. Слой = линейное преобразование + нелинейность

Один «слой» сети делает две вещи подряд:

1. **Линейное преобразование** `z = x·W + b`. Здесь вход `x` — это батч из `N` примеров,
   каждый длиной `in` признаков, форма `(N, in)`. Матрица весов `W` имеет форму
   `(in, out)`, вектор сдвига `b` — форму `(out,)`. На выходе `z` имеет форму `(N, out)`:
   каждый из `N` примеров превратился в `out` новых чисел. По сути это `out` разных
   взвешенных сумм исходных признаков, плюс сдвиг.

2. **Нелинейность** `a = g(z)` — поэлементная функция (форма не меняется, `(N, out)`).

**ПОЧЕМУ нужна нелинейность?** Допустим, мы поставили два линейных слоя подряд без `g`
между ними: `(x·W1 + b1)·W2 + b2`. Раскрой скобки — получится `x·(W1·W2) + (b1·W2 + b2)`,
то есть **снова одно линейное преобразование** с новыми весами `W1·W2`. Сколько линейных
слоёв ни ставь, без нелинейности между ними сеть умеет ровно столько же, сколько один
слой — рисовать прямые границы. Нелинейность `g` — это то, что делает «глубину» осмысленной.

Маленький пример форм: `x` это `(4, 3)` (4 примера по 3 признака), `W` это `(3, 5)`,
`b` это `(5,)`. Тогда `x·W` это `(4, 5)`, прибавляем `b` (broadcasting по строкам) — и
выход слоя `(4, 5)`.

## Идея 2. ReLU и softmax

Две нелинейности, которые тебе понадобятся.

**ReLU** (rectified linear unit) — самая популярная активация в скрытых слоях:

```
relu(z) = max(0, z)   (поэлементно)
```

Отрицательные значения обнуляются, положительные проходят как есть. Форма не меняется.
Пример: `relu([-2, -0.5, 0, 3])` = `[0, 0, 0, 3]`. **ПОЧЕМУ именно она?** Дёшево считать,
не «насыщается» для больших положительных значений (в отличие от сигмоиды), а её
производная — это просто 0 или 1, что очень удобно для backprop в следующем модуле.

**Softmax** превращает вектор произвольных чисел (логитов) в распределение вероятностей —
неотрицательные числа, которые суммируются в 1:

```
softmax(z)_i = exp(z_i) / sum_j exp(z_j)
```

Это «мягкий argmax»: самый большой логит получает наибольшую вероятность, но остальные
тоже ненулевые. Применяется по **последней оси**: если вход `(N, C)` (N примеров, C
классов), то softmax считается отдельно для каждой строки, и каждая строка выхода
суммируется в 1; форма остаётся `(N, C)`.

**Численная стабильность — обязательна.** `exp(1000)` переполнит float и даст `inf`.
Спасает простой трюк: вычти из каждой строки её максимум **перед** `exp`. Это не меняет
результат (множитель `exp(-max)` сокращается в числителе и знаменателе), но удерживает
аргумент `exp` в диапазоне `(-∞, 0]`:

```
z' = z - max(z)        # по последней оси, с keepdims
softmax(z) = exp(z') / sum(exp(z'))
```

Маленький пример: `softmax([1, 1, 1])` = `[1/3, 1/3, 1/3]` — все равны, значит равные
вероятности. А `softmax([0, 0, 100])` ≈ `[0, 0, 1]` — третий класс уверенно побеждает.

## Идея 3. Многоклассовый выход: логиты → вероятности

Последний слой сети для классификации на `C` классов выдаёт `C` чисел на каждый пример —
это **логиты**. Логит — это «сырой счёт» класса: чем больше, тем увереннее сеть в этом
классе. Логиты могут быть любыми: отрицательными, большими, какими угодно.

Чтобы из логитов получить **вероятности**, мы прогоняем их через softmax (по оси классов).
Форма пути: вход `(N, in)` → ... → логиты `(N, C)` → softmax → вероятности `(N, C)`, где в
каждой строке `C` чисел, сумма равна 1.

**ПОЧЕМУ держать логиты и вероятности раздельно?** Во-первых, для предсказания класса
softmax не нужен вовсе — argmax по логитам и по вероятностям даёт один и тот же класс
(softmax монотонен). Во-вторых, и это важнее, функция потерь (cross-entropy) численно
устойчивее, когда считается **прямо из логитов**, объединяя в себе log и softmax (см.
Идею 5). Поэтому слой сети возвращает логиты, а не вероятности — это осознанное решение,
а не лень.

Чтобы получить предсказанный класс примера: `argmax` по последней оси логитов даёт индекс
класса с наибольшим счётом. Для `(N, C)` это вектор длины `N` из индексов в `[0, C)`.

## Идея 4. Forward двухслойной сети

Собираем всё вместе. Двухслойная сеть (один скрытый слой + выходной слой) — это композиция:

```
h      = relu(x·W1 + b1)     # скрытый слой: линейка + ReLU
logits = h·W2 + b2           # выходной слой: только линейка (БЕЗ нелинейности)
```

Проследи формы на примере: вход `x` это `(N, D)`, скрытая размерность `H`, число классов `C`.

- `W1` это `(D, H)`, `b1` это `(H,)`. Тогда `x·W1 + b1` это `(N, H)`, и после ReLU — `(N, H)`.
- `W2` это `(H, C)`, `b2` это `(C,)`. Тогда `h·W2 + b2` это `(N, C)` — это логиты.

**ПОЧЕМУ на выходе нет нелинейности?** Потому что дальше идёт softmax + cross-entropy, и
именно они отвечают за превращение логитов в вероятности и за «сигнал ошибки». Ставить
ещё одну активацию на выход — лишнее (и для большинства функций потерь вредное). Скрытый
слой получает ReLU (см. Идею 1: иначе два линейных слоя схлопнутся в один), а выходной —
нет.

Это и есть весь forward pass MLP (multi-layer perceptron). Тот же приём — «линейка,
нелинейность, линейка, нелинейность, …» — масштабируется на любое число слоёв.

## Идея 5. Cross-entropy: насколько мы ошиблись

Чтобы потом сеть **обучать**, нужно число — насколько предсказание далеко от правды.
Для классификации это **кросс-энтропия**. У каждого примера есть правильный класс `y_i`
(целое число — индекс класса, не one-hot). Потеря для одного примера:

```
loss_i = -log( p_{i, y_i} )     где p = softmax(logits)
```

То есть: берём вероятность, которую сеть присвоила **правильному** классу, и считаем
`-log` от неё. Если сеть уверена и права (вероятность ≈ 1), `-log(1) = 0` — штраф нулевой.
Если сеть присвоила правильному классу маленькую вероятность (≈ 0), `-log(0) → +∞` —
штраф огромный. Итоговый loss по батчу — это **среднее** `loss_i` по всем `N` примерам
(одно число, скаляр).

**Численно устойчивая форма.** Не считай `softmax`, а потом `log` — `log(0)` даст `-inf`.
Вместо этого раскрой формулу. По определению `p_{i,c} = exp(z_{i,c}) / sum_k exp(z_{i,k})`,
поэтому:

```
-log p_{i, y_i} = -z_{i, y_i} + log( sum_k exp(z_{i,k}) )
```

А `log(sum(exp(·)))` считают через тот же сдвиг на максимум (log-sum-exp trick), что и в
softmax — вычитаем `max` по строке перед `exp`, прибавляем его обратно к логарифму. Это
устойчиво при любых логитах.

Маленький пример. Пусть для одного примера логиты `[2.0, 1.0, 0.1]`, правильный класс — 0.
Softmax даёт примерно `[0.66, 0.24, 0.10]`. Потеря = `-log(0.66)` ≈ `0.42` — сеть в целом
права, но не абсолютно уверена, штраф небольшой. Если бы правильным был класс 2 с
вероятностью `0.10`, потеря была бы `-log(0.10)` ≈ `2.30` — заметно больше.

Формы: `logits` это `(N, C)`, `y` это `(N,)` целочисленный вектор индексов в `[0, C)`,
результат — **один float** (среднее по батчу).

1. `relu(x: np.ndarray) -> np.ndarray` — поэлементный `max(0, x)`. Форма выхода = форме
   входа.

2. `softmax(x: np.ndarray) -> np.ndarray` — softmax **по последней оси**, численно
   стабильный (сдвиг на максимум перед `exp`). Форма не меняется; каждая строка (срез по
   последней оси) суммируется в 1.

3. `linear(x: np.ndarray, W: np.ndarray, b: np.ndarray) -> np.ndarray` — `x·W + b`.
   `x` это `(N, in)`, `W` это `(in, out)`, `b` это `(out,)`; выход `(N, out)`.

4. `forward2(x, W1, b1, W2, b2) -> np.ndarray` — forward двухслойной сети, возвращает
   **логиты** `(N, C)`. Между слоями — ReLU; на выходе нелинейности **нет**. Используй
   `linear` и `relu`, не дублируй код.

5. `cross_entropy(logits: np.ndarray, y: np.ndarray) -> float` — средняя кросс-энтропия по
   батчу. `logits` это `(N, C)`, `y` это `(N,)` (целочисленные индексы классов). Возвращай
   **обычный python `float`**, считай численно устойчиво (log-sum-exp).

### Подсказки

<details><summary>relu</summary>
`np.maximum(0, x)` — поэлементный максимум между нулём (broadcast) и `x`. Это не то же
самое, что `np.max`, которая схлопывает ось. Тебе нужна именно `np.maximum`.
</details>

<details><summary>softmax и численная стабильность</summary>
Найди максимум по последней оси с сохранением размерности:
`m = x.max(axis=-1, keepdims=True)`. `keepdims=True` оставляет форму `(N, 1)`, чтобы
вычитание `x - m` корректно broadcast'илось по строкам. Дальше `e = np.exp(x - m)` и
`e / e.sum(axis=-1, keepdims=True)`. Проверь себя: сумма каждой строки выхода должна быть 1.
</details>

<details><summary>linear и формы</summary>
Матричное умножение — это `x @ W` (оператор `@`) или `np.matmul`. `b` формы `(out,)`
прибавляется к каждой строке автоматически (broadcasting `(N, out) + (out,)`). Если
ловишь ошибку формы — выпиши на бумажке `(N, in) · (in, out)` и проверь, что внутренние
размерности `in` совпадают.
</details>

<details><summary>forward2</summary>
Буквально две строки поверх готовых кирпичиков:
`h = relu(linear(x, W1, b1))`, затем `logits = linear(h, W2, b2)`. Обрати внимание: на
второй `linear` **ReLU не навешиваем** — выход это логиты.
</details>

<details><summary>cross_entropy без переполнений</summary>
Не зови свой `softmax`, а потом `log`. Посчитай `log_sum_exp` устойчиво:
`m = logits.max(axis=1, keepdims=True)`; `lse = m + np.log(np.exp(logits - m).sum(axis=1, keepdims=True))`.
Логит правильного класса достань продвинутой индексацией:
`correct = logits[np.arange(N), y]` (форма `(N,)`). Тогда потеря примера —
`lse.ravel() - correct`, а ответ — её среднее: `float(np.mean(...))`.
</details>

Когда всё зелёное — останови взгляд на том, что ты построил: это полноценный прямой ход
нейросети, написанный руками, без единого вызова фреймворка. В следующем модуле мы научим
её **учиться** — посчитаем, как менять `W1, b1, W2, b2`, чтобы этот самый `cross_entropy`
уменьшался. Спроси меня, если хочешь заранее разобрать, откуда берутся градиенты.
