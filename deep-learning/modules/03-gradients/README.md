# Модуль 03 — Производные, цепное правило, градиенты

**Большая идея модуля:** обучение нейросети — это спуск с горы в тумане. Мы не видим всю
гору (всю функцию потерь), но в каждой точке можем нащупать **наклон под ногами** и шагнуть
вниз. «Наклон» в одной переменной — это **производная**. «Наклон» сразу по всем параметрам —
это **градиент**. А **цепное правило** — это правило, которое позволяет посчитать наклон даже
у длинной цепочки вложенных операций (а сеть — это именно длинная цепочка). Освоишь эти три
вещи — и весь backprop в модуле 07 будет для тебя не магией, а арифметикой.

В этом модуле всё руками на numpy, без torch. Мы намеренно сначала научимся считать градиент
**численно** (тупо, медленно, но почти без шансов ошибиться), а потом — **аналитически**
(быстро, точно, но надо думать). Численный градиент станет нашим «детектором лжи»: им мы будем
проверять аналитические формулы. Этот приём (`gradient checking`) спасёт тебя в модуле 07,
когда руками написанный backprop вдруг «почти работает».

## Идея 1. Производная — это скорость изменения

Возьми функцию одной переменной `f(x)`. Производная `f'(x)` отвечает на вопрос: **если я чуть-чуть
сдвину `x`, насколько и в какую сторону сдвинется `f`?** Формально это предел

```
f'(x) = lim_{h→0} ( f(x + h) − f(x) ) / h
```

Читается по-человечески так: берём приращение входа `h`, смотрим приращение выхода
`f(x+h) − f(x)`, делим одно на другое — получаем «выход на единицу входа». Устремляем `h` к нулю,
чтобы это была мгновенная скорость, а не средняя.

**Почему это для нас важно.** Знак производной говорит, куда двигать `x`, чтобы `f` росла
(в сторону `+f'`) или убывала (в сторону `−f'`). Обучение — это минимизация `f` (функции потерь),
поэтому мы всегда шагаем **против** производной. Величина `|f'|` говорит, насколько круто —
большой наклон — большой шаг имеет смысл, почти ноль — мы у дна.

**Маленький численный пример.** Пусть `f(x) = x²`, тогда честно (аналитически) `f'(x) = 2x`.
В точке `x = 3`: `f'(3) = 6`. Проверим «на пальцах» приращением `h = 0.001`:

```
( f(3.001) − f(3) ) / 0.001 = (9.006001 − 9) / 0.001 = 6.001
```

Почти ровно `6`. Лишняя `0.001` — это ошибка метода (мы взяли не нулевое `h`). Запомни эту
картину: **формула даёт 6, численная проба даёт ≈6** — так мы и будем сверять реализации.

## Идея 2. Градиент — вектор частных производных

В реальности `f` зависит не от одного числа, а от **вектора** `x = (x₁, x₂, …, x_D)` (форма
`x: (D,)`). Тогда у `f` нет одной производной — есть **частная производная по каждой координате**:
«насколько изменится `f`, если шевельнуть только `x_i`, а остальные держать замороженными».
Сложим эти частные производные в один вектор той же формы, что и `x` — это и есть **градиент**:

```
∇f(x) = ( ∂f/∂x₁, ∂f/∂x₂, …, ∂f/∂x_D )    — форма (D,), как у самого x
```

**Почему вектор, а не число.** Каждая координата `x_i` тянет `f` в свою сторону со своей силой.
Градиент собирает все эти «тяги» вместе. Его геометрический смысл: `∇f` показывает направление
**самого быстрого роста** `f`. Значит `−∇f` — направление самого быстрого спуска. Весь
градиентный спуск умещается в строчку `x ← x − lr · ∇f(x)`.

**Маленький пример.** `f(x) = x₁² + x₂²` (сумма квадратов). Частные производные:
`∂f/∂x₁ = 2x₁`, `∂f/∂x₂ = 2x₂`. Значит `∇f = (2x₁, 2x₂) = 2·x`. В точке `x = (3, −4)`
градиент равен `(6, −8)`. Форма градиента `(2,)` — ровно как у `x`. Это общее правило:
**градиент всегда имеет форму того, по чему дифференцируем.** Если параметр — матрица `(D, H)`,
то и градиент по ней — матрица `(D, H)`.

## Идея 3. Цепное правило — фундамент backprop

Сеть — это композиция функций: вход проходит слой, потом ещё слой, потом считается loss.
Символически `loss = f(g(h(x)))`. Как найти производную такой матрёшки? **Цепное правило:**
производные перемножаются вдоль цепочки.

Для одной переменной: если `y = g(x)` и `z = f(y)`, то

```
dz/dx = (dz/dy) · (dy/dx)
```

Читается как «скорость `z` по `x` = скорость `z` по `y`, умноженная на скорость `y` по `x`».
Интуиция-рычаг: если `y` меняется в 2 раза быстрее `x`, а `z` — в 3 раза быстрее `y`, то `z`
меняется в `3·2 = 6` раз быстрее `x`. Локальные скорости **перемножаются**.

**Численный пример.** Пусть `y = 3x` (значит `dy/dx = 3`) и `z = y²` (значит `dz/dy = 2y`).
В точке `x = 2`: `y = 6`, `dz/dy = 12`, `dy/dx = 3`, поэтому `dz/dx = 12·3 = 36`.
Проверка «в лоб»: `z(x) = (3x)² = 9x²`, `dz/dx = 18x`, при `x = 2` это `36`. Сходится.

**Почему это фундамент backprop.** Backprop — это и есть цепное правило, применённое от loss
назад ко всем параметрам, с переиспользованием уже посчитанных кусочков (чтобы не считать одно
и то же дважды). Сейчас мы это правило прочувствуем на простых функциях; в модуле 07 развернём
его на целую сеть.

## Идея 4. Численный градиент — центральная разность

Производную можно оценить, не зная формулы, — просто пошевелив вход. Наивный способ
(`forward difference`) `( f(x+h) − f(x) ) / h` работает, но грубоват. Гораздо точнее
**центральная разность** — шевелим в обе стороны:

```
∂f/∂x_i ≈ ( f(x + h·eᵢ) − f(x − h·eᵢ) ) / (2h)
```

где `eᵢ` — шаг только по `i`-й координате (остальные не трогаем). По очереди возмущаем каждую
координату вектора и собираем все ответы в массив формы `(D,)` — получаем численный градиент.

**Почему центральная разность, а не односторонняя.** Разложим в ряд Тейлора: у односторонней
разности ошибка пропорциональна `h` (порядок `O(h)`), а у центральной симметричные члены
сокращаются, и ошибка пропорциональна `h²` (порядок `O(h²)`). На `h = 1e-5` это разница между
ошибкой `~1e-5` и `~1e-10` — то есть почти даром получаем на порядки точнее.

**Численный пример.** `f(x) = sum(x²)` в точке `x = (1, 2, 3)`, `h = 1e-5`. По первой координате:
`( f(1+h,2,3) − f(1−h,2,3) ) / (2h) ≈ 2.0`. И по остальным — `4.0` и `6.0`. То есть численный
градиент `≈ (2, 4, 6)`, что совпадает с аналитическим `2·x = (2, 4, 6)`. Вот так численный метод
и работает «детектором лжи» для аналитики.

**Подводный камень — выбор `h`.** Слишком большой `h` — плохо приближаем производную (ошибка
метода). Слишком маленький `h` — числа `f(x+h)` и `f(x−h)` становятся почти равны, и при вычитании
теряются значащие цифры (ошибка округления float). Золотая середина для float64 — около `1e-5`.

## Задания

Открой `gradients.py` и реализуй функции так, чтобы тесты позеленели. Везде работаем с numpy,
сиды в тестах зафиксированы. Проговаривай форму вслух: вход `x: (D,)` → градиент тоже `(D,)`.

1. **`numerical_gradient(f, x, eps=1e-5) -> np.ndarray`**
   Численный градиент скалярной функции `f` в точке `x` методом **центральной разности**.
   `f` принимает массив формы `x.shape` и возвращает одно число (скаляр). Возмущай **каждую**
   координату по очереди на `±eps`, считай `( f(x+) − f(x−) ) / (2·eps)`. Результат — массив той же
   формы, что `x`. Важно: не порти исходный `x` (работай с копией) и оставляй его неизменным после
   вызова.

2. **`grad_sum_squares(x) -> np.ndarray`**
   Аналитический градиент функции `f(x) = sum(x²)`. Ответ — `2·x`, форма как у `x`.

3. **`grad_of_affine(x, w) -> np.ndarray`**
   Аналитический градиент функции `f(x) = dot(w, x)` (скалярное произведение) **по `x`**.
   Поскольку `f` линейна по `x`, ответ — это просто `w` (форма как у `x` и `w`, они совпадают).

Прогон тестов модуля:

```bash
./deep-learning/run.sh 03
```

### Подсказки

<details>
<summary>Подсказка 1 — с чего начать численный градиент</summary>

Сделай `grad = np.zeros_like(x, dtype=float)`. Заведи `x_plus = x.astype(float).copy()` и
`x_minus = x.astype(float).copy()`. Пробегись по индексам через `np.ndindex(x.shape)` — это
аккуратно перебирает координаты для любой формы, не только 1D.

</details>

<details>
<summary>Подсказка 2 — одна координата за шаг</summary>

Внутри цикла по индексу `idx`: прибавь `eps` к `x_plus[idx]`, вычти `eps` из `x_minus[idx]`,
посчитай `f(x_plus)` и `f(x_minus)`, запиши в `grad[idx]` значение `(fp − fm) / (2*eps)`.
Затем **верни** `x_plus[idx]` и `x_minus[idx]` к исходным значениям `x[idx]`, чтобы следующая
координата считалась от чистой точки. Иначе возмущения «накопятся» и всё поедет.

</details>

<details>
<summary>Подсказка 3 — аналитические формулы</summary>

`grad_sum_squares`: производная `x²` по `x` равна `2x`, по координатам это `2*x` — одна строчка,
не забудь привести к float, чтобы не было целочисленного деления неожиданностей.
`grad_of_affine`: `f = w₁x₁ + … + w_Dx_D`, частная по `x_i` равна `w_i`, то есть весь градиент —
это просто `w` (верни копию, чтобы не отдавать наружу тот же объект, что пришёл).

</details>

<details>
<summary>Подсказка 4 — почему численный и аналитический должны совпасть</summary>

Тесты сравнивают твой `numerical_gradient` с твоими же аналитическими формулами через
`np.allclose` с разумным `atol`. Если они не сходятся — ищи баг в одной из двух реализаций.
Чаще всего виноваты: забыл «откатить» возмущение координаты, делил на `eps` вместо `2*eps`,
или испортил исходный `x`. Это и есть `gradient checking` в миниатюре — твой главный инструмент
отладки в модуле 07.

</details>
