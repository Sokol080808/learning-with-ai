# Установка и окружение

Хорошая новость: ставить вручную почти ничего не нужно. Скрипт `run.sh` при первом
запуске сам создаёт виртуальное окружение и ставит всё необходимое. От тебя нужен только
подходящий **Python** и интернет **один раз**.

---

## 1. Python 3.11–3.13 (важно — не 3.14)

Для PyTorch нужен Python, под который собраны готовые «колёса» (wheels). На момент курса
это **3.11, 3.12 или 3.13**.

> **Не используй Python 3.14.** Он слишком свежий: готовых колёс torch под него может ещё
> не быть, и установка упадёт на сборке из исходников. Если у тебя только 3.14 — поставь
> рядом 3.13 (см. заметки по ОС ниже), системный Python трогать не обязательно.

Проверь, что нужная версия есть:

```bash
python3.13 --version    # ожидаем: Python 3.13.x
# или
python3.12 --version
python3.11 --version
```

Скрипт `run.sh` сам ищет совместимый интерпретатор в таком порядке:
`python3.12 → python3.13 → python3.11 → python3.10 → python3`. Тебе достаточно, чтобы
хотя бы один из них был установлен.

---

## 2. Что поставит скрипт сам (при первом запуске)

При первом `./deep-learning/run.sh` скрипт:

1. создаёт виртуальное окружение `deep-learning/.venv`;
2. ставит туда **numpy** (модули 00–10), **torch** (CPU-версия, модули 11–17 и капстоун)
   и **pytest** (проверка заданий).

> **Первый запуск долгий.** `torch` — большой пакет (сотни мегабайт). Его скачивание
> займёт время и **требует интернета один раз**. Дальше окружение лежит локально, и тесты
> идут за секунды — интернет больше не нужен.

Ничего глобально в систему не ставится: всё живёт в `.venv` внутри папки курса. Хочешь
снести и переустановить начисто — `./deep-learning/run.sh clean`, затем снова `run.sh`.

---

## 3. GPU не нужен

Весь курс считается на **CPU**. Данные в тестах крошечные и синтетические (генерируются
прямо в тесте), модели маленькие, итераций мало — всё проходит за секунды. **Видеокарта,
CUDA, облачные GPU — ничего этого не требуется.** Ставится именно CPU-сборка torch, она
легче и проще.

---

## 4. Опционально: matplotlib

Для самого курса он **не нужен** — тесты от него не зависят. Но если захочешь сам
порисовать графики (как падает loss, как выглядит граница решения, как меняются веса),
поставь его в то же окружение:

```bash
./deep-learning/.venv/bin/pip install matplotlib
```

Это твоя личная песочница для экспериментов; на проверку заданий не влияет.

---

## 5. Проверка, что всё работает

После первого запуска `run.sh` убедись, что пакеты на месте:

```bash
./deep-learning/.venv/bin/python -c "import numpy; print('numpy', numpy.__version__)"
./deep-learning/.venv/bin/python -c "import torch; print('torch', torch.__version__)"
./deep-learning/.venv/bin/python -c "import torch; print('CUDA доступна:', torch.cuda.is_available())"
```

Последняя строка почти наверняка напечатает `CUDA доступна: False` — и это **нормально**,
мы и так на CPU. Если все три команды отработали без ошибок импорта — окружение готово.

Прогон тестов одного модуля как финальная проверка (красные тесты — это ожидаемо, это
твои задания; главное, что они **запускаются**, а не падают на импорте):

```bash
./deep-learning/run.sh 00
```

## 6. Заметки по ОС

**macOS**
- Системный Python лучше не трогать. Поставь нужную версию через [Homebrew](https://brew.sh):
  `brew install python@3.13`. После этого появится `python3.13`.
- На Apple Silicon (M1/M2/M3…) ставится обычная CPU-сборка torch — это нормально, всё
  работает. Можно не разбираться с MPS/Metal: курсу хватает CPU.

**Linux**
- Через пакетный менеджер дистрибутива (`apt`, `dnf`, …) или через
  [pyenv](https://github.com/pyenv/pyenv), если нужной версии нет в репозиториях.
  Пример для Debian/Ubuntu: `sudo apt install python3.13 python3.13-venv`.
- Пакет `*-venv` обязателен — без него `python -m venv` не создаст окружение.

**Windows**
- Проще всего работать через **WSL2** (Ubuntu внутри Windows): тогда все команды из курса
  (`./deep-learning/run.sh …`) работают как есть, без правок.
- Нативно на Windows можно через [python.org](https://www.python.org/downloads/) (поставь
  3.13) и Git Bash, но скрипт `run.sh` рассчитан на bash-окружение — WSL2 надёжнее.

---

## Если что-то пошло не так

- **`Не найден python3`** — нет ни одного подходящего интерпретатора. Поставь Python
  3.11–3.13 (см. выше) и попробуй снова.
- **Установка падает на сборке torch** — почти всегда это Python 3.14 (нет колёс). Поставь
  3.13 рядом и перезапусти `run.sh clean`, затем `run.sh`.
- **Долгая «тишина» при первом запуске** — это качается torch (сотни МБ). Дай ему время и
  проверь интернет.

Застрял на установке — покажи мне точный текст ошибки и вывод `python3 --version`. Разберём.
