# Глоссарий курса Deep Learning

Короткие определения терминов, которые встречаются в модулях. Это не замена теории из
README — это **шпаргалка для быстрого вспоминания**. Рядом с термином указан модуль, где
он разбирается подробно. Если определение туманно — открой модуль или спроси меня.

> Совет: не зубри это. Возвращайся сюда, когда встретил слово и забыл, что оно значит. За
> словом должно стоять понимание, а не заученная строчка.

## Тензоры и формы

- **Тензор** — многомерный массив чисел: число (0D), вектор (1D), матрица (2D) и выше.
  Главный объект DL; в numpy это `ndarray`, в PyTorch — `Tensor`. *(01)*
- **Форма (shape)** — кортеж размеров по осям. `x: (N, D)` — N объектов по D признаков.
  90% багов в DL — это перепутанные формы; всегда проговаривай форму вслух. *(01)*
- **Ось (axis/dim)** — измерение тензора. Многие операции (`sum`, `mean`, `softmax`)
  принимают `axis=` — «вдоль какой оси сворачиваем». *(01)*
- **Broadcasting** — правило numpy/torch «растягивать» меньшую форму до большей без
  копий, чтобы складывать/умножать тензоры разных форм (например, `(N, D) + (D,)`). *(01)*
- **Батч (batch)** — пачка объектов, обрабатываемых за один проход; обычно первая ось `N`.
  Считать батчем сразу — быстрее и стабильнее, чем по одному. *(01, 12, 16)*

## Линейная алгебра и слои

- **Матричное умножение (matmul, `@`)** — основа линейного слоя; `(N, D) @ (D, H) → (N, H)`.
  Внутренние размерности должны совпадать. *(02)*
- **Линейный (полносвязный) слой** — `y = xW + b`. Геометрически: поворот/масштаб/сдвиг
  пространства признаков. `W: (D, H)`, `b: (H,)`. *(02, 06)*
- **Веса и смещения (weights, bias)** — обучаемые параметры `W` и `b`; именно их меняет
  обучение. *(02, 04)*

## Производные и градиенты

- **Производная** — чувствительность функции к малому изменению входа: «насколько
  изменится выход, если чуть сдвинуть вход». *(03)*
- **Градиент** — вектор частных производных по всем параметрам; показывает направление
  самого быстрого роста. Учимся, двигаясь **против** градиента. *(03, 04)*
- **Цепное правило (chain rule)** — производная композиции: `(f∘g)' = f'(g)·g'`.
  Математический фундамент всего обратного распространения. *(03, 07)*

## Обучение и функции потерь

- **Loss (функция потерь)** — число «насколько модель ошибается»; обучение = минимизация
  loss. *(04, 05)*
- **MSE (среднеквадратичная ошибка)** — `mean((y_pred − y)²)`; loss для регрессии. *(04)*
- **Кросс-энтропия** — loss для классификации; штрафует за низкую вероятность,
  присвоенную правильному классу. *(05, 06)*
- **Градиентный спуск** — шаг за шагом сдвигаем параметры против градиента loss:
  `θ ← θ − lr · ∇loss`. *(04, 09)*
- **Learning rate (скорость обучения, `lr`)** — размер шага. Слишком большой —
  «перелетаем» и расходимся; слишком малый — учимся мучительно долго. *(04, 09, 16)*
- **Эпоха / шаг (epoch / step)** — эпоха = один проход по всем данным; шаг = одно
  обновление параметров (обычно на одном батче). *(12, 16)*

## Функции активации

- **Сигмоида (sigmoid)** — `1/(1+e^{−x})`, сжимает число в `(0, 1)`; вероятность для
  бинарной классификации. *(05)*
- **Softmax** — превращает вектор чисел в распределение вероятностей (положительные,
  сумма = 1); выход многоклассовой классификации. *(06)*
- **ReLU** — `max(0, x)`; самая ходовая нелинейность. Без нелинейностей сеть из многих
  слоёв эквивалентна одному линейному. *(06)*

## Forward, backprop, autograd

- **Forward pass (прямой проход)** — данные текут через слои от входа к предсказанию и
  loss. *(06)*
- **Backprop (обратное распространение)** — считаем градиенты loss по всем параметрам,
  проходя граф в обратном порядке и применяя цепное правило. Сердце DL. *(07)*
- **Граф вычислений** — запись операций как графа; по нему backprop знает, как «развернуть»
  производные назад. *(07, 08)*
- **Autograd (автодифференцирование)** — механизм, считающий градиенты автоматически по
  графу вычислений. В модуле 08 ты пишешь свой (`micrograd`), в 11 — пользуешься
  torch.autograd. *(08, 11)*
- **`Value` (micrograd)** — учебный скалярный узел графа с полями `data`, `grad` и методом
  `backward()`; мини-версия torch-тензора с автоградом. *(08)*

## Оптимизаторы

- **SGD (стохастический градиентный спуск)** — базовый оптимизатор: шаг против градиента
  на батче данных. *(09)*
- **Momentum (инерция)** — накапливает «скорость» по прошлым градиентам; быстрее в
  оврагах, меньше дёргается. *(09)*
- **Adam** — адаптивный оптимизатор: свой эффективный шаг на каждый параметр (momentum +
  масштаб по истории квадратов градиентов). Рабочая лошадка по умолчанию. *(09, 16)*

## Обобщение и регуляризация

- **Переобучение (overfitting)** — модель «зубрит» обучающие данные и плохо работает на
  новых; train-loss маленький, val-loss большой. *(10)*
- **Train / validation (обучающая / валидационная выборка)** — учимся на train, честно
  оцениваем на отложенном val. *(10, 16)*
- **Регуляризация** — приёмы против переобучения: упростить/«придержать» модель. *(10)*
- **L2-регуляризация (weight decay)** — штраф за большие веса; делает модель «спокойнее». *(10)*
- **Dropout** — во время обучения случайно «выключает» часть нейронов; мешает сети
  полагаться на отдельные связи. *(10)*
- **Ранняя остановка (early stopping)** — прекращаем обучение, когда val-loss перестал
  улучшаться. *(10, 16)*

## PyTorch

- **`torch.Tensor`** — тензор PyTorch; как numpy-массив, но с автоградом и (опционально)
  GPU. В курсе — CPU. *(11)*
- **`requires_grad` / `.backward()`** — пометить тензор «по нему нужны градиенты» и
  запустить backprop по графу; градиенты появятся в `.grad`. *(11)*
- **`nn.Module`** — базовый класс модели/слоя в PyTorch; хранит параметры, реализует
  `forward`. *(12)*
- **Оптимизатор (`torch.optim`)** — объект, обновляющий параметры (`optim.SGD`,
  `optim.Adam`); цикл `zero_grad → backward → step`. *(12, 16)*
- **`state_dict`** — словарь всех обучаемых параметров модели; то, что сохраняешь и
  загружаешь для чекпойнтов. *(17)*

## Архитектуры

- **Свёртка (convolution) / фильтр (kernel)** — небольшое окно весов, скользящее по входу;
  ловит локальные паттерны (края, текстуры). Основа CNN. *(13)*
- **Пулинг (pooling)** — уменьшение пространственного размера (`max`/`average`); меньше
  данных, устойчивость к сдвигам. *(13)*
- **CNN (свёрточная сеть)** — сеть из свёрток и пулинга; «родной» инструмент для
  изображений. *(13)*
- **Эмбеддинг (embedding)** — обучаемая таблица «id → вектор»; превращает слова/символы в
  числа, с которыми работает сеть. *(14, 15)*
- **RNN (рекуррентная сеть)** — обрабатывает последовательность шаг за шагом, нося с собой
  «память» (скрытое состояние). *(14)*

## Внимание и трансформеры

- **Внимание (attention)** — механизм «на что смотреть»: каждый элемент собирает
  информацию с других, взвешивая по релевантности. *(15)*
- **Query / Key / Value (запрос / ключ / значение)** — три проекции входа: query сравнивают
  с keys (схожесть → веса), затем этими весами усредняют values. *(15)*
- **Self-attention (самовнимание)** — внимание последовательности на саму себя; каждый
  токен смотрит на остальные токены той же последовательности. *(15)*
- **Causal mask (причинная маска)** — запрещает токену «подсматривать» в будущее (видеть
  только себя и левее); нужна для авторегрессионной генерации. *(15, капстоун)*
- **LayerNorm (нормализация слоя)** — нормирует активации по признакам; стабилизирует и
  ускоряет обучение глубоких сетей. *(15)*
- **Трансформер** — архитектура на стеке блоков «внимание + полносвязная сеть» с
  residual-связями и LayerNorm; основа современных языковых моделей. *(15, капстоун)*
- **Residual-связь (skip connection)** — `x + f(x)`; помогает градиентам течь сквозь
  глубокую сеть, упрощает обучение. *(15)*

## Языковые модели (капстоун)

- **Char-level (посимвольная) модель** — словарь = символы текста; модель предсказывает
  следующий символ. *(капстоун)*
- **Токенизация** — превращение текста в последовательность id (у нас — по символам). *(14, капстоун)*
- **Авторегрессия** — генерация по одному токену за раз: предсказали → добавили к входу →
  предсказываем следующий. *(капстоун)*
- **nanoGPT** — минималистичный GPT-подобный трансформер; ориентир капстоуна. *(капстоун)*
