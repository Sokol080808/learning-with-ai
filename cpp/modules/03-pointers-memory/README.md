# Модуль 03 — Указатели, ссылки и модель памяти

Это сердце C++ и то, чем он отличается от Python/JS. Здесь ты впервые работаешь с
**памятью напрямую**. Будет непривычно — это нормально. Поняв этот модуль, ты поймёшь
половину «странностей» C++.

## Идея 1. Стек и куча

- **Стек (stack)** — где живут локальные переменные. Память выделяется автоматически
  при входе в функцию и освобождается при выходе. Быстро, но время жизни ограничено
  областью видимости `{ ... }`.
- **Куча (heap)** — память «по запросу», живёт пока ты её не освободишь. Гибко, но
  ответственность на тебе. В современном C++ кучу почти всегда трогают через
  контейнеры и умные указатели (модули 05–06), а не вручную.

```cpp
int x = 5;          // x на стеке, исчезнет в конце блока
int* p = new int(5); // значение на куче; p (сам указатель) — на стеке
delete p;            // обязательно освободить, иначе утечка
```
> В этом модуле мы НЕ пишем `new`/`delete` в заданиях — сначала надо понять сами
> указатели. К ручному управлению и почему его избегать вернёмся в модулях 05–06.

## Идея 2. Что такое указатель

Указатель — это переменная, которая хранит **адрес** другой переменной.

```cpp
int x = 42;
int* p = &x;   // & — «взять адрес». p указывает на x
std::cout << *p;   // * — «разыменование»: значение по адресу == 42
*p = 100;          // меняем x ЧЕРЕЗ указатель; теперь x == 100
```
Два разных `*`: в объявлении `int* p` звёздочка — часть типа («указатель на int»);
в выражении `*p` — операция разыменования. И `&` тоже двулик: в типе `int&` — ссылка,
в выражении `&x` — взятие адреса.

## Идея 3. `nullptr` — указатель «в никуда»

```cpp
int* p = nullptr;   // ни на что не указывает
if (p != nullptr) { /* безопасно разыменовывать */ }
```
**Разыменование `nullptr` — это краш (UB).** Всегда думай: может ли указатель быть
нулевым? Если да — проверяй перед `*p`.

## Идея 4. Массивы и арифметика указателей

Имя массива легко «распадается» в указатель на первый элемент:

```cpp
int a[3] = {10, 20, 30};
int* p = a;        // p указывает на a[0]
*(p + 1);          // == a[1] == 20   (p+1 сдвигает на ОДИН int, а не на 1 байт)
p[2];              // == a[2] == 30   (индексация — это и есть *(p+2))
```
Важно: «сырой» массив **не знает свой размер**. Поэтому в функцию его передают вместе
с длиной: `f(int* arr, int n)`. (Позже `std::vector`/`std::array` решат это за нас.)

## Идея 5. Указатель vs ссылка

| | Указатель `int*` | Ссылка `int&` |
|---|---|---|
| может быть «пустым» | да (`nullptr`) | нет, всегда привязана |
| можно перенаправить | да | нет, навсегда к одному объекту |
| синтаксис доступа | `*p`, `p->` | как обычная переменная |

Правило большого пальца: если значение **обязано** существовать и не меняет «адресата» —
бери ссылку (проще и безопаснее). Если может отсутствовать — указатель.

## Идея 6. const-correctness с указателями

```cpp
const int* p;        // нельзя менять *p (значение), можно менять p (куда смотрит)
int* const p = &x;   // можно менять *p, нельзя менять p
const int* const p;  // нельзя ни то, ни другое
```
Читай справа налево: `const int*` → «pointer to const int». `const&`-параметр —
твой друг: даёт доступ без копии и обещает не портить данные.

## Идея 7. Висячие указатели/ссылки (dangling)

```cpp
int* bad() {
    int local = 5;
    return &local;   // ОПАСНО: local умрёт, адрес станет мусором
}
```
Никогда не возвращай адрес/ссылку на локальную переменную. Это источник коварных багов.
