# Модуль 01 — Типы, переменные, `const` и ссылки

Большая идея модуля: в C++ **тип — это контракт и обещание о памяти**. От типа зависит,
сколько байт занимает значение, какие операции допустимы и как они себя ведут. Здесь
много языков «прощают» небрежность, а C++ — нет, и это его сила: ошибки видны рано.

## Идея 1. Фундаментальные типы и их размер

- Целые: `int`, `long`, `short`, `unsigned …`. Размер зависит от платформы! Когда
  важна точная ширина — бери `<cstdint>`: `int32_t`, `int64_t`, `uint8_t` и т.д.
- Дробные: `double` (по умолчанию выбирай его), `float` (меньше точность).
- `bool` (`true`/`false`), `char` (один байт, символ).

```cpp
#include <cstdint>
int32_t a = 5;     // ровно 32 бита
double  x = 2.5;   // дробное
bool    ok = true;
```

## Идея 2. Целочисленное деление — классическая ловушка

```cpp
int a = 7, b = 2;
a / b;            // == 3, а не 3.5 ! деление двух int — целочисленное
double r = a / b; // == 3.0 — деление уже произошло в int, поздно
double r2 = static_cast<double>(a) / b; // == 3.5 — приводим ДО деления
```

Запомни правило: тип результата определяется типами **операндов**, а не тем, куда ты
присваиваешь. Хочешь дробное деление — сделай хотя бы один операнд дробным.

## Идея 3. `auto` — пусть компилятор выведет тип

```cpp
auto n = 42;          // int
auto x = 3.14;        // double
auto s = std::string{"hi"};
```

`auto` не делает язык «динамическим» — тип всё равно фиксируется на этапе компиляции,
просто его выводит компилятор. Это удобно и уменьшает дублирование.

## Идея 4. `const` — «я обещаю это не менять»

```cpp
const int limit = 100;   // limit менять нельзя
limit = 200;             // ошибка компиляции — и это хорошо
```

`const` — не про оптимизацию, а про **намерение** и защиту от случайных ошибок. Чем
больше `const`, тем меньше способов сломать код по неосторожности. Это важная привычка.

## Идея 5. Ссылки (`&`) и передача в функции — value vs reference

Когда ты передаёшь аргумент **по значению**, функция получает *копию*:

```cpp
void f(int x) { x = 99; }   // меняет КОПИЮ, снаружи ничего не изменится
```

**Ссылка** (`int&`) — это псевдоним для существующей переменной. Через неё функция
меняет оригинал:

```cpp
void g(int& x) { x = 99; }  // меняет ОРИГИНАЛ
```

А `const&` — «дай доступ без копии, но менять не буду» (для больших объектов это и
быстро, и безопасно — к этому вернёмся не раз):

```cpp
void h(const std::string& s);  // не копируем строку и не портим её
```

## Идея 6. Ввод/вывод

```cpp
#include <iostream>
int x;
std::cin >> x;                       // прочитать число
std::cout << "x = " << x << "\n";    // вывести
```

В заданиях ввод/вывод не нужен (тесты вызывают функции напрямую), но это базовый
навык — поэкспериментируй отдельно.

<details><summary>Задание 4 — почему void и ссылка</summary>
Функция ничего не возвращает (`void`), но обязана изменить переданную переменную.
Это возможно только если параметр — ссылка (`int&`), а не копия (`int`).
</details>

</details>
