# Модуль 02 — Управление потоком и функции

Большая идея: программа — это **решения** (что выполнять) и **повторения** (сколько
раз), упакованные в **функции** (переиспользуемые блоки с именем). Хорошая программа
читается как текст: маленькие функции с понятными именами, каждая делает одно дело.

## Идея 1. Ветвление: `if`, `else`, тернарный оператор, `switch`

```cpp
if (n > 0)        std::cout << "положительное";
else if (n < 0)   std::cout << "отрицательное";
else              std::cout << "ноль";

int sign = (n >= 0) ? 1 : -1;   // тернарный оператор: условие ? a : b

switch (op) {                   // switch — выбор по значению
    case '+': return a + b;
    case '-': return a - b;
    default:  return 0;         // если ничего не подошло
}
```
Важно про `switch`: после `case` без `break`/`return` выполнение «проваливается» в
следующий `case`. В этом курсе мы обычно `return` прямо из `case`.

## Идея 2. Циклы: `while`, `for`, range-based for

```cpp
for (int i = 0; i < n; ++i) { /* i от 0 до n-1 */ }

int i = 0;
while (i < n) { ++i; }

for (int x : {10, 20, 30}) { /* x по очереди = 10,20,30 */ }  // range-based for
```
`++i` (преинкремент) и `i++` (постинкремент) оба увеличивают `i`; в заголовке цикла
разницы по смыслу нет, привыкай писать `++i`.

## Идея 3. Функции: объявление и определение

```cpp
int square(int n);            // объявление (прототип): «такая функция есть»
int square(int n) { return n * n; }   // определение: тело
```
В заголовочном файле (`.hpp`) — **объявления**, в `.cpp` — **определения**. Так
несколько файлов могут пользоваться функцией, видя только её прототип. Это и есть
разделение интерфейса и реализации, на котором держатся большие проекты.

## Идея 4. Рекурсия

Функция может вызывать саму себя. Главное — **база** (когда остановиться):

```cpp
int factorial(int n) {
    if (n <= 1) return 1;          // база
    return n * factorial(n - 1);   // шаг к базе
}
```
Рекурсию почти всегда можно переписать циклом, и наоборот. Выбирай то, что **читается
понятнее** для конкретной задачи.

## Идея 5. Перегрузка и значения по умолчанию

Можно иметь несколько функций с одним именем, если различаются параметры —
компилятор выберет подходящую (**перегрузка**):

```cpp
int area(int side);          // квадрат
int area(int w, int h);      // прямоугольник
```
И задавать значения по умолчанию:
```cpp
int power(int base, int exp = 2);   // power(5) == 25
```

## Задания

Файл `src/task02.cpp` (интерфейс — `include/task02.hpp`):

1. `int factorial(int n)` — `n!` (считаем `factorial(0) == 1`).
2. `int fib(int n)` — n-е число Фибоначчи: `fib(0)=0, fib(1)=1, fib(n)=fib(n-1)+fib(n-2)`.
3. `int gcd(int a, int b)` — наибольший общий делитель (алгоритм Евклида).
4. `std::string fizzbuzz(int n)` — `"Fizz"` если делится на 3, `"Buzz"` если на 5,
   `"FizzBuzz"` если на 15, иначе само число строкой (см. `std::to_string`).
5. `int area(int side)` и `int area(int w, int h)` — **перегрузка**: площадь квадрата
   и прямоугольника.

## Микро-проект: CLI-калькулятор

В папке `project/` — мини-приложение. Ядро — функция
`double calc(double a, char op, double b)` (`project/include/calc.hpp`,
реализация в `project/src/calc.cpp`), которая выполняет `+ - * /`. Её проверяют тесты.

Файл `project/src/main.cpp` — это **готовая** программа: читает `a op b` со
стандартного ввода и печатает результат, используя твою `calc`. Когда реализуешь
`calc`, собери и запусти приложение:

```bash
./cpp/run.sh 02                  # тесты модуля
echo "3 * 4" | ./cpp/build/modules/02-control-flow/m02_calc_app
```

> В тестах `calc` гарантируется, что оператор — один из `+ - * /`, а делитель не ноль.
> Проверку «ноль/неизвестный оператор» делает `main.cpp`. Полноценную обработку ошибок
> мы разберём в модуле 09 — пока просто посмотри, как это сделано в `main.cpp`.

### Подсказки

<details><summary>gcd — алгоритм Евклида</summary>
Пока `b != 0`: одновременно `a, b = b, a % b`. Когда `b` стал нулём — ответ в `a`.
</details>
<details><summary>fizzbuzz — порядок проверок</summary>
Сначала проверяй делимость на 15 (или одновременно на 3 и на 5), иначе «FizzBuzz»
никогда не сработает.
</details>

Покажи мне свою `calc` или любую из функций — обсудим, что можно улучшить.
