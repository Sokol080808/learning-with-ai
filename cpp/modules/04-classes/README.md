# Модуль 04 — Классы, ООП и введение в RAII

Большая идея: класс связывает **данные** и **поведение** и устанавливает **инварианты** —
правила, которые всегда истинны для корректного объекта. Хороший класс невозможно
привести в «сломанное» состояние извне. Это и есть инкапсуляция.

## Идея 1. `struct` vs `class`

Технически разница одна: в `struct` члены по умолчанию **public**, в `class` —
**private**. По соглашению: `struct` — для простых «мешков данных» без инвариантов,
`class` — когда есть логика и что защищать.

## Идея 2. Инкапсуляция и спецификаторы доступа

```cpp
class Fraction {
public:                 // интерфейс — доступен всем
    Fraction(int num, int den);
    int numerator() const;
private:                // реализация — скрыта, видна только классу
    int num_;
    int den_;
};
```
Поля делаем `private`, доступ — через методы. Зачем? Чтобы никто не выставил, скажем,
знаменатель в 0 в обход правил. Класс сам стоит на страже своих инвариантов.

## Идея 3. Конструктор и список инициализации

Конструктор приводит объект в корректное начальное состояние:

```cpp
Fraction::Fraction(int num, int den)
    : num_(num), den_(den)   // список инициализации членов (предпочтительно)
{
    // здесь можно дополнительно нормализовать (сократить дробь и т.п.)
}
```
Инициализируй члены в **списке инициализации** (после `:`), а не присваиванием в теле —
это эффективнее и иногда единственно возможно (для `const`-членов и ссылок).

## Идея 4. `const`-методы

```cpp
int numerator() const;   // const после () — «этот метод не меняет объект»
```
Помечай `const` каждый метод, который только читает. Тогда такой метод можно вызывать
у `const`-объектов, и компилятор поймает случайное изменение. Это часть const-correctness.

## Идея 5. Деструктор и RAII (главная идея модуля!)

**Деструктор** вызывается автоматически, когда объект уничтожается (выходит из области
видимости). Из этого рождается ключевая идиома C++ — **RAII** (Resource Acquisition Is
Initialization):

> Ресурс (память, файл, сокет, блокировка) захватывается в **конструкторе** и
> освобождается в **деструкторе**. Тогда он освобождается *сам* — гарантированно,
> даже при исключении.

```cpp
class FileGuard {
public:
    FileGuard(const char* path) { f_ = std::fopen(path, "r"); }  // захват
    ~FileGuard() { if (f_) std::fclose(f_); }                    // освобождение
private:
    std::FILE* f_;
};
// Файл закроется автоматически, как только объект FileGuard выйдет из области видимости.
```
RAII — причина, по которой в современном C++ почти не пишут `delete` руками: за тебя
это делают деструкторы контейнеров и умных указателей. Запомни эту идею — мы будем
возвращаться к ней постоянно (модули 05, 06, 09).

## Идея 6. Перегрузка операторов

Класс может определить, что для него значат `+`, `==` и т.д.:

```cpp
bool operator==(const Fraction& other) const;   // a == b
```
Перегружай операторы только когда смысл очевиден (для дроби `+` — естественно).

## Задание: класс `Fraction` (рациональная дробь)

Интерфейс — `include/fraction.hpp` (он зафиксирован), реализуй методы в
`src/fraction.cpp`. Инвариант, который нужно поддерживать **всегда**:

- дробь хранится в **сокращённом** виде (делим на НОД), напр. `2/4` → `1/2`;
- знак хранится в числителе, знаменатель всегда **положительный**: `1/-2` → `-1/2`;
- ноль представляется как `0/1`.

Методы:
1. Конструктор `Fraction(int num, int den)` — сразу нормализует дробь (тесты не
   передают `den == 0`).
2. `int numerator() const`, `int denominator() const`.
3. `Fraction add(const Fraction& o) const` — сумма.
4. `Fraction multiply(const Fraction& o) const` — произведение.
5. `bool operator==(const Fraction& o) const`.
6. `std::string to_string() const` — вид `"num/den"` (например `"-1/2"`).
