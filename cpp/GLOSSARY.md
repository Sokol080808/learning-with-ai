# Глоссарий курса

Короткие определения терминов, которые встречаются в модулях. Это не замена теории из
README — это **шпаргалка для быстрого вспоминания**. Рядом с термином указан модуль, где
он разбирается подробно. Если определение непонятно — открой модуль или спроси меня.

> Совет: не зубри это. Возвращайся сюда, когда встретил слово и забыл, что оно значит.

## Сборка и инструменты

- **Translation unit (единица трансляции)** — один `.cpp` со всеми подставленными
  `#include`. Компилятор обрабатывает по одной такой единице за раз. *(00)*
- **Препроцессор** — этап до компиляции: обрабатывает `#include`, `#define`. `#include`
  буквально вставляет содержимое заголовка. *(00)*
- **Компиляция** — превращение `.cpp` в объектный файл `.o` (машинный код с «дырами» на
  месте внешних функций). *(00)*
- **Линковка (компоновка)** — сшивание объектных файлов и библиотек в программу.
  `undefined reference` — это ошибка **линковки**: функция объявлена, но не определена. *(00, 11)*
- **CMake** — система, которая по `CMakeLists.txt` генерирует команды сборки. *(00, 11)*
- **GoogleTest** — фреймворк тестирования. `TEST(Group, Case){ EXPECT_EQ(a,b); }`.
  `EXPECT_*` не прерывает тест при провале, `ASSERT_*` — прерывает. *(00)*
- **Санитайзер** — детектор ошибок в рантайме. ASan ловит выход за границы / use-after-free /
  утечки, UBSan — неопределённое поведение. Флаг `-fsanitize=address,undefined`. *(12)*
- **ODR (One Definition Rule)** — у каждой сущности ровно одно определение во всей
  программе. Объявлять можно много раз, определять — один. Тела `inline`/шаблонов/`constexpr`
  живут в заголовках как исключение. *(11)*

## Типы, значения, память

- **`const`** — обещание «не менять». Про намерение и защиту, а не про оптимизацию. *(01)*
- **Ссылка (`T&`)** — псевдоним существующей переменной; через неё меняют оригинал. *(01, 03)*
- **`const T&`** — доступ без копии и без права изменения (быстро и безопасно для крупных
  объектов). *(01)*
- **Целочисленное деление** — `7/2 == 3`. Тип результата определяют **операнды**, а не то,
  куда присваиваешь. Хочешь дробное — сделай операнд `double`. *(01)*
- **Указатель (`T*`)** — переменная, хранящая **адрес**. Может быть `nullptr`. Разыменование
  `*p`, доступ к полю `p->field`. *(03)*
- **Стек (stack)** — память под локальные переменные, освобождается автоматически при выходе
  из области видимости. **Куча (heap)** — память под `new`/`delete`, живёт пока не освободишь. *(03)*
- **Висячий указатель (dangling)** — указывает на уже освобождённую память. Обращение к нему — UB. *(03, 06)*
- **UB (Undefined Behavior)** — неопределённое поведение: выход за границы, разыменование
  `nullptr`, гонка данных. Может «работать», а может всё сломать. Избегаем всегда. *(03, 14)*

## Объекты и время жизни

- **RAII (Resource Acquisition Is Initialization)** — ресурс захватывается в конструкторе,
  освобождается в деструкторе. Сердце «безопасного» C++: память/файлы/мьютексы освобождаются
  сами. *(04, 06, 14)*
- **Конструктор / деструктор** — вызываются при создании / уничтожении объекта. *(04)*
- **lvalue / rvalue** — у lvalue есть имя и адрес (`x`), rvalue временный (`x + 1`, результат
  функции). Перемещение «крадёт» ресурсы у rvalue. *(05)*
- **Копирование vs перемещение** — копия дублирует данные; перемещение **переносит** владение,
  оставляя источник в валидном «пустом» состоянии (дёшево). *(05)*
- **Правило 0/3/5** — либо не пиши спец-методы вовсе (правило 0, лучший вариант), либо, если
  пишешь один из {деструктор, копи-конструктор, копи-присваивание}, продумай все 3 (и +2 move
  — итого 5). *(05)*
- **Умный указатель** — объект-владелец динамической памяти, освобождающий её автоматически
  (RAII). `unique_ptr` — единоличное владение, `shared_ptr` — разделяемое (счётчик ссылок),
  `weak_ptr` — наблюдение без владения. *(06)*

## Обобщённый код и STL

- **Шаблон (template)** — «рецепт» кода, параметризованный типом. `my_max<int>` и `my_max<double>`
  компилятор генерирует из одного шаблона. Тела шаблонов — в заголовках. *(07)*
- **Concept (C++20)** — ограничение на тип-параметр шаблона (`std::integral T`). Делает ошибки
  понятными и документирует требования. *(07)*
- **Контейнер** — структура для хранения элементов: `vector`, `map`, `unordered_map`, `set`… *(08)*
- **Итератор** — «указатель-в-контейнер»; алгоритмы работают через них, не зная типа контейнера. *(08)*
- **Алгоритм STL** — готовая функция (`std::sort`, `std::find`, `std::accumulate`), работающая
  через итераторы. *(08)*
- **Лямбда** — анонимная функция «на месте»: `[](int x){ return x*x; }`. `[&]`/`[=]` —
  захват окружения по ссылке/значению. *(08)*
- **`std::optional<T>`** — «значение или ничего» (`nullopt`). Замена «волшебным» -1/nullptr. *(09)*
- **`std::variant<A,B>`** — типобезопасное «или то, или это». *(09)*
- **Исключение (exception)** — механизм сообщить об ошибке через `throw` / `try`–`catch`. *(09)*
- **`auto`** — вывод типа компилятором (на этапе компиляции, не «динамика»). *(01, 10)*
- **Structured bindings** — `auto [k, v] = pair;` распаковка в несколько переменных. *(10)*
- **`constexpr`** — вычисляется на этапе компиляции. *(10)*
- **Ranges (C++20)** — композиция операций над диапазонами (`views::filter`, `views::transform`). *(10)*

## Алгоритмы и многопоточность

- **Big-O** — как растёт число операций с размером входа `n`: `O(1)`, `O(log n)`, `O(n)`,
  `O(n log n)`, `O(n²)`. Оцениваем рост, отбрасывая константы. *(13)*
- **Хеш-таблица** — `unordered_map`/`unordered_set`: поиск/вставка в среднем `O(1)`. Классика:
  заменить двойной цикл `O(n²)` на «проход + хеш-множество» `O(n)`. *(13)*
- **Бинарный поиск** — `O(log n)`, но только в **отсортированном** массиве. *(13)*
- **Поток (`std::thread`)** — независимая нить исполнения. Каждый поток нужно `join()` или
  `detach()`. *(14)*
- **Гонка данных (data race)** — несколько потоков меняют одни данные без синхронизации. Это UB. *(14)*
- **Мьютекс (`std::mutex`)** — взаимное исключение: критическую секцию выполняет один поток.
  Захватывай через `std::lock_guard` (RAII!). *(14)*
- **`std::atomic<T>`** — неделимые операции над переменной без мьютекса; для простых счётчиков. *(14)*
