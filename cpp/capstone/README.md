# Капстоун — Mini Key-Value Store (kvstore)

Это финальный pet-проект курса. Здесь нет пошаговых инструкций «сделай строку 5» — есть
**спецификация** и приёмочные тесты. Ты сам проектируешь решение, опираясь на всё, что
прошёл. Именно так выглядит работа junior: дана задача и критерии готовности.

## Что строим

Маленькое хранилище «ключ → значение» (как упрощённый Redis) с текстовым интерфейсом
(REPL — read-eval-print loop) и сохранением на диск. Пользователь вводит команды:

```
> SET name Anna
OK
> GET name
Anna
> SET greeting Hello World
OK
> GET greeting
Hello World
> COUNT
2
> KEYS
greeting name
> DEL name
1
> GET name
(nil)
> SAVE data.txt
OK
> QUIT
```

## Майлстоуны (привязка к модулям курса)

**Майлстоун 1 — ядро хранилища** (модули 04, 05, 08, 09)
Класс `KvStore` поверх `std::map<std::string,std::string>`:
- `set(key, value)` — вставить/перезаписать;
- `get(key) -> std::optional<std::string>` — значение или `nullopt`;
- `erase(key) -> bool` — `true`, если ключ существовал;
- `size()`; `keys() -> std::vector<std::string>` (отсортированные).
Тесты: `tests/test_kvstore.cpp` (часть Core).

**Майлстоун 2 — обработка команд** (модули 02, 08, 09)
`execute(KvStore&, const std::string& line) -> std::string` — разобрать строку и вернуть
ответ:
- `SET key value` → `"OK"` (значение — это весь остаток строки после ключа, может
  содержать пробелы);
- `GET key` → значение или `"(nil)"`;
- `DEL key` → `"1"` или `"0"`;
- `COUNT` → число записей строкой;
- `KEYS` → отсортированные ключи через пробел, или `"(empty)"`;
- неизвестная команда → `"ERR unknown command"`.
Тесты: `tests/test_command.cpp`.

**Майлстоун 3 — персистентность** (модули 09, 12)
- `save(path) -> bool` — записать все пары в файл (формат — на твой выбор, напр.
  `key<TAB>value` по строке);
- `load(path) -> bool` — прочитать файл, заменив текущее содержимое; при отсутствии
  файла вернуть `false`.
- Команды `SAVE path` / `LOAD path` в `execute` → `"OK"` или `"ERR io"`.
Тесты: часть Persistence в `tests/test_kvstore.cpp`.

**Майлстоун 4 — REPL и стретч-цели**
`app/main.cpp` уже собран и работает с твоими `KvStore`/`execute`. Запусти, поиграй.
Затем выбери одну-две стретч-цели (по желанию, на них тестов нет — это твоё
исследование, обсудим вместе):
- сделать значение обобщённым: `KvStore<Value>` через шаблон (модуль 07);
- фоновое автосохранение в отдельном потоке каждые N секунд (модуль 14) — аккуратно с
  синхронизацией доступа к данным (`std::mutex`);
- команда `KEYS prefix` — только ключи с заданным префиксом (ranges, модуль 10);
- прогнать всё под AddressSanitizer (модуль 12) и убедиться, что памяти не течёт.
