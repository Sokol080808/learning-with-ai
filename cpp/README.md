# Курс C++: от основ до уровня Junior

Это не сборник задачек «напиши код ради кода». Это **учебный курс**, цель которого —
чтобы ты *понимал*, как и почему работает C++, и к концу мог уверенно претендовать
на позицию **junior C++ разработчика**.

Стандарт: **C++20**. Проверка заданий: **CMake + GoogleTest**.

---

## Как устроен курс

Курс разбит на **модули** (`modules/00-setup` … `modules/14-concurrency`) и
финальный **pet-проект** (`capstone/`). Каждый модуль — это одна большая идея.

Внутри модуля:

```
modules/NN-name/
  README.md            ← теория простым языком + ПОЧЕМУ так + описание заданий + подсказки
  include/*.hpp        ← интерфейс: что именно нужно реализовать
  src/*.cpp            ← СТАБЫ с // TODO  ← ТЫ ПИШЕШЬ ЗДЕСЬ
  tests/*.cpp          ← готовые тесты    ← их трогать не нужно, это критерий «сдано»
  project/             ← (в некоторых модулях) микро-проект
```

Логика проста: тест знает, как должна вести себя твоя функция/класс. Ты дописываешь
стаб → запускаешь тесты → видишь, что прошло, а что нет → исправляешь.

## Что нужно установить

Всё уже есть в этом окружении. На своей машине понадобится:

- компилятор с поддержкой C++20: **g++ ≥ 11** или **clang++ ≥ 14**;
- **CMake ≥ 3.20**;
- интернет при первом запуске (CMake один раз скачает GoogleTest).

Проверь:
```bash
g++ --version
cmake --version
```

## Рабочий цикл (запомни его — он один на весь курс)

```bash
# собрать всё и прогнать ВСЕ тесты
./cpp/run.sh

# прогнать тесты только одного модуля (например, 01)
./cpp/run.sh 01

# удалить сборку и начать с чистого листа
./cpp/run.sh clean
```

В самом начале почти все тесты **красные** — это нормально, это и есть твои задания.
Твоя работа — делать их зелёными, по одному, понимая, что ты пишешь.

> Совет: запускай тесты часто. Маленький шаг → тест → следующий шаг. Так ты сразу
> видишь, где ошибся, и не копишь баги.

## Как со мной работать (режим «преподаватель»)

Я — твой преподаватель, а не решебник. Поэтому:

- **Я не выдаю готовых решений заданий по умолчанию.** Если попросишь «дай ответ» —
  сначала задам наводящие вопросы, помогу самому дойти до решения. Так знание
  остаётся в голове, а не в буфере обмена.
- **Спрашивай свободно.** «Почему здесь ссылка, а не значение?», «Я думаю сделать
  так — это нормально?», «Не понимаю, почему тест падает» — это правильные вопросы.
- **Показывай свой код.** Я отвечу, в верном ли направлении ты идёшь, на что
  обратить внимание, какие есть подводные камни.
- Если ты реально застрял — попроси **подсказку** (hint). В README модулей подсказки
  устроены ступенчато: сначала лёгкий намёк, потом сильнее.
- Полное решение я дам только если ты явно об этом попросишь, и всё равно с разбором
  «почему именно так».

Не бойся ошибаться — компилятор и тесты для того и нужны. Ошибка с понятным
сообщением — это бесплатный урок.

## Программа курса

**Часть I. Фундамент**
- **00 — Setup и как C++ собирается.** Препроцессор, компиляция, линковка; CMake и
  GoogleTest; цикл build → test → fix. Разминочное задание.
- **01 — Типы, переменные, ввод/вывод, `const` и ссылки.**
- **02 — Управление потоком и функции.** *Микро-проект: CLI-калькулятор.*

**Часть II. Память и объектная модель**
- **03 — Указатели, ссылки, модель памяти.**
- **04 — Классы и ООП, введение в RAII.**
- **05 — Копирование/перемещение, правило 0/3/5.** *Микро-проект: свой контейнер.*
- **06 — Умные указатели и динамическая память.**

**Часть III. Обобщённый и стандартный C++**
- **07 — Шаблоны и concepts (C++20).**
- **08 — STL: контейнеры, итераторы, алгоритмы, лямбды.** *Микро-проект: частотный словарь.*
- **09 — Обработка ошибок: исключения, `optional`, `variant`.**
- **10 — Современные идиомы: `auto`, structured bindings, `ranges`, `constexpr`.**

**Часть IV. Инженерия и CS-фундамент**
- **11 — Многофайловые проекты и библиотеки.** *Микро-проект: lib + app + tests.*
- **12 — Инструменты и качество: gdb, санитайзеры, clang-format/tidy.**
- **13 — Структуры данных и алгоритмы, Big-O.**
- **14 — Введение в многопоточность.**

**Капстоун (`capstone/`)** — большой pet-проект, в котором сходится всё.

Отмечай прогресс в [PROGRESS.md](PROGRESS.md).
