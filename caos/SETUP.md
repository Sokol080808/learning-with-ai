# Что установить перед началом курса CAOS

Курсу нужно немного: компилятор **C/C++** (он один умеет оба языка), **CMake** и `git`.
GoogleTest скачивать вручную не надо — CMake подтянет его сам при первой сборке (нужен
интернет один раз).

Для модуля 14 («Инструменты, ввод-вывод и отладка») пригодятся ещё **отладчик** и
**санитайзеры** — без них модуль тоже проходится, но с ними нагляднее. Что именно ставить,
зависит от ОС (см. ниже).

## Минимум

| Инструмент | Версия | Зачем |
|---|---|---|
| Компилятор C/C++ | **gcc ≥ 11** или **clang ≥ 14** | собирать твой C-код и C++-тесты |
| **CMake** | **≥ 3.20** | система сборки курса |
| git | любая свежая | версионировать свой прогресс |
| интернет | один раз | CMake скачает GoogleTest при первом запуске |

> Один пакет компилятора (gcc или clang) даёт сразу и C-компилятор, и C++-компилятор —
> ставить два инструмента не нужно.

## Установка по ОС

### Linux (Debian/Ubuntu)
```bash
sudo apt update
# компилятор + CMake + git:
sudo apt install -y build-essential cmake git
# инструменты отладки для модуля 14:
sudo apt install -y gdb valgrind strace
```
- **gdb** — отладчик: пошагово смотреть, что делает программа.
- **valgrind** — ищет ошибки памяти (выход за границы, утечки, обращение к освобождённому).
- **strace** — показывает, какие *системные вызовы* делает программа (пригодится в модулях
  про ОС и в 14-м).

Проверь, что `gcc --version` ≥ 11. Если нет — поставь новее (`sudo apt install gcc-12`).

### macOS
```bash
# Компилятор (Apple Clang) + lldb идут в Command Line Tools:
xcode-select --install
# CMake (и при желании современный gcc) через Homebrew:
brew install cmake
# опционально: brew install gcc
```
- **lldb** — отладчик на macOS (аналог gdb), входит в Command Line Tools.
- **Санитайзеры** (ASan/UBSan) встроены в clang из Command Line Tools — отдельно ставить
  не нужно. Включаются флагом `-fsanitize=address,undefined`.

> Важно про **valgrind на macOS**: на свежих версиях macOS он, как правило, **не работает**
> (не успевает за новыми чипами Apple Silicon и обновлениями системы). Это нормально — на
> macOS его роль выполняет **AddressSanitizer (ASan)**: ту же работу (поиск ошибок памяти)
> он делает встроенными средствами компилятора. В модуле 14 покажу, как им пользоваться.

### Windows
- Проще всего — **WSL** (подсистема Linux внутри Windows): поставь Ubuntu из Microsoft
  Store и дальше действуй по инструкции для **Linux** выше. Так доступны gdb/valgrind/strace,
  и `./caos/run.sh` запускается без плясок.
- Без WSL: поставь **MSYS2/MinGW-w64** (даёт gcc) + **CMake** с сайта cmake.org. Отладчики
  ОС-уровня (valgrind/strace) под чистым Windows недоступны — для модуля 14 всё равно удобнее
  WSL.

## Проверка

```bash
gcc --version      # или: clang --version  → C-компилятор на месте
g++ --version      # или: clang++ --version → C++-компилятор для тестов
cmake --version    # ≥ 3.20
git --version
```

Для модуля 14 (по своей ОС):
```bash
# Linux:
gdb --version && valgrind --version && strace -V
# macOS:
lldb --version    # valgrind не нужен — вместо него ASan (флаг -fsanitize=address)
```

Готово? Тогда первый прогон (соберёт всё и один раз скачает GoogleTest):
```bash
./caos/run.sh 00
```
Почти все тесты будут **красными** — это нормально, это твои задания. Дальше — по
[`README.md`](README.md), начиная с `modules/00-setup/README.md`.
