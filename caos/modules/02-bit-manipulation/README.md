# Модуль 02 — Битовые операции: маски, сдвиги, флаги

В модуле 01 ты разобрался, что число внутри компьютера — это просто набор битов
(нулей и единиц). Теперь научимся этими битами **управлять напрямую**: включать,
выключать, переключать и переставлять. Это первый момент в курсе, где ты по-настоящему
говоришь с машиной на её языке.

## Большая идея

Любое число `uint32_t` — это **32 лампочки в ряд**, каждая горит (1) или погашена (0).
Привычная арифметика (`+`, `-`, `*`) смотрит на ряд как на одно большое число.
А битовые операции смотрят на **каждую лампочку по отдельности** и позволяют щёлкать
выключателями. Зачем это нужно? Чтобы хранить много «да/нет» в одном числе (компактно),
проверять и менять их за один такт процессора (быстро), и понимать, что происходит на
уровне железа (а в этом весь смысл курса CAOS).

Договоримся о нумерации: биты считаем **справа налево, с нуля**. Самый правый
(младший) бит — это бит 0. В числе `0b1010` бит 0 = 0, бит 1 = 1, бит 2 = 0, бит 3 = 1.

```
число 0b1010 (это десятичное 10):

  бит:   3   2   1   0
         1   0   1   0
         ^               ^
      старший          младший
```

## Идея 1. Четыре базовые операции: AND, OR, XOR, NOT и сдвиги

Эти операции работают **побитово** — сравнивают i-й бит одного числа с i-м битом
другого и кладут результат в i-й бит ответа.

| Операция | Знак в C | Правило | Бытовая аналогия |
|---|---|---|---|
| AND (И) | `&` | 1, только если **оба** бита 1 | «фильтр»: пропускает только там, где у маски 1 |
| OR (ИЛИ) | `\|` | 1, если **хотя бы один** бит 1 | «включатель»: где у маски 1 — там точно загорится |
| XOR (исключающее ИЛИ) | `^` | 1, если биты **разные** | «переключатель»: где у маски 1 — там бит щёлкнет |
| NOT (НЕ) | `~` | инвертирует **каждый** бит | «инверсия»: все лампочки наоборот |

```c
0b1100 & 0b1010  ==  0b1000   // 1 только там, где у обоих 1
0b1100 | 0b1010  ==  0b1110   // 1 там, где хоть у одного 1
0b1100 ^ 0b1010  ==  0b0110   // 1 там, где биты разные
~0b00000000u      ==  0xFFFFFFFFu  // все нули стали единицами (для 32 бит)
```

Ещё два инструмента — **сдвиги**:

- `x << n` — сдвиг **влево** на `n` позиций. Все биты уезжают влево, справа дописываются
  нули. Это то же самое, что умножить на 2ⁿ.
- `x >> n` — сдвиг **вправо** на `n`. Биты уезжают вправо. Для беззнаковых
  (`uint32_t`) слева дописываются нули. Это деление на 2ⁿ.

```c
1u << 0   ==  0b0001   // единица на месте 0
1u << 3   ==  0b1000   // единица «уехала» на место 3
0b1000u >> 2  ==  0b0010
```

> **ПОЧЕМУ это важно.** Связка `1u << i` — твой главный инструмент в этом модуле.
> Она строит число, где горит **ровно один** бит — i-й. Имея такую «иголку», можно
> добраться до любого конкретного бита: проверить его, включить, выключить или
> переключить.

> **Подводный камень.** Сдвигать на число `>= 32` (ширина типа) — это
> **неопределённое поведение** в C: результат может быть любым. Не пиши `1u << 32`.
> И обязательно используй `1u` (беззнаковую единицу), а не `1`: у знакового `int`
> сдвиг в старший бит — тоже ловушка.

## Идея 2. Маски: упаковываем флаги в одно число

**Маска** — это просто число, которое ты используешь как трафарет: его биты говорят,
«с какими позициями работаем». Это позволяет хранить набор «галочек» в одном числе.

Представь настройки: жирный шрифт, курсив, подчёркивание. Можно завести три отдельные
переменные `bool`. А можно — три бита в одном `uint32_t`:

```c
#define BOLD      (1u << 0)   // 0b001
#define ITALIC    (1u << 1)   // 0b010
#define UNDERLINE (1u << 2)   // 0b100

uint32_t style = 0;
style |= BOLD;      // включить жирный  -> 0b001
style |= UNDERLINE; // включить подчёрк -> 0b101
if (style & ITALIC) { /* курсив включён? нет */ }
style &= ~BOLD;     // выключить жирный -> 0b100
```

Заметил приёмы? `OR` (`|=`) включает флаг, `AND` с маской (`& FLAG`) проверяет,
`AND` с инверсией (`&= ~FLAG`) выключает. Эти же три приёма ты реализуешь в заданиях.

> **ПОЧЕМУ это важно.** Так устроены реальные API: права доступа к файлу
> (чтение/запись/исполнение), флаги открытия файла, регистры устройств. Один
> `int` вместо десятка `bool` — это и память, и скорость, и совместимость с железом.

## Идея 3. Зачем биты вообще: компактность и скорость

Два аргумента, почему это не «трюк ради трюка»:

1. **Компактность.** 32 «да/нет» помещаются в одно 4-байтовое число вместо 32
   отдельных байтов (а `bool` в C++ обычно занимает целый байт). Это в 8 раз меньше
   памяти — критично для сетевых протоколов, форматов файлов, встраиваемых систем.
2. **Скорость.** `&`, `|`, `^`, `<<` — это **одна инструкция процессора**. Проверить
   25 флагов одним `&` — мгновенно. А ещё есть фокусы: `x & (x - 1)` гасит младший
   единичный бит, `x << 1` умножает на 2 быстрее, чем `* 2` в наивном коде.

## Идея 4. Полезные трюки на битах

Парочка приёмов, которые выглядят как магия, пока их не разберёшь:

- **Проверка степени двойки.** У степени двойки (1, 2, 4, 8, …) установлен **ровно
  один** бит. Хитрость: `x & (x - 1)` обнуляет самый младший единичный бит. Если после
  этого получился 0 — значит, единичный бит был один-единственный. Но не забудь
  отдельно отсечь `x == 0` (у нуля битов нет вообще, это не степень двойки).

  ```c
  // x = 0b1000 (это 8). x - 1 = 0b0111. x & (x-1) = 0b0000 -> степень двойки!
  // x = 0b1010 (это 10). x - 1 = 0b1001. x & (x-1) = 0b1000 -> НЕ степень двойки.
  ```

- **Перестановка байтов.** Число из 4 байт `[B3 B2 B1 B0]` иногда нужно перевернуть в
  `[B0 B1 B2 B3]`. Это пригодится в модуле 03, когда будем разбирать **порядок байт**
  (endianness) — почему один и тот же файл по-разному читается на разных машинах.
  Каждый байт вытаскивается сдвигом и маской `& 0xFF`, а потом ставится на новое место
  сдвигом в другую сторону.

  ```c
  // 0x11223344  ->  0x44332211
  // младший байт 0x44 уезжает в старший, и наоборот.
  ```

## Задания

Открой `src/bits.c` (интерфейс — `include/bits.h`). Шесть функций над `uint32_t`.
Заглушки уже компилируются, но возвращают неверный результат — почини их.

1. `int get_bit(uint32_t x, int i);` — вернуть i-й бит числа `x` (0 или 1).
2. `uint32_t set_bit(uint32_t x, int i);` — вернуть `x` с установленным в 1 i-м битом.
3. `uint32_t clear_bit(uint32_t x, int i);` — вернуть `x` со сброшенным в 0 i-м битом.
4. `uint32_t toggle_bit(uint32_t x, int i);` — вернуть `x` с инвертированным i-м битом.
5. `int is_power_of_two(uint32_t x);` — вернуть 1, если `x` — степень двойки и не ноль.
6. `uint32_t reverse_bytes(uint32_t x);` — вернуть `x` с перевёрнутым порядком 4 байт.

Во всех функциях, кроме `reverse_bytes` и `is_power_of_two`, **остальные биты трогать
нельзя** — меняем только i-й.

Запусти тесты:

```bash
./caos/run.sh 02
```

Чини, пока все случаи не станут зелёными.

### Подсказки (открывай, только если застрял)

<details><summary>Задание 1 — get_bit</summary>

Тебе нужен только i-й бит. Сдвинь `x` вправо на `i` позиций — тогда нужный бит окажется
самым младшим (на месте 0). А чтобы отбросить всё остальное слева, наложи маску `& 1u`.
Подумай: `(x >> i) & 1u` — что это даёт?
</details>

<details><summary>Задание 2 — set_bit</summary>

Нужна «иголка» с единицей ровно на i-м месте: это `1u << i`. Теперь подумай, какая
операция гарантированно ставит 1 там, где у маски 1, и не трогает остальное. Подсказка:
это `OR` (`|`).
</details>

<details><summary>Задание 3 — clear_bit</summary>

Снова возьми `1u << i`, но теперь её надо **инвертировать** (`~`): получится число, где
все биты — единицы, кроме i-го. Если наложить такую маску через `AND` (`&`), i-й бит
обнулится, а остальные сохранятся.
</details>

<details><summary>Задание 4 — toggle_bit</summary>

Какая операция «щёлкает» бит на противоположный, если у маски там 1? Это `XOR` (`^`).
`x ^ (1u << i)` — переключит ровно i-й бит.
</details>

<details><summary>Задание 5 — is_power_of_two</summary>

Сначала отдельно отсеки `x == 0` (ноль — не степень двойки). Потом вспомни трюк
`x & (x - 1)`: он гасит самый младший единичный бит. Если результат равен 0 — значит,
единичный бит был только один, и это степень двойки. Верни 1 или 0 (а не `bool`).
</details>

<details><summary>Задание 6 — reverse_bytes</summary>

Вытащи каждый из 4 байтов и поставь его на новое место. Младший байт: `x & 0xFFu` — его
надо сдвинуть влево на 24. Следующий байт: `(x >> 8) & 0xFFu` — сдвинуть влево на 8.
И так далее симметрично. Сложи (или объедини через `|`) четыре получившихся куска.
</details>
