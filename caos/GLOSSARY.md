# Глоссарий курса CAOS

Короткие определения терминов, которые встречаются в модулях. Это не замена теории из
README — это **шпаргалка для быстрого вспоминания**. Рядом с термином указан модуль, где он
разбирается подробно. Если определение непонятно — открой модуль или спроси меня.

> Совет: не зубри это. Возвращайся сюда, когда встретил слово и забыл, что оно значит.

## Сборка и как код становится битами

- **Бит** — наименьшая единица информации: 0 или 1. **Байт** — 8 бит; обычно наименьшая
  ячейка памяти, у которой есть адрес. *(00)*
- **Препроцессор** — этап до компиляции: обрабатывает `#include`, `#define`. *(00)*
- **Компиляция** — превращение `.c` в объектный файл `.o` (машинный код с «дырами» на месте
  внешних функций). **Линковка** сшивает `.o` и библиотеки в исполняемый файл. *(00)*
- **Машинный код / исполняемый файл** — числа, которые процессор понимает напрямую. Твой
  `.c` через конвейер превращается именно в них. *(00, 07)*
- **CMake** — система, которая по `CMakeLists.txt` генерирует команды сборки. *(00)*
- **GoogleTest** — фреймворк тестирования (тесты курса). `TEST(Group, Case){ EXPECT_EQ(a,b); }`.
  `EXPECT_*` не прерывает тест при провале, `ASSERT_*` — прерывает. *(00)*
- **`extern "C"`** — обёртка в заголовке, которая позволяет C++-тесту вызывать C-функции
  (отключает «искажение имён», name mangling). В заданиях уже проставлена. *(00)*

## Представление чисел

- **Двоичная система** — числа из 0 и 1 (основание 2): так данные хранятся физически. *(01)*
- **Шестнадцатеричная (hex)** — основание 16 (цифры 0–9 и A–F); компактная запись битов:
  одна hex-цифра = 4 бита. Пишется с префиксом `0x` (`0xFF` = 255). *(01)*
- **Дополнительный код (two's complement)** — стандартный способ хранить **отрицательные**
  целые: старший бит «весит» со знаком минус. Благодаря ему сложение работает одинаково для
  плюса и минуса. *(01, 03)*
- **Старший / младший бит (MSB / LSB)** — самый «дорогой» / самый «дешёвый» бит числа. В
  знаковых числах старший бит — знак. *(01)*

## Биты и битовые операции

- **Маска (bitmask)** — число, в котором интересующие биты выставлены в 1; через `&`/`|`
  «вырезают» или выставляют отдельные биты. *(02)*
- **Сдвиг (`<<`, `>>`)** — сдвинуть биты влево/вправо. Сдвиг влево на 1 ≈ умножение на 2. *(02)*
- **Флаги** — отдельные биты-«галочки», упакованные в одно число (экономия места и проверка
  одним сравнением). *(02)*
- **Побитовые `&` `|` `^` `~`** — И, ИЛИ, исключающее ИЛИ, инверсия. Работают по каждому
  биту отдельно (не путать с `&&`/`||`, которые про «истина/ложь»). *(02)*

## Целые числа и память чисел

- **Переполнение (overflow)** — результат не влез в тип (например, `int` «перескочил» через
  максимум и стал отрицательным). Источник коварных багов. *(03)*
- **Порядок байт (endianness)** — в каком порядке байты многобайтного числа лежат в памяти:
  **little-endian** (младший байт первым, x86) или **big-endian**. Важно при работе с файлами
  и сетью. *(03)*
- **Расширение знака (sign extension)** — при переводе числа в более широкий тип знаковое
  число дополняется копиями знакового бита, чтобы значение сохранилось. *(03)*
- **Знаковое / беззнаковое (`signed`/`unsigned`)** — может ли тип хранить отрицательные.
  Смешивать их в сравнениях опасно — частый баг. *(03)*

## Числа с плавающей точкой

- **IEEE 754** — стандарт хранения дробных чисел (`float`, `double`): знак + порядок
  (экспонента) + мантисса. *(04)*
- **Мантисса и экспонента** — «значащие цифры» и «куда сдвинуть запятую». Так одно число
  покрывает и очень большие, и очень маленькие величины. *(04)*
- **Ошибка округления** — не каждое десятичное число точно представимо в двоичном виде,
  поэтому `0.1 + 0.2 != 0.3`. Сравнивать дробные «на равно» нельзя — только с допуском. *(04)*

## Память, адреса, указатели

- **Адрес** — «номер ячейки» в памяти. Память — это огромный пронумерованный массив байтов. *(05)*
- **Указатель (`T*`)** — переменная, хранящая адрес. Разыменование `*p` — «значение по
  адресу». В C может быть `NULL` (указатель «в никуда»). *(05)*
- **Стек (stack)** — память под локальные переменные функции; выделяется/освобождается
  автоматически при входе/выходе. Быстро, но время жизни ограничено функцией. *(05)*
- **Куча (heap)** — память «по запросу» (`malloc`), живёт пока не освободишь (`free`). Гибко,
  но ответственность на тебе. *(05, 09)*
- **Выравнивание (alignment)** — требование, чтобы данные лежали по «круглым» адресам
  (например, `int` — по адресу, кратному 4). Процессору так быстрее; нарушение — медленно
  или вовсе ошибка. *(05)*
- **Сегментация (segfault)** — крах при обращении к чужому/невалидному адресу (например,
  разыменование `NULL` или висячего указателя). *(05, 06)*

## Массивы, строки, безопасность

- **Си-строка** — массив `char`, заканчивающийся нулевым байтом `'\0'`. Длину строка «не
  знает» — её ищут до нуля. *(06)*
- **Переполнение буфера (buffer overflow)** — запись за границу массива; портит соседнюю
  память. Историческая уязвимость №1 (через неё взламывают программы). *(06)*
- **Висячий указатель (dangling)** — указывает на уже освобождённую/исчезнувшую память.
  Обращение к нему — UB. *(06, 09)*
- **UB (Undefined Behavior)** — неопределённое поведение: выход за границы, разыменование
  `NULL`, гонка данных. Может «работать», а может всё сломать. Избегаем всегда. *(06, 13)*

## Машина и архитектура

- **Процессор (CPU)** — исполняет машинные инструкции одну за другой. *(07)*
- **Регистр** — крошечная сверхбыстрая ячейка внутри процессора (`rax`, `rbx`… в x86-64).
  Процессор считает в регистрах, а не прямо в памяти. *(07)*
- **Ассемблер (assembly)** — человекочитаемая запись машинных инструкций (`mov`, `add`,
  `jmp`). Один шаг над «голыми» битами кода. *(07)*
- **Инструкция (опкод)** — одна команда процессору (сложить, сравнить, перейти). *(07, капстоун)*
- **x86-64** — самая распространённая архитектура десктопов/серверов; её ассемблер мы учимся
  *читать* (не писать). *(07)*

## Иерархия памяти и кэш

- **Иерархия памяти** — уровни хранения от быстрых-маленьких к медленным-большим: регистры →
  кэш L1/L2/L3 → оперативная память (RAM) → диск. *(08)*
- **Кэш (cache)** — быстрая память между CPU и RAM; хранит недавно использованные данные,
  чтобы не ходить в медленную RAM. *(08)*
- **Кэш-линия (cache line)** — блок (обычно 64 байта), которым кэш оперирует целиком: читаешь
  один байт — подтянулись и соседние. Поэтому соседние данные «дешёвые». *(08)*
- **Локальность (locality)** — данные, к которым обращаются рядом во времени или в адресах,
  лежат в кэше вместе. Хорошая локальность = быстрый код. *(08)*
- **Промах кэша (cache miss)** — нужных данных в кэше не оказалось, идём в медленную RAM. *(08)*

## Динамическая память и аллокаторы

- **`malloc` / `free`** — выделить / освободить память на куче (C-аналог `new`/`delete`). *(09)*
- **Аллокатор** — код, управляющий кучей: ведёт учёт свободных кусков, выдаёт по запросу. *(09)*
- **Арена (arena/pool)** — аллокатор, который выдаёт память из заранее взятого большого блока
  и освобождает всё разом. Быстро для однотипных мелких объектов. *(09)*
- **Фрагментация** — память формально есть, но «дырками» — нет непрерывного куска нужного
  размера. *(09)*
- **Утечка памяти (leak)** — выделил и не освободил; память «теряется» до конца работы
  программы. *(09)*

## Процессы, потоки, планирование

- **Процесс** — запущенная программа со своей **отдельной** памятью; ОС их изолирует друг от
  друга. *(10, 12)*
- **Поток (thread)** — независимая нить исполнения *внутри* процесса; потоки одного процесса
  делят общую память (отсюда и удобство, и опасность). *(10, 13)*
- **Контекст и переключение (context switch)** — ОС сохраняет состояние одной задачи и
  загружает другую, создавая иллюзию одновременности на одном ядре. *(10, 11)*
- **Планировщик (scheduler)** — часть ОС, решающая, какой задаче сейчас дать процессор. *(11)*
- **FCFS (First-Come-First-Served)** — «кто первый пришёл — тот и считает до конца». Простой,
  но долгие задачи задерживают всех. *(11)*
- **Round-Robin (RR)** — каждой задаче дают по кусочку времени (квант) по кругу. Справедливо
  и отзывчиво. *(11)*
- **Квант времени (time slice)** — отрезок процессорного времени, который планировщик выдаёт
  задаче за раз. *(11)*

## Виртуальная память

- **Виртуальная память** — каждой программе ОС показывает «своё» непрерывное адресное
  пространство, будто вся память её. Реальное расположение прячет железо (MMU). *(12)*
- **Страница (page)** — блок памяти фиксированного размера (обычно 4 КБ), которым ОС управляет
  при отображении виртуального на физическое. *(12)*
- **Таблица страниц (page table)** — карта «виртуальная страница → физический кадр». *(12)*
- **TLB (Translation Lookaside Buffer)** — кэш недавних переводов адресов, чтобы не лезть в
  таблицу страниц каждый раз. *(12)*
- **Страничный промах (page fault)** — обращение к странице, которой сейчас нет в RAM; ОС её
  подгружает (например, с диска). *(12)*

## Конкурентность и синхронизация

- **Конкурентность (concurrency)** — несколько задач продвигаются «как бы одновременно»;
  требует аккуратности с общими данными. *(13)*
- **Гонка данных (data race)** — два потока меняют одни данные без синхронизации; результат
  непредсказуем. Это UB. *(13)*
- **Критическая секция** — участок кода, который одновременно должен выполнять только один
  поток. *(13)*
- **Мьютекс (mutex)** — «замок»: захватил — вошёл в критическую секцию, отпустил — впустил
  следующего. *(13)*
- **Условная переменная (condition variable)** — даёт потоку «ждать» события и быть
  разбуженным другим потоком (без бессмысленного опроса в цикле). *(13)*
- **Семафор (semaphore)** — счётчик доступных ресурсов: пускает не больше N потоков сразу. *(13)*
- **Атомарная операция (atomic)** — неделимая операция над переменной без мьютекса (для
  простых счётчиков). *(13)*
- **Deadlock (взаимная блокировка)** — потоки вечно ждут друг друга (A держит замок, нужный
  B, и наоборот). Никто не двигается. *(13)*

## Инструменты, ввод-вывод, отладка

- **Системный вызов (syscall)** — запрос программы к ядру ОС сделать «привилегированное»
  (открыть файл, выделить память, создать процесс). Граница между программой и ОС. *(14)*
- **Файловый дескриптор (fd)** — целочисленный «номер-ручка» на открытый файл/поток. 0 — ввод
  (stdin), 1 — вывод (stdout), 2 — ошибки (stderr). *(14)*
- **gdb / lldb** — отладчики: пошагово исполнять программу, смотреть переменные, ловить место
  краха. *(14)*
- **valgrind** — инструмент поиска ошибок памяти (Linux). На macOS его роль выполняет ASan. *(14)*
- **Санитайзер (ASan/UBSan)** — детектор ошибок в рантайме, встроенный в компилятор. ASan
  ловит выход за границы / use-after-free / утечки; флаг `-fsanitize=address,undefined`. *(14)*
- **strace** — показывает поток системных вызовов программы; помогает понять, что она реально
  делает с ОС. *(14)*

## Капстоун

- **Виртуальная машина (VM)** — программа, которая исполняет **байткод** так, как процессор
  исполняет машинный код: читает инструкции и выполняет их. *(капстоун)*
- **Байткод** — поток простых инструкций (опкодов) для VM — «машинный код для VM». *(капстоун)*
- **Стековая машина** — VM, где операнды берутся со **стека** и результат кладётся обратно
  (`PUSH 2; PUSH 3; ADD` → на стеке 5). *(капстоун)*
- **Цикл fetch-decode-execute** — «выбери инструкцию → распознай → выполни» — сердце любого
  процессора и нашей VM. *(капстоун)*
