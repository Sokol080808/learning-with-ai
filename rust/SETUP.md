# Установка и настройка окружения

Чтобы пройти курс, тебе нужны **две вещи**: компилятор Rust (`rustc`) и менеджер сборки
**cargo**. Они ставятся вместе одной установкой. Этот файл — пошагово, что поставить и
как проверить, что всё работает.

> Если хоть на одном шаге что-то пошло не так — не гадай, покажи мне вывод команды
> целиком. Проблемы установки решаются за пару минут, если видно сообщение.

---

## Шаг 1. Установить Rust через `rustup` (рекомендуемый способ)

**rustup** — официальный установщик и менеджер версий Rust. Он ставит сразу `rustc`,
`cargo` и стандартную библиотеку, а потом позволяет обновлять их одной командой и
добавлять компоненты (clippy, rustfmt). Это рекомендуемый способ на **всех** ОС.

Открой <https://rustup.rs> и следуй инструкции, либо выполни в терминале:

**macOS / Linux:**
```bash
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
```
Согласись с вариантом по умолчанию (`1) Proceed with standard installation`). После
установки перезапусти терминал или выполни:
```bash
source "$HOME/.cargo/env"
```

**Windows:**
скачай и запусти `rustup-init.exe` со страницы <https://rustup.rs>. Если предложит
поставить «Microsoft C++ Build Tools» — согласись: они нужны как линковщик.

### Почему именно rustup, а не пакетный менеджер?

На macOS можно поставить и через Homebrew (`brew install rust`), и это сработает. Но
**rustup гибче**:
- умеет ставить несколько версий (stable / beta / nightly) и переключаться между ними;
- одной командой обновляет всё: `rustup update`;
- управляет компонентами `clippy` и `rustfmt`, которые нам понадобятся в модуле 14.

С Homebrew всё это придётся делать руками, поэтому для курса предпочитай rustup. Если ты
уже поставил Rust через brew и не хочешь переустанавливать — тоже нормально, просто
учти, что компоненты добавляются иначе (через `rustup` их не будет; ставь нужные
инструменты отдельно или перейди на rustup).

## Шаг 2. Проверить, что всё установилось

Выполни обе команды — обе должны напечатать версию (числа могут отличаться):

```bash
rustc --version
# rustc 1.XX.0 (……)

cargo --version
# cargo 1.XX.0 (……)
```

Если терминал пишет `command not found` — почти всегда это значит, что не подхватился
`PATH`. Перезапусти терминал; на macOS/Linux помогает `source "$HOME/.cargo/env"`. На
Windows — открой новое окно терминала после установки.

## Шаг 3. Про зависимости — cargo всё качает сам

Почти все модули курса работают **только на стандартной библиотеке (`std`)** — внешних
зависимостей нет. Единственное исключение: модуль **16 (async)** использует крейт-раннер
`futures` (cargo скачает его при первой сборке этого модуля). Но полезно знать заранее: когда у крейта в `Cargo.toml` появляется
секция `[dependencies]`, **cargo сам** скачивает указанные пакеты (крейты) с
<https://crates.io> при первой сборке и кеширует их. Тебе не нужно ничего качать руками —
достаточно `cargo build` или `cargo test`. Поэтому при самой первой сборке нужен
интернет (cargo обновит индекс пакетов); дальше всё работает офлайн.

## Шаг 4. Поставить компоненты качества: clippy и rustfmt

Их мы используем в модуле 14, но поставь сейчас — это пара секунд. Через rustup:

```bash
rustup component add clippy
rustup component add rustfmt
```

Проверка:
```bash
cargo clippy --version    # линтер: ловит «пахнущий» код и подсказывает идиомы
cargo fmt --version       # автоформаттер: единый стиль одной командой
```

- **rustfmt** (`cargo fmt`) форматирует код по официальному стилю — больше не споришь
  про отступы.
- **clippy** (`cargo clippy`) — линтер: помимо ошибок компилятора подсказывает, как
  написать идиоматичнее (часто прямо с предложением исправления).

Если ставил Rust через Homebrew, а не rustup, команда `rustup component add` будет
недоступна — в этом случае проще перейти на rustup.

## Шаг 5. Редактор: VS Code + rust-analyzer

Можно писать в любом редакторе, но сильно рекомендую настроить **rust-analyzer** — это
языковой сервер, который даёт подсветку ошибок прямо в редакторе (часто ещё до запуска
тестов), автодополнение, переход к определению и показ выведенных типов. В Rust это
особенно ценно: ошибки владения видно сразу, по ходу набора.

- **VS Code:** установи расширение **rust-analyzer** (издатель — The Rust Programming
  Language). НЕ ставь старое расширение «Rust» (rls) — оно устарело.
- **JetBrains:** подойдёт RustRover или плагин Rust для IntelliJ IDEA.
- **Neovim / Helix / Emacs:** все умеют подключать rust-analyzer как LSP-сервер.

Открой проект так, чтобы корнем была папка `rust/` (где лежит `Cargo.toml` воркспейса) —
тогда rust-analyzer увидит все модули-крейты.

## Команды на каждый день

```bash
rustc --version          # версия компилятора
cargo --version          # версия cargo
rustup update            # обновить Rust до свежего stable
./rust/run.sh            # собрать и прогнать все тесты курса
./rust/run.sh 03         # тесты только модуля 03
./rust/run.sh clean      # cargo clean (удалить target/)
```

## Заметки по операционным системам

- **macOS.** Понадобятся инструменты командной строки Xcode (для линковщика). Если их
  нет, при первой сборке система сама предложит их поставить, либо выполни
  `xcode-select --install`. rustup и cargo работают на Intel и Apple Silicon одинаково.
- **Linux.** Нужны базовые средства сборки: на Debian/Ubuntu это
  `sudo apt install build-essential` (даёт линковщик `cc`). Без них cargo соберёт код,
  но не слинкует, и ты увидишь ошибку про `linker 'cc' not found`.
- **Windows.** rustup по умолчанию использует toolchain `msvc`, которому нужны
  «Microsoft C++ Build Tools» (rustup-init предложит их поставить). Альтернатива —
  toolchain `gnu`. Запускать `./rust/run.sh` удобнее всего из **Git Bash** или **WSL**,
  потому что это bash-скрипт.
