# Глоссарий курса Rust

Короткие определения терминов, которые встречаются в модулях. Это не замена теории из
README — это **шпаргалка для быстрого вспоминания**. Рядом с термином указан модуль, где
он разбирается подробно. Если определение непонятно — открой модуль или спроси меня.

> Совет: не зубри. Возвращайся сюда, когда встретил слово и забыл, что оно значит.

## Сборка и инструменты

- **Крейт (crate)** — единица компиляции в Rust: библиотека или бинарник. Каждый модуль
  курса — отдельный крейт. *(00)*
- **Воркспейс (workspace)** — набор крейтов с общим `Cargo.toml` и общим `target/`. Весь
  курс — один воркспейс. *(00)*
- **cargo** — менеджер сборки: собирает (`cargo build`), запускает тесты (`cargo test`),
  качает зависимости с crates.io, форматирует и линтует. *(00)*
- **`Cargo.toml`** — манифест крейта: имя, редакция (`edition = "2021"`), зависимости.
  `Cargo.lock` фиксирует точные версии. *(00)*
- **`todo!()`** — макрос-заглушка: компилируется (тип «любой»), но паникует в рантайме.
  Им помечены тела в стабах, поэтому тесты красные, пока не напишешь логику. *(00)*
- **Тест (`#[test]`)** — функция-проверка; падает, если `assert_eq!`/`assert!` не
  выполнились или код запаниковал. Запуск — `cargo test`. *(00)*
- **clippy** — линтер: подсказывает идиоматичный код помимо ошибок компилятора. *(14)*
- **rustfmt (`cargo fmt`)** — автоформаттер по официальному стилю. *(14)*
- **panic** — аварийное завершение потока (`panic!`, `unwrap()` на ошибке, выход за
  границы среза). Не то же, что возвращаемая ошибка `Result`. *(02, 07)*

## Переменные и типы

- **Иммутабельность по умолчанию** — `let x = 5;` нельзя менять; для изменяемой
  переменной нужно `let mut x = 5;`. *(01)*
- **Вывод типов** — компилятор сам определяет тип по значению; аннотация `: T`
  обязательна лишь там, где он неоднозначен. *(01)*
- **Скалярные типы** — `i32`/`u64`/…, `f64`, `bool`, `char`. **Составные** — кортеж
  `(T, U)` и массив `[T; N]`. *(01)*
- **Выражение vs инструкция** — почти всё в Rust — выражение со значением; блок `{ … }`
  возвращает последнее выражение (без `;`). `if`/`match`/`loop` тоже дают значение. *(01, 02)*
- **Затенение (shadowing)** — повторный `let x = …;` создаёт новую переменную с тем же
  именем (можно сменить тип). Это не мутация. *(01)*

## Поток управления

- **`match`** — исчерпывающий разбор по образцам; компилятор требует покрыть **все**
  варианты (или поставить `_`). Это выражение — возвращает значение. *(02, 06)*
- **`loop` / `while` / `for`** — циклы; `loop` может вернуть значение через `break expr`.
  `for x in iter` идёт по итератору. *(02)*
- **Образец (pattern)** — шаблон для разбора значения: литерал, `Some(v)`, `(a, b)`,
  диапазон `1..=5`, `_`. Используется в `match`, `if let`, `let`. *(02, 06)*

## Владение и заимствование — сердце Rust

- **Владение (ownership)** — у каждого значения ровно один владелец; когда владелец
  выходит из области видимости, значение освобождается. Без сборщика мусора. *(03)*
- **Перемещение (move)** — при присваивании/передаче владение **переходит**; исходная
  переменная становится недействительной (`value used after move` — ошибка). *(03)*
- **`Copy`** — типы, которые при присваивании **копируются**, а не перемещаются (числа,
  `bool`, `char`, кортежи из них). Поэтому `let b = a;` оставляет `a` валидным. *(03)*
- **`Clone`** — явное создание глубокой копии: `a.clone()`. В отличие от move — оба
  значения остаются. *(03, 08)*
- **Заимствование (borrowing)** — взять **ссылку** на значение, не забирая владение. По
  ссылке смотрят/меняют оригинал, освобождать не нужно. *(04)*
- **`&T` (общая ссылка)** — только чтение; таких ссылок одновременно может быть много. *(04)*
- **`&mut T` (изменяемая ссылка)** — чтение и запись; одновременно может быть **только
  одна**, и при ней нет других ссылок. *(04)*
- **Правило заимствования** — в один момент: либо сколько угодно `&T`, **либо** ровно
  один `&mut T`. Именно это статически исключает гонки данных. *(04)*
- **Borrow checker** — часть компилятора, проверяющая правила владения и заимствования.
  Его «ругань» — это пойманный баг с памятью, а не каприз. *(03, 04)*
- **Время жизни (lifetime, `'a`)** — насколько долго ссылка остаётся валидной.
  Компилятор следит, чтобы ссылка не пережила то, на что указывает; иногда просит
  аннотацию `'a`. *(12)*
- **Висячая ссылка (dangling)** — ссылка на уже освобождённое значение. В Rust её
  **невозможно** создать: компилятор не даст скомпилировать. *(04, 12)*

## Структуры, перечисления, образцы

- **`struct`** — собственный тип-«запись» из именованных полей. *(05)*
- **`impl`** — блок методов и ассоциированных функций для типа. **Метод** принимает
  `&self`/`&mut self`/`self`; **ассоциированная функция** (например `new`) — нет. *(05)*
- **`enum`** — тип-«одно из вариантов»; варианты могут нести данные
  (`enum Shape { Circle(f64), Rect(f64, f64) }`). *(06)*
- **`Option<T>`** — `Some(T)` или `None`; безопасная замена `null`. Достать значение —
  через `match`, `if let`, `unwrap_or`, `map`. *(06)*
- **`if let` / `while let`** — короткий разбор одного образца без полного `match`. *(06)*

## Обработка ошибок

- **`Result<T, E>`** — `Ok(T)` или `Err(E)`; ошибки в Rust — это **значения**, а не
  исключения. Функция честно объявляет, что может не получиться. *(07)*
- **Оператор `?`** — в функции, возвращающей `Result`/`Option`: на `Ok(v)` достаёт `v`,
  на `Err(e)` — немедленно возвращает `Err(e)` наружу. Убирает лестницу `match`. *(07)*
- **`unwrap` / `expect`** — достать значение, паникуя при `Err`/`None`. Удобно в тестах
  и прототипах, опасно в проде. *(07)*
- **`panic!` vs `Result`** — паника для «так быть не должно» (баг); `Result` для
  ожидаемых, восстановимых ошибок (нет файла, неверный ввод). *(07)*

## Коллекции и строки

- **`Vec<T>`** — растущий массив, владеет своими элементами; индексация `v[i]`, добавление
  `push`. *(08)*
- **`String` vs `&str`** — `String` владеет строкой в куче и растёт; `&str` — это
  заимствованный **срез** строки (только чтение). Строковый литерал имеет тип `&str`. *(08, 12)*
- **`HashMap<K, V>`** — ассоциативный массив (ключ → значение), доступ в среднем `O(1)`.
  Основа капстоуна. *(08)*
- **Срез (slice, `&[T]` / `&str`)** — заимствованное «окно» в часть массива/вектора/строки
  без копирования. *(12)*

## Обобщения, трейты, диспетчеризация

- **Обобщение (generic, `<T>`)** — код, параметризованный типом; компилятор генерирует
  конкретные версии (мономорфизация), накладных расходов в рантайме нет. *(09)*
- **Трейт (trait)** — набор методов-«интерфейс», который тип может реализовать
  (`impl Trait for Type`). *(09)*
- **Ограничение трейтом (`T: Trait`)** — «годится любой тип, реализующий `Trait`».
  Основа обобщённого кода. *(09)*
- **Статическая диспетчеризация** — вызов метода известен на этапе компиляции (через
  обобщения); быстро, без косвенности. *(09)*
- **Трейт-объект (`dyn Trait`)** — значение, тип которого известен только в рантайме;
  обычно за указателем: `Box<dyn Trait>`, `&dyn Trait`. *(10)*
- **Динамическая диспетчеризация** — метод выбирается в рантайме по таблице (vtable).
  Гибко (разные типы в одном `Vec`), но с небольшой ценой косвенного вызова. *(10)*

## Замыкания и итераторы

- **Замыкание (closure)** — анонимная функция `|x| x + 1`, которая может **захватывать**
  переменные окружения. *(11)*
- **`Fn` / `FnMut` / `FnOnce`** — трейты замыканий по способу захвата: только читает /
  изменяет / потребляет захваченное. *(11)*
- **Итератор (`Iterator`)** — «ленивый поток» значений; `next()` даёт следующий.
  Комбинаторы `map`/`filter`/`take` тоже ленивые — ничего не делают, пока не «потребишь». *(11)*
- **`collect()`** — материализовать итератор в коллекцию (`Vec`, `String`, `HashMap`…).
  Завершает ленивую цепочку. *(11)*

## Умные указатели

- **`Box<T>`** — единоличное владение значением в **куче**. Нужен для рекурсивных типов
  и трейт-объектов. *(13)*
- **`Rc<T>`** — счётчик ссылок: **разделяемое** владение в одном потоке; значение живёт,
  пока счётчик > 0. Только для чтения (без `RefCell`). *(13)*
- **`RefCell<T>`** — внутренняя изменяемость: позволяет менять данные через общую ссылку,
  проверяя правила заимствования **в рантайме** (паника при нарушении). *(13)*
- **`Rc<RefCell<T>>`** — частый дуэт: несколько владельцев, которым нужно ещё и менять
  значение. *(13)*
