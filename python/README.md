# Курс Python: от основ до уровня Junior

Это не сборник задачек «напиши код ради кода». Это **учебный курс**, цель которого —
чтобы ты *понимал*, как и почему работает Python, и к концу мог уверенно претендовать
на позицию **junior Python разработчика**.

Версия языка: **Python 3.11+** (лучше 3.12/3.13). Проверка заданий: **pytest**.

---

## Как устроен курс

Курс разбит на **модули** (`modules/00-setup` … `modules/14-tooling`) и финальный
**капстоун** (`capstone/` — CLI-менеджер задач). Каждый модуль — это одна большая идея.

Внутри модуля:

```
modules/NN-slug/
  README.md            ← теория простым языком + ПОЧЕМУ так + описание заданий + подсказки
  <slug>.py            ← СТАБ с raise NotImplementedError  ← ТЫ ПИШЕШЬ ЗДЕСЬ
  test_<slug>.py       ← готовые тесты    ← их трогать не нужно, это критерий «сдано»
```

Логика проста: тест знает, как должна вести себя твоя функция/класс. Ты дописываешь
стаб → запускаешь тесты → видишь, что прошло, а что нет → исправляешь. Красное
становится зелёным — но с пониманием, а не наугад.

В трёх модулях есть **микро-проект** — маленькое цельное приложение, в котором идеи
модуля собираются вместе:

- **02** — CLI-калькулятор;
- **04** — частотный словарь слов;
- **12** — анализатор логов.

А в самом конце — **капстоун**: CLI-менеджер задач (todo) с хранением в JSON, где
сходится почти всё, что ты прошёл.

## Что нужно установить

Почти всё ставит сам скрипт `run.sh` при первом запуске. От тебя нужен только сам
**Python 3.11+** и (желательно) **git**. Проверь:

```bash
python3 --version
```

Если версия 3.11 или выше — ты готов. `.venv` и `pytest` скрипт создаст сам (один раз
понадобится интернет). Подробности по установке на macOS / Linux / Windows и про
редактор — в [SETUP.md](SETUP.md).

---

## Рабочий цикл (запомни его — он один на весь курс)

```bash
# прогнать ВСЕ тесты всех модулей
./python/run.sh

# прогнать тесты только одного модуля (например, 01)
./python/run.sh 01

# удалить виртуальное окружение и кэши, начать с чистого листа
./python/run.sh clean
```

В самом начале почти все тесты **красные** — это нормально, это и есть твои задания.
Твоя работа — делать их зелёными, по одному, понимая, что ты пишешь.

> Совет: запускай тесты часто. Маленький шаг → тест → следующий шаг. Так ты сразу
> видишь, где ошибся, и не копишь баги. Падающий тест с понятным сообщением —
> это бесплатный урок, а не повод расстраиваться.

Первый запуск дольше остальных: скрипт создаёт `.venv` и ставит туда `pytest`.
Дальше запускается мгновенно.

## Как со мной работать (режим «преподаватель»)

Я — твой преподаватель, а не решебник. Поэтому:

- **Я не выдаю готовых решений заданий по умолчанию.** Если попросишь «дай ответ» —
  сначала задам наводящие вопросы, помогу самому дойти до решения. Так знание
  остаётся в голове, а не в буфере обмена.
- **Спрашивай свободно.** «Почему здесь кортеж, а не список?», «Я думаю сделать так —
  это нормально?», «Не понимаю, почему тест падает» — это правильные вопросы.
- **Показывай свой код.** Я отвечу, в верном ли направлении ты идёшь, на что обратить
  внимание, какие есть подводные камни.
- Если реально застрял — попроси **подсказку** (hint). В README каждого модуля
  подсказки устроены ступенчато: сначала лёгкий намёк, потом сильнее, и только в самом
  конце — почти разбор. Открывай по одной.
- Полное решение я дам только если ты явно об этом попросишь, и всё равно с разбором
  «почему именно так».

Не бойся ошибаться — интерпретатор и тесты для того и нужны.

## Программа курса

**Часть I. Основы**
- **00 — Setup и как запускается Python.** Интерпретатор, байт-код, `python3`, REPL,
  виртуальное окружение, цикл test → fix. Разминочное задание.
- **01 — Типы, переменные, числа, строки.** Динамическая типизация, `int`/`float`/`bool`,
  f-строки, неизменяемость строк.
- **02 — Управление потоком и функции.** `if`/`for`/`while`, аргументы по умолчанию,
  `*args`/`**kwargs`, возврат значений. *Микро-проект: CLI-калькулятор.*

**Часть II. Данные и структуры**
- **03 — Коллекции: list, tuple, dict, set.** Когда что; изменяемость; comprehension'ы.
- **04 — Строки и обработка текста.** Методы строк, срезы, `split`/`join`, нормализация.
  *Микро-проект: частотный словарь.*

**Часть III. Функции и ООП**
- **05 — Функции глубже: замыкания, декораторы.** Функции как объекты, `*`, область
  видимости, `functools.wraps`.
- **06 — ООП: классы, dunder-методы, свойства.** `__init__`, `__repr__`, `@property`,
  инкапсуляция.
- **07 — Наследование, dataclasses, утиная типизация.** `super()`, `@dataclass`,
  протоколы и «если крякает как утка».
- **08 — Ошибки, исключения и контекст-менеджеры.** `try/except/finally`, свои
  исключения, `with`, `contextmanager`.

**Часть IV. Инженерия**
- **09 — Итераторы и генераторы.** Протокол итератора, `yield`, ленивость, генераторные
  выражения.
- **10 — Модули, пакеты и импорты.** `import`, `__name__`, `__init__.py`, абсолютные и
  относительные импорты.
- **11 — Полезная стандартная библиотека.** `collections`, `itertools`, `pathlib`,
  `datetime`, `re` и друзья.
- **12 — Файлы и данные (текст, JSON, CSV).** Чтение/запись, кодировки, `json`, `csv`.
  *Микро-проект: анализатор логов.*
- **13 — Аннотации типов и тестирование.** Type hints, `mypy` обзорно, как писать
  тесты на pytest самому.
- **14 — Инструменты и качество кода.** Форматтеры и линтеры (`black`, `ruff`),
  структура проекта, gitignore, что значит «чистый код».

**Капстоун (`capstone/`)** — CLI-менеджер задач (todo) с хранением в JSON: большой
pet-проект, где сходится всё пройденное.

Отмечай прогресс в [PROGRESS.md](PROGRESS.md).
