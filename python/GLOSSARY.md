# Глоссарий курса

Короткие определения терминов, которые встречаются в модулях. Это не замена теории из
README — это **шпаргалка для быстрого вспоминания**. Рядом с термином указан модуль, где
он разбирается подробно. Если определение непонятно — открой модуль или спроси меня.

> Совет: не зубри это. Возвращайся сюда, когда встретил слово и забыл, что оно значит.

## Запуск и инструменты

- **Интерпретатор** — программа (`python3` / CPython), которая читает твой `.py`, компилирует
  его в **байт-код** и исполняет. В отличие от C++, отдельного шага «собрать exe» нет. *(00)*
- **Байт-код** — промежуточное представление (`.pyc` в папке `__pycache__`), которое исполняет
  виртуальная машина Python. Кэшируется, чтобы не перекомпилировать каждый раз. *(00)*
- **REPL** — интерактивный режим (`python3` без файла): вводишь выражение — сразу видишь
  результат. Удобно проверять идеи. *(00)*
- **venv (виртуальное окружение)** — изолированная «песочница» с Python и пакетами под один
  проект, чтобы зависимости не конфликтовали глобально. Создаётся `python3 -m venv .venv`. *(00, 14)*
- **pip** — менеджer пакетов Python: `pip install <имя>` ставит пакет (обычно с PyPI) в
  текущее окружение. *(00, 14)*
- **PEP 8** — официальный стиль кода Python (отступы, имена, длина строк). Линтеры/форматтеры
  опираются на него. *(14)*
- **black / ruff** — инструменты качества: `black` форматирует код единообразно, `ruff` — быстрый
  линтер, ловит ошибки и нарушения стиля. *(14)*

## Типы и значения

- **Динамическая типизация** — тип хранит значение, а не имя переменной. Имя — лишь ярлык,
  его можно «перевесить» на объект другого типа. *(01)*
- **Изменяемый / неизменяемый (mutable / immutable)** — можно ли поменять объект на месте.
  `int`, `float`, `str`, `tuple`, `bool` — неизменяемые; `list`, `dict`, `set` — изменяемые. *(01, 03)*
- **`int` / `float` / `bool`** — целое (произвольной длины), число с плавающей точкой, логическое.
  `bool` — подтип `int` (`True == 1`). *(01)*
- **f-строка** — строковый литерал с подстановкой: `f"x={x}"`. Удобный способ собирать текст. *(01, 04)*
- **Срез (slice)** — `s[a:b:step]` — подпоследовательность строки/списка/кортежа. Не включает `b`. *(01, 03, 04)*
- **`None`** — единственное значение типа `NoneType`, «ничего/нет результата». Сравнивают через
  `is None`. *(01, 02)*
- **Истинность (truthiness)** — в `if` пустые коллекции, `0`, `""`, `None` считаются ложными,
  всё остальное — истинным. *(02)*

## Коллекции

- **`list`** — изменяемый упорядоченный список: `[1, 2, 3]`. Индексация, срезы, `append`. *(03)*
- **`tuple`** — неизменяемый упорядоченный кортеж: `(1, 2)`. Годится как ключ словаря. *(03)*
- **`dict`** — словарь «ключ → значение»: `{"a": 1}`. Ключи уникальны и хешируемы. Сохраняет
  порядок вставки. *(03)*
- **`set`** — множество уникальных элементов: `{1, 2}`. Быстрая проверка `in`, операции
  объединения/пересечения. *(03)*
- **Хешируемый** — объект с неизменным `__hash__` (обычно неизменяемый). Только такие могут быть
  ключами `dict` и элементами `set`. *(03)*
- **Comprehension** — компактное построение коллекции: `[x*x for x in nums if x > 0]`. Бывает
  для list, dict, set. *(03)*

## Строки и текст

- **`str.split` / `str.join`** — разбить строку на части по разделителю / собрать список строк
  обратно в строку. *(04)*
- **Нормализация** — приведение текста к единому виду перед обработкой (`.lower()`, `.strip()`),
  чтобы «Привет» и «привет » считались одинаковыми. *(04)*
- **`re` (регулярные выражения)** — язык шаблонов для поиска/замены в тексте: `re.findall(pat, s)`. *(04, 11, 15)*

## Функции

- **Аргумент по умолчанию** — значение параметра, если его не передали: `def f(x, y=0)`. Не
  делай изменяемым (`[]`, `{}`) — классическая ловушка. *(02)*
- **`*args` / `**kwargs`** — приём произвольного числа позиционных / именованных аргументов. *(02)*
- **Область видимости (scope)** — где видно имя: локальная функции, объемлющая, глобальная,
  встроенная (правило LEGB). *(05)*
- **Замыкание (closure)** — функция, «запомнившая» переменные из объемлющей функции после её
  завершения. *(05)*
- **Декоратор** — функция, оборачивающая другую функцию, чтобы добавить поведение, не меняя её
  код: `@my_decorator`. Внутри обычно `functools.wraps`. *(05)*
- **`functools.wraps`** — переносит имя/докстринг исходной функции на обёртку, чтобы декоратор
  не «стирал» её идентичность. *(05)*

## ООП

- **Класс / экземпляр** — `class` описывает тип, экземпляр — конкретный объект этого типа. *(06)*
- **`__init__`** — конструктор: настраивает поля нового экземпляра. Первый параметр — `self`. *(06)*
- **`self`** — ссылка на текущий экземпляр внутри метода; передаётся неявно при вызове. *(06)*
- **dunder-метод** — «double underscore», метод вида `__repr__`, `__eq__`, `__len__`, который
  Python вызывает за тебя в нужный момент (печать, сравнение, `len`). *(06)*
- **`@property`** — превращает метод в «вычисляемое поле»: доступ как к атрибуту, но с логикой
  под капотом. *(06)*
- **Наследование** — класс берёт поведение родителя и расширяет его; `super()` зовёт версию
  родителя. *(07)*
- **`@dataclass`** — декоратор, который сам генерирует `__init__`, `__repr__`, `__eq__` по
  объявленным полям. *(07)*
- **Утиная типизация (duck typing)** — «если крякает как утка — это утка»: важно не имя класса,
  а наличие нужных методов. *(07)*
- **Protocol** — описание набора методов, которые должен иметь объект, для статической проверки
  утиной типизации. *(07, 13)*

## Ошибки и ресурсы

- **Исключение (exception)** — объект-ошибка, прерывающий обычный ход программы:
  `ValueError`, `KeyError`, `TypeError`… *(08)*
- **`try / except / else / finally`** — перехват исключений; `finally` выполняется всегда
  (для очистки). *(08)*
- **`raise`** — бросить исключение самому; своё исключение — класс-наследник `Exception`. *(08)*
- **Контекст-менеджер** — объект для `with`, который сам открывает и гарантированно закрывает
  ресурс (файл, соединение). *(08, 12)*
- **`contextlib.contextmanager`** — декоратор, превращающий генератор с одним `yield` в
  контекст-менеджер. *(08)*

## Итерация

- **Итерируемое (iterable)** — то, по чему можно пройти в `for` (список, строка, файл). *(09)*
- **Итератор (iterator)** — объект с `__next__`, выдающий элементы по одному; помнит позицию. *(09)*
- **Генератор** — функция с `yield`: возвращает значения лениво, по запросу, не строя весь
  список в памяти. *(09)*
- **Генераторное выражение** — `(x*x for x in nums)` — ленивый аналог list comprehension. *(09)*
- **Ленивость (lazy)** — значения вычисляются в момент запроса, а не заранее. Экономит память. *(09)*

## Модули, пакеты, стандартная библиотека

- **Модуль** — один `.py`-файл, который можно импортировать. *(10)*
- **Пакет** — каталог с модулями (исторически — с `__init__.py`). *(10)*
- **`import` / `from ... import ...`** — подключение модуля целиком / отдельных имён. *(10)*
- **`if __name__ == "__main__":`** — блок, который выполняется, только когда файл запущен
  напрямую, а не импортирован. *(10)*
- **Стандартная библиотека** — модули «из коробки»: `collections`, `itertools`, `pathlib`,
  `datetime`, `json`, `csv`, `re`. *(11, 12)*
- **`pathlib.Path`** — объектный путь к файлу/папке вместо строковых склеек. *(11, 12)*

## Файлы и данные

- **Кодировка (encoding)** — как байты файла соответствуют символам. По умолчанию используй
  `encoding="utf-8"`. *(12)*
- **JSON** — текстовый формат обмена данными; `json.load`/`json.dump` ↔ словари и списки Python. *(12)*
- **CSV** — таблица «значения через запятую»; читается модулем `csv`. *(12)*

## Типы и тестирование

- **Type hints (аннотации типов)** — подсказки о типах: `def f(x: int) -> str`. Не проверяются
  в рантайме, но помогают редактору и `mypy`. *(13)*
- **mypy** — статический проверщик типов: ловит несоответствия аннотаций до запуска. *(13)*
- **pytest** — фреймворк тестов курса. Тест — функция `test_*`, проверка через `assert`. *(00, 13)*
- **fixture** — заготовка-зависимость для теста (например `tmp_path` — временная папка). *(13)*
- **assert** — утверждение: если условие ложно, тест падает с понятным сообщением. *(13)*

## Регулярные выражения

- **Шаблон (pattern)** — строка с метасимволами (`\d`, `.`, `*`, `+`, `?`, `[]`), описывающая, что искать в тексте. *(15)*
- **`re.search` / `re.findall` / `re.sub`** — найти первое совпадение / все совпадения / заменить совпадения строкой или функцией. *(15)*
- **Группа (group)** — часть шаблона в скобках `(...)`, которую можно извлечь отдельно: `m.group(1)`. *(15)*
- **Именованная группа** — группа с именем: `(?P<name>...)`, достать через `m.group("name")`. Удобна, когда групп много. *(15)*
- **Жадность / ленивость (greedy / lazy)** — `*` и `+` по умолчанию захватывают как можно больше (жадно); добавь `?` (`*?`, `+?`), чтобы захватить как можно меньше. *(15)*
- **Флаги** — модификаторы поведения: `re.IGNORECASE` (`re.I`) — игнорировать регистр, `re.MULTILINE` (`re.M`) — `^`/`$` совпадают с началом/концом каждой строки. *(15)*

## Асинхронность

- **Корутина (coroutine)** — функция, объявленная через `async def`; при вызове возвращает объект-корутину, которую нужно явно запустить через `await` или `asyncio.run`. *(16)*
- **`async` / `await`** — ключевые слова для написания неблокирующего кода: `await` приостанавливает корутину до получения результата, отдавая управление другим задачам. *(16)*
- **`asyncio`** — стандартный модуль Python для асинхронного I/O: цикл событий, `asyncio.run()`, `asyncio.gather()`, примитивы синхронизации. *(16)*
- **`asyncio.gather`** — запустить несколько корутин параллельно и дождаться всех результатов: `await asyncio.gather(coro1(), coro2())`. *(16)*
- **`asyncio.Queue` / `asyncio.Semaphore`** — примитивы для координации корутин: очередь передачи данных и счётчик-ограничитель параллелизма. *(16)*
- **I/O-bound vs CPU-bound** — задача ограничена ожиданием I/O (сеть, диск) или вычислениями. `asyncio` ускоряет I/O-bound; для CPU-bound нужны процессы. *(16)*

## Метапрограммирование

- **Дескриптор (descriptor)** — объект, управляющий доступом к атрибуту другого класса через методы `__get__`, `__set__`, `__delete__`. *(17)*
- **`__set_name__`** — метод дескриптора, который Python вызывает при создании класса, передавая имя атрибута: позволяет дескриптору знать, как он называется. *(17)*
- **`__get__` / `__set__`** — протокол дескриптора: `__get__` вызывается при чтении атрибута, `__set__` — при записи. *(17)*
- **`__init_subclass__`** — хук, который Python вызывает в базовом классе каждый раз, когда от него создают подкласс; позволяет регистрировать или проверять подклассы без метакласса. *(17)*
- **`__getattr__`** — вызывается, когда обычный поиск атрибута завершился неудачей; позволяет реализовывать «динамические» атрибуты (например, проксирование). *(17)*
