# Установка: что нужно ДО начала курса

Хорошая новость: ставить почти ничего не надо. Скрипт `run.sh` сам создаёт
виртуальное окружение и ставит туда `pytest` при первом запуске. От тебя нужны только
**Python** и (желательно) **git** и редактор. Разберём по порядку.

---

## 1. Python 3.11+ (обязательно)

Курсу нужен **Python версии 3.11 или новее**. Лучше — **3.12 или 3.13**: новее синтаксис
сообщений об ошибках, быстрее, и всё, что мы используем, точно поддерживается.

Проверь, что уже стоит:

```bash
python3 --version
```

Ожидаемый вывод — что-то вроде `Python 3.12.4`. Если число `3.11` или больше — отлично,
ты готов, можно пропускать установку и идти к шагу 2.

> Почему именно `python3`, а не `python`? На macOS и многих Linux команда `python`
> либо отсутствует, либо ведёт на старый Python 2. Привыкай писать `python3` —
> в курсе и в `run.sh` используется именно она.

### macOS

Самый простой путь — [Homebrew](https://brew.sh):

```bash
brew install python@3.12
python3 --version
```

Если Homebrew ещё нет — поставь его по инструкции с сайта, затем команду выше.
(Системный Python в macOS трогать не нужно — Homebrew ставит отдельный, свежий.)

### Linux (Debian/Ubuntu)

```bash
sudo apt update
sudo apt install python3 python3-venv python3-pip
python3 --version
```

Пакет `python3-venv` важен: без него не создастся виртуальное окружение. На Fedora/Arch
ставь аналогичные пакеты через `dnf`/`pacman`.

### Windows

1. Скачай установщик с [python.org/downloads](https://www.python.org/downloads/).
2. В установщике **обязательно** поставь галочку **«Add python.exe to PATH»**.
3. Проверь в новом окне терминала (PowerShell):

   ```powershell
   py --version
   ```

На Windows удобнее работать через **py launcher** (`py` вместо `python3`). Также
рекомендуем поставить **Git Bash** или **WSL** — тогда команды `./python/run.sh`
из курса заработают так же, как на macOS/Linux. В чистом PowerShell `.sh`-скрипты
напрямую не запускаются.

## 2. venv и pytest — ставит сам скрипт

Тебе **не нужно** вручную создавать виртуальное окружение или ставить `pytest`.
При первом запуске `./python/run.sh` сделает это сам:

- создаст папку `python/.venv` — изолированное окружение, чтобы пакеты курса не
  смешивались с системными;
- установит в неё `pytest` (нужен интернет — **один раз**).

```bash
./python/run.sh        # первый запуск дольше: ставится окружение
```

Дальше окружение уже готово, и тесты запускаются мгновенно. Если что-то сломалось —
`./python/run.sh clean` снесёт `.venv` и кэши, и следующий запуск создаст всё заново.

> Что такое виртуальное окружение и зачем оно? Это отдельная «песочница» с
> Python и пакетами под конкретный проект. Без него пакеты ставятся глобально и
> проекты начинают конфликтовать. Подробнее — в модуле 00 и в [GLOSSARY.md](GLOSSARY.md).

---

## 3. git (желательно)

git нужен, чтобы фиксировать свой прогресс и при желании выложить решения. Проверь:

```bash
git --version
```

- **macOS:** `brew install git` (или установится сам при первом вызове `git`).
- **Linux:** `sudo apt install git`.
- **Windows:** [git-scm.com](https://git-scm.com) — заодно даёт Git Bash.

Курс работает и без git, но привычка коммитить рабочие шаги — часть культуры junior'а.

---

## 4. Редактор (желательно)

Рекомендуем **VS Code** + официальное расширение **Python** (от Microsoft):

1. Поставь [VS Code](https://code.visualstudio.com/).
2. В разделе Extensions найди и установи **Python** (Microsoft) — даёт автодополнение,
   подсветку ошибок, запуск тестов кнопкой и выбор интерпретатора.
3. Открой папку проекта и выбери интерпретатор из `python/.venv` (Command Palette →
   «Python: Select Interpreter»).

Подойдёт и PyCharm, и любой редактор, в котором тебе удобно. Главное — чтобы был виден
вывод тестов.

## Быстрая проверка готовности

Выполни три команды — если все три отвечают без ошибок, можно начинать:

```bash
python3 --version     # 3.11 или новее
git --version         # любой (желательно)
./python/run.sh 00    # должно создать .venv и прогнать тесты модуля 00
```

Тесты модуля 00 сначала будут **красными** — это нормально, это твоё первое задание.
Переходи к [`modules/00-setup/README.md`](modules/00-setup/README.md).
