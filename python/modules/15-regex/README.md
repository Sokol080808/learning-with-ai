# Модуль 15 — Регулярные выражения

Регулярные выражения (или «регэкспы», `regex`) — это мини-язык для описания паттернов в
тексте. Одной строкой вроде `r"\d{4}-\d{2}-\d{2}"` ты описываешь «дата в формате
ISO». Без регэкспов пришлось бы писать десятки строк с `split`, `find`, `isdigit`.

**Большая идея модуля:** регулярное выражение — это *спецификация* того, как выглядит
нужный кусок текста. Ты говоришь системе «что ищу», а она сама разбирается «как найти».
Понять несколько базовых примитивов — и ты сможешь читать и писать большинство
реальных регэкспов.

---

## Идея 1. Raw-строки `r"..."` и зачем они нужны

Прежде чем учить синтаксис паттернов — одна техническая вещь, которая спасёт от
головной боли.

Обратный слэш `\` — служебный символ и в Python-строках, и в регэкспах. Это значит,
что они могут «воевать» друг с другом:

```python
import re

# Хочем найти буквальный "\n" (два символа) в строке, а не перенос строки.
re.search("\\n", text)    # "\\" в Python-строке → один \, затем n → \n в регэкспе
re.search(r"\n", text)    # raw-строка: \ не экранируется → \n попадает в регэксп как есть
```

**Правило:** почти всегда пиши паттерны как `r"..."`. Это избавляет от удвоения
слэшей. Например, `\d` — спецпоследовательность регэкспа («любая цифра»): в обычной
строке это надо писать `"\\d"`, в raw-строке — просто `r"\d"`. Без `r"..."` легко
случайно написать `"\b"` и получить символ Backspace вместо границы слова.

> **Подвох.** `r"\n"` — это строка из двух символов `\` и `n`. Но `"\n"` — один символ
> (перенос строки). В паттерне `r"\n"` ищет буквальный перенос строки — потому что
> регэксп сам обрабатывает `\n` как метасимвол новой строки. Это нормально: raw-строка
> просто передаёт управление регэкспу без Python-интерпретации.

## Идея 2. Основные функции модуля `re`

Python предоставляет семь главных инструментов:

```python
import re

text = "Ошибка 404: файл не найден. Ошибка 500: сервер упал."

# search — найти первое вхождение где угодно в строке; возвращает Match или None
m = re.search(r"\d+", text)
print(m.group())       # "404"

# match — проверить только НАЧАЛО строки (якорит старт автоматически!)
m = re.match(r"\d+", text)
print(m)               # None — строка начинается с "О", а не с цифры

# fullmatch — строка должна целиком соответствовать паттерну
m = re.fullmatch(r"\d+", "42")
print(m)               # <Match object>
m = re.fullmatch(r"\d+", "42abc")
print(m)               # None

# findall — вернуть ВСЕ вхождения списком строк
nums = re.findall(r"\d+", text)
print(nums)            # ["404", "500"]

# finditer — вернуть итератор Match-объектов (полезно, когда нужны позиции)
for m in re.finditer(r"\d+", text):
    print(m.group(), m.start(), m.end())

# sub — заменить все вхождения на строку или функцию
result = re.sub(r"\d+", "XXX", text)
print(result)          # "Ошибка XXX: файл не найден. Ошибка XXX: сервер упал."

# compile — скомпилировать паттерн в объект для повторного использования
pattern = re.compile(r"\d+")
pattern.findall(text)  # то же, что re.findall(r"\d+", text)
```

**Почему `re.compile`?** Если ты используешь один паттерн много раз (в цикле, в
функции, вызываемой тысячи раз) — `compile` экономит время на разборе паттерна.
В одноразовых скриптах разница незаметна.

**Частая ошибка: `re.match` якорит начало.** Многие думают, что `match` ищет
«где-нибудь в строке». Нет: он проверяет только с позиции 0. Для поиска в произвольном
месте — всегда `re.search`.

## Идея 3. Классы символов, `\d \w \s`, квантификаторы

Паттерн собирается из атомов — описаний «одного символа» — и квантификаторов — «сколько
раз».

**Атомы:**

| Паттерн | Значение |
|---------|----------|
| `a`     | буквально `a` |
| `.`     | любой символ кроме `\n` |
| `\d`    | цифра `[0-9]` |
| `\D`    | не-цифра |
| `\w`    | «слово»: `[a-zA-Z0-9_]` |
| `\W`    | не `\w` |
| `\s`    | пробельный символ (пробел, `\t`, `\n`, ...) |
| `\S`    | не пробел |
| `[abc]` | один из `a`, `b`, `c` |
| `[a-z]` | любая строчная латинская буква |
| `[^a-z]`| любой символ, *кроме* строчных латинских |

```python
re.findall(r"[A-Z][a-z]+", "Hello World!")   # ["Hello", "World"]
re.findall(r"\d{3}", "abc123def456")          # ["123", "456"]
```

**Квантификаторы** (насколько раз повторяется предыдущий атом):

| Квантификатор | Значение |
|---------------|----------|
| `*`           | 0 или более |
| `+`           | 1 или более |
| `?`           | 0 или 1 (необязательный) |
| `{n}`         | ровно n раз |
| `{m,n}`       | от m до n раз |

**Жадность vs ленивость.** По умолчанию квантификаторы *жадные*: захватывают как можно
больше символов.

```python
text = "<b>жирный</b> и <i>курсив</i>"

re.search(r"<.+>", text).group()   # "<b>жирный</b> и <i>курсив</i>" — всё, жадно!
re.search(r"<.+?>", text).group()  # "<b>" — ленивый: остановился как можно раньше
```

Добавь `?` после квантификатора (`*?`, `+?`, `{m,n}?`), чтобы сделать его ленивым.

> **Подвох: катастрофический бэктрекинг.** Паттерны вроде `(a+)+` на строке `"aaaaab"`
> заставляют движок перебирать экспоненциальное число вариантов разбора. Если регэксп
> работает секунды — проверь вложенные квантификаторы. Для промышленного кода на
> подозрительных входах используй тайм-аут (`re.timeout` не существует в stdlib — обворачивай
> вызов в `signal.alarm` или используй `regex` из PyPI с поддержкой `timeout`).

## Идея 4. Группы и именованные группы

Группы `(...)` выполняют две роли: **выделяют** часть совпадения для извлечения и
**позволяют применить квантификатор к нескольким символам**.

```python
m = re.search(r"(\d{4})-(\d{2})-(\d{2})", "сегодня 2026-06-22 праздник")

m.group(0)    # "2026-06-22"  — всё совпадение
m.group(1)    # "2026"        — первая группа
m.group(2)    # "06"
m.group(3)    # "22"
m.groups()    # ("2026", "06", "22")  — все группы кортежем
```

**Именованные группы** `(?P<name>...)` делают код читаемее и устойчивее к рефакторингу:

```python
m = re.search(
    r"(?P<year>\d{4})-(?P<month>\d{2})-(?P<day>\d{2})",
    "2026-06-22"
)
m.group("year")    # "2026"
m.groupdict()      # {"year": "2026", "month": "06", "day": "22"}
```

**`findall` с группами** возвращает не строки, а кортежи:

```python
re.findall(r"(\d+)-(\d+)", "12-34, 56-78")
# [("12", "34"), ("56", "78")]  — список кортежей, по группе на элемент
```

## Идея 5. Якоря и флаги

**Якоря** — это позиции, а не символы. Они не захватывают символ, а проверяют, *где*
находится курсор.

| Якорь | Значение |
|-------|----------|
| `^`   | начало строки (начало всего текста при `re.M` — каждой строки) |
| `$`   | конец строки |
| `\b`  | граница слова (переход между `\w` и `\W`) |
| `\B`  | не-граница слова |

```python
re.findall(r"^\d+", "42 foo\n99 bar", re.MULTILINE)   # ["42", "99"]
re.findall(r"\bcat\b", "concatenate cat scat")          # ["cat"]
```

**Флаги** модифицируют поведение движка:

| Флаг | Короткая форма | Смысл |
|------|----------------|-------|
| `re.IGNORECASE` | `re.I` | Не различать регистр |
| `re.MULTILINE`  | `re.M` | `^` и `$` — начало/конец *каждой* строки |
| `re.DOTALL`     | `re.S` | `.` совпадает в том числе с `\n` |

```python
re.findall(r"hello", "Hello HELLO hello", re.I)   # ["Hello", "HELLO", "hello"]
```

## Задания

Файл `patterns.py`. Реализуй четыре функции:

### Разминки

1. **`find_all_numbers(text: str) -> list[int]`**
   Найди все целые числа (возможно, с минусом) в тексте и верни их как `list[int]`.
   `"цена: -10, скидка: 3, остаток: 500"` → `[-10, 3, 500]`.

2. **`is_valid_identifier(name: str) -> bool`**
   Проверь, является ли строка допустимым Python-идентификатором: начинается с буквы
   (`[a-zA-Z]`) или подчёркивания, состоит только из `[a-zA-Z0-9_]`, длина ≥ 1.
   Реализуй через `re.fullmatch`.

3. **`slugify(title: str) -> str`**
   Преобразуй заголовок статьи в URL-slug:
   1. Привести к нижнему регистру.
   2. Заменить любую последовательность символов, не являющихся `[a-z0-9]`, на дефис.
   3. Убрать ведущий и конечный дефис.
   `"  Hello, World!  "` → `"hello-world"`.
   `"Python 3.12 Release"` → `"python-3-12-release"`.

### Основная задача: `parse_log_line(line: str) -> dict | None`

Каждая строка серверного лога выглядит так:

```
2026-06-22 14:05:33 ERROR   [auth] Login failed for user alice@example.com
2026-06-22 14:06:01 INFO    [db]   Query executed in 42ms
```

Формат: `YYYY-MM-DD HH:MM:SS LEVEL [module] message`

Напиши функцию, которая разбирает одну строку с помощью **именованных групп** и
возвращает словарь:

```python
{
    "date":    "2026-06-22",
    "time":    "14:05:33",
    "level":   "ERROR",
    "module":  "auth",
    "message": "Login failed for user alice@example.com",
}
```

Если строка не соответствует формату — вернуть `None`.

**Требования к реализации:**
- Один вызов `re.fullmatch` (или `re.match`) с именованными группами.
- Поле `level` должно возвращаться как написано (без нормализации).
- `message` — весь хвост строки после `[module]` и пробелов.

```bash
./python/run.sh 15
```

### Подсказки

<details><summary>Разминка 1 — find_all_numbers</summary>
Числа могут начинаться с минуса. В регэкспе это `-?` (минус необязательный), затем
`\d+` (одна или более цифр). Итого паттерн: <code>r"-?\d+"</code>. После <code>findall</code>
у тебя будет список строк — преобразуй каждую в <code>int</code>.
</details>

<details><summary>Разминка 2 — is_valid_identifier</summary>
Паттерн должен охватывать <em>всю</em> строку — используй <code>re.fullmatch</code>.
Идентификатор начинается с <code>[a-zA-Z_]</code>, затем идут нуль или более
<code>[a-zA-Z0-9_]</code>. Итого: <code>r"[a-zA-Z_][a-zA-Z0-9_]*"</code>.
Верни <code>True</code>, если <code>fullmatch</code> вернул не <code>None</code>.
</details>

<details><summary>Разминка 3 — slugify</summary>
Шаги: 1) <code>.lower()</code>; 2) <code>re.sub(r"[^a-z0-9]+", "-", ...)</code> —
всё, что не буква или цифра, заменяется дефисом (квантификатор <code>+</code> схлопывает
несколько таких символов подряд в один дефис); 3) <code>.strip("-")</code> уберёт дефисы
по краям.
</details>

<details><summary>Основная задача — структура паттерна</summary>
Разбей шаблон на именованные части:
<ul>
  <li><code>(?P&lt;date&gt;\d{4}-\d{2}-\d{2})</code></li>
  <li>пробел</li>
  <li><code>(?P&lt;time&gt;\d{2}:\d{2}:\d{2})</code></li>
  <li>пробел(ы)</li>
  <li><code>(?P&lt;level&gt;\w+)</code></li>
  <li>пробел(ы)</li>
  <li><code>\[(?P&lt;module&gt;\w+)\]</code> — квадратные скобки нужно экранировать</li>
  <li>пробел(ы)</li>
  <li><code>(?P&lt;message&gt;.+)</code> — весь хвост</li>
</ul>
Используй <code>re.fullmatch(pattern, line)</code>. Если результат <code>None</code> — верни <code>None</code>,
иначе <code>m.groupdict()</code>.
</details>

<details><summary>Основная задача — почему именованные группы лучше нумерованных</summary>
Если добавишь новую группу в начале паттерна, нумерация всех остальных сдвинется —
сломаются все <code>group(3)</code>. С именованными группами это не страшно: ты
обращаешься по имени, а не по позиции. <code>m.groupdict()</code> сразу даёт нужный
словарь.
</details>
