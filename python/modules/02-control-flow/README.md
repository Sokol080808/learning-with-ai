# Модуль 02 — Управление потоком и функции

В модуле 01 ты работал с отдельными значениями: числа, строки, типы. Но программа —
это не прямая линия сверху вниз. Иногда нужно **выбрать** одну из веток, иногда —
**повторить** действие много раз, а ещё — собрать кусок логики в **функцию** и
переиспользовать её. Большая идея модуля: *поток выполнения — это инструмент, которым
ты управляешь явно*, а функция — способ дать куску логики имя и контракт.

Поняв этот модуль, ты перестанешь писать «простыни» кода и начнёшь думать блоками:
«здесь развилка», «здесь цикл», «а это — отдельная функция со своим входом и выходом».

## Идея 1. Ветвление и циклы: `if`/`elif`/`else`, `for` + `range`, `while`, `break`/`continue`

**Развилка.** `if` проверяет условие; `elif` — следующую проверку, если предыдущая не
сработала; `else` — «во всех остальных случаях». Проверки идут сверху вниз, и
выполняется **ровно одна** ветка — первая подошедшая.

```python
if x > 0:
    знак = "положительное"
elif x < 0:
    знак = "отрицательное"
else:
    знак = "ноль"
```

Блок в Python задаётся **отступом** (обычно 4 пробела), а не фигурными скобками. Это не
украшение — отступ и есть синтаксис. Сдвинул строку не туда — изменил смысл программы.

**Цикл по диапазону.** `range(n)` даёт числа `0, 1, …, n-1` (правый край НЕ включён —
запомни это, отсюда половина ошибок «на единицу»).

```python
for i in range(5):       # 0, 1, 2, 3, 4
    print(i)

for i in range(1, 6):    # 1, 2, 3, 4, 5  (старт включён, стоп — нет)
    print(i)
```

ПОЧЕМУ правый край не включён: так длина диапазона равна `stop - start`, а `range(len(a))`
аккуратно перебирает все индексы списка `a`, не вылезая за границу. Это соглашение, на
котором держится весь Python.

**Цикл по условию.** `while` крутится, пока условие истинно. Им пользуются, когда
заранее **не знаешь** число повторений (например, «пока число больше нуля»).

```python
n = 10
while n > 0:
    n = n // 2     # рано или поздно дойдём до 0 и выйдем
```

> Подвох: если условие никогда не станет ложным — это **бесконечный цикл**. Всегда
> убеждайся, что внутри тела что-то двигает тебя к выходу.

**`break` и `continue`.** `break` досрочно выходит из цикла целиком; `continue`
пропускает остаток текущей итерации и переходит к следующей.

```python
for i in range(100):
    if i == 5:
        break        # выйти из цикла, как только дошли до 5
    if i % 2 == 0:
        continue     # чётные пропускаем, ничего не печатаем
    print(i)         # напечатает только 1, 3
```

## Идея 2. Функции: `def`, аргументы, `return` vs `print`

Функция — это именованный кусок логики с входом (аргументы) и выходом (`return`).

```python
def square(x):
    return x * x

y = square(4)   # y == 16
```

Самая частая путаница новичка: **`return` и `print` — это разные вещи.**

- `return` **отдаёт значение** вызывающему коду. Его можно сохранить в переменную,
  передать дальше, сравнить в тесте.
- `print` просто **печатает текст на экран** и возвращает `None`. Тест печать «не
  видит» как результат.

```python
def bad_square(x):
    print(x * x)       # печатает, но НЕ возвращает

def good_square(x):
    return x * x       # возвращает — это и проверит тест

z = bad_square(4)      # на экране 16, но z == None  ← так тесты не проходят
```

ПОЧЕМУ это важно здесь: **все** задания модуля проверяются по тому, что функция
`return`-ит. Если ты `print`-нёшь ответ вместо `return`, тест увидит `None` и упадёт.
Печать — для человека; `return` — для программы.

Если функция дошла до конца без `return`, она возвращает `None` — неявно. Это норма,
когда функция «что-то делает», а не «что-то вычисляет».

## Идея 3. Truthiness: что считается `False`

В `if`/`while` условие не обязано быть `True`/`False`. Python проверяет «истинность»
любого значения. **Ложными** (falsy) считаются:

- `False`, `None`,
- ноль любого числового типа: `0`, `0.0`,
- **пустые** контейнеры и строки: `""`, `[]`, `{}`, `()`.

Всё остальное — **истинно** (truthy): непустая строка, ненулевое число, непустой список.

```python
s = ""
if s:                     # пустая строка — ложь, ветка НЕ выполнится
    print("есть текст")

nums = [1, 2, 3]
if nums:                  # непустой список — истина
    print("список не пуст")
```

ПОЧЕМУ это удобно: вместо `if len(s) > 0:` пишут просто `if s:` — короче и читается как
английская фраза «если строка есть». Но будь аккуратен: `if x:` отсеет и `0`, и `""`, и
`None` одинаково. Если тебе важно отличить «ноль» от «значения нет» — сравнивай явно:
`if x is None:`.

## Идея 4. Аргументы по умолчанию, `*args`/`**kwargs` (кратко)

**Значение по умолчанию** позволяет вызвать функцию, не передавая часть аргументов.

```python
def greet(name, greeting="Привет"):
    return f"{greeting}, {name}!"

greet("Аня")                 # "Привет, Аня!"
greet("Аня", "Здравствуй")   # "Здравствуй, Аня!"
```

> Подвох на будущее: **никогда не делай изменяемое значение по умолчанию** (`def f(x=[])`).
> Оно создаётся один раз и «запоминается» между вызовами. Подробно разберём позже —
> пока просто запомни: по умолчанию ставь неизменяемое (число, строку, `None`).

**`*args` и `**kwargs`** — способ принять «сколько угодно» аргументов. `*args` собирает
лишние позиционные аргументы в кортеж, `**kwargs` — именованные в словарь.

```python
def total(*args):           # args — кортеж всех переданных чисел
    s = 0
    for x in args:
        s += x
    return s

total(1, 2, 3)              # 6
total(10)                   # 10
total()                    # 0
```

ПОЧЕМУ это нужно: когда число аргументов заранее неизвестно (как у `print`, который
принимает сколько угодно значений). В заданиях модуля это не понадобится — но ты должен
узнавать `*args`/`**kwargs` в чужом коде и не пугаться звёздочек.

### Подсказки

<details><summary>fizzbuzz — порядок проверок</summary>
Делимость на 3 и на 5 одновременно — это делимость на 15. Если сначала проверишь
«делится на 3», то для 15 уже сработает ветка `"Fizz"` и до `"FizzBuzz"` дело не дойдёт.
Вывод: проверяй **15 первым** (либо проверяй оба условия сразу). Само число превращай в
строку через `str(i)`.
</details>
<details><summary>factorial — цикл или рекурсия</summary>
Самый простой путь — цикл: заведи результат `1` и умножай его на каждое число от `2` до
`n` включительно (`range(2, n + 1)`). Подумай, почему для `n == 0` и `n == 1` ответ уже
получается `1` сам собой, без отдельной проверки.
</details>
<details><summary>fib — два числа «бегут» по последовательности</summary>
Не считай рекурсивно «в лоб» — это медленно. Заведи две переменные `a, b = 0, 1` и `n`
раз делай шаг `a, b = b, a + b`. После цикла ответ в `a`. Проверь руками на `fib(0)`,
`fib(1)`, `fib(2)`.
</details>
<details><summary>count_vowels — регистр и принадлежность</summary>
Приведи строку к нижнему регистру (`s.lower()`), тогда не нужно отдельно проверять
заглавные. Дальше — цикл по символам: если символ входит в `"aeiou"`, прибавь к счётчику.
Оператор `in` проверяет принадлежность: `c in "aeiou"`.
</details>
<details><summary>gcd — алгоритм Евклида</summary>
Идея: НОД(a, b) == НОД(b, a % b), и так пока второе число не станет нулём — тогда первое
и есть ответ. В цикле `while b:` делай `a, b = b, a % b`. Заметь: `while b:` — это
truthiness из Идеи 3, «пока b не ноль».
</details>

## Микро-проект: CLI-калькулятор (ядро)

Первый микро-проект курса. Здесь идеи модуля собираются в одно маленькое цельное
приложение — **ядро калькулятора**. Это ещё не интерфейс командной строки целиком, а
сердце: одна функция, которая по двум числам и знаку операции возвращает результат.

Файл `calc.py`. Реализуй:

```python
calc(a: float, op: str, b: float) -> float
```

Поддержи четыре операции: `"+"`, `"-"`, `"*"`, `"/"`. Например, `calc(2, "+", 3) == 5.0`,
`calc(10, "/", 4) == 2.5`.

Особое внимание — **ошибкам**. Хорошая функция не «молча» возвращает мусор на плохой
вход, а честно сообщает о проблеме исключением:

- деление на ноль (`calc(1, "/", 0)`) — кидай `ValueError` с понятным сообщением;
- неизвестная операция (`calc(1, "?", 2)`) — тоже `ValueError` с понятным сообщением.

ПОЧЕМУ `ValueError`, а не «вернуть `None`» или напечатать ошибку: исключение **нельзя
случайно проигнорировать** — оно прервёт выполнение и заставит вызывающий код решить, что
делать. Это и есть культурный способ сказать «так нельзя». Тесты ловят оба случая через
`pytest.raises(ValueError)`.

### Подсказки

<details><summary>calc — структура</summary>
Серия `if op == "+": return a + b` / `elif op == "-": ...` и так далее. Деление вынеси
отдельно: перед ним проверь `if b == 0:` и в этом случае `raise ValueError("деление на ноль")`.
</details>
<details><summary>calc — неизвестная операция</summary>
Если ни одна ветка не подошла, значит `op` — не из разрешённых. В самом конце функции
(после всех веток) сделай `raise ValueError(...)` с сообщением вроде «неизвестная операция: ?».
Текст сообщения тесты не сверяют дословно — важен сам тип `ValueError`.
</details>
