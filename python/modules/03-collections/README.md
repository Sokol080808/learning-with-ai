# Модуль 03 — Коллекции: list, tuple, dict, set

В модулях 01–02 ты работал с одиночными значениями: число, строка, флаг. Но настоящие
программы оперируют **наборами** данных: список покупок, словарь «слово → количество»,
множество уникальных id. В Python для этого есть четыре «рабочие лошадки» — `list`,
`tuple`, `dict`, `set`. Они встроены в язык, и junior обязан чувствовать, **когда брать
какую**.

**Большая идея модуля:** структура данных — это не «куда сложить значения», а *выбор,
который заранее задаёт, какие операции будут дешёвыми, а какие — дорогими и неудобными*.
Список хорош для упорядоченной последовательности, словарь — для поиска по ключу,
множество — для уникальности и проверки «есть/нет». Возьмёшь не ту коллекцию — и простая
задача превратится в борьбу.

## Идея 1. list (изменяемый) vs tuple (неизменяемый)

`list` — упорядоченная **изменяемая** последовательность. Можно добавлять, удалять,
менять элементы на месте:

```python
xs = [10, 20, 30]
xs.append(40)     # [10, 20, 30, 40]  — добавили в конец
xs[0] = 99        # [99, 20, 30, 40]  — поменяли элемент
```

`tuple` — тоже упорядоченная последовательность, но **неизменяемая**. После создания её
нельзя править:

```python
point = (3, 4)
point[0] = 99     # TypeError: 'tuple' object does not support item assignment
```

**ПОЧЕМУ так, а не «список умеет всё, кортеж лишний»?** Неизменяемость — это не
ограничение, а гарантия. Кортеж говорит читателю кода: «эти данные — единое целое и не
поменяются» (координата `(x, y)`, строка таблицы, запись). А ещё неизменяемость даёт
бонус: кортеж можно положить ключом в словарь или элементом в множество, а список —
**нельзя** (он «нехешируемый»). Правило большого пальца: однородная коллекция, которая
растёт и меняется → `list`; фиксированный набор разнородных полей → `tuple`.

> Подвох: `(5)` — это просто число 5 в скобках, НЕ кортеж. Кортеж из одного элемента
> пишется с запятой: `(5,)`. А запятая вообще-то и делает кортеж: `a, b = 1, 2` справа —
> это кортеж `(1, 2)`.

## Идея 2. dict: ключ→значение; set: множество уникальных

`dict` (словарь) хранит пары **ключ → значение** и даёт почти мгновенный поиск по ключу:

```python
ages = {"Аня": 25, "Боря": 30}
ages["Аня"]            # 25      — достаём по ключу
ages["Витя"] = 19      # добавили новую пару
"Боря" in ages         # True    — проверка наличия ключа, очень быстрая
ages.get("Кто-то", 0)  # 0       — безопасно: нет ключа → значение по умолчанию
```

`set` (множество) хранит только **уникальные** значения, без порядка:

```python
seen = {1, 2, 2, 3}    # {1, 2, 3} — дубликат сам отвалился
3 in seen              # True
seen.add(2)            # ничего не изменится — 2 уже есть
```

**ПОЧЕМУ они такие быстрые на поиск?** И `dict`, и `set` внутри устроены на хеш-таблице.
Проверка `x in d` или `x in s` в среднем не зависит от размера коллекции (это называют
O(1)), тогда как `x in список` перебирает элементы по одному (O(n)). Поэтому, если тебе
нужно много раз спрашивать «а есть ли такой элемент?» — это явный сигнал взять `set` или
`dict`, а не `list`.

> Подвох: ключи словаря и элементы множества обязаны быть **хешируемыми** (то есть
> неизменяемыми) — число, строка, кортеж подойдут, а список или другой словарь — нет.

## Идея 3. Индексация и срезы xs[a:b:c]

К элементам последовательности обращаются по индексу, начиная с **0**. Отрицательные
индексы считают с конца:

```python
xs = [10, 20, 30, 40, 50]
xs[0]    # 10   — первый
xs[-1]   # 50   — последний
xs[-2]   # 40   — предпоследний
```

Срез `xs[a:b:c]` вырезает **под-последовательность**: с индекса `a` (включительно) до `b`
(не включая), с шагом `c`. Любую границу можно опустить:

```python
xs[1:4]    # [20, 30, 40]  — с 1 по 3
xs[:3]     # [10, 20, 30]  — с начала по 2
xs[2:]     # [30, 40, 50]  — со 2 до конца
xs[::2]    # [10, 30, 50]  — каждый второй
xs[::-1]   # [50, 40, 30, 20, 10] — задом наперёд (шаг -1)
```

**ПОЧЕМУ правая граница не включается?** Это соглашение `[a, b)` делает арифметику
приятной: длина среза ровно `b - a`, а соседние срезы `xs[:k]` и `xs[k:]` стыкуются без
дыр и нахлёста. Важно: срез **создаёт новый список** (копию участка), исходный не
портится. Срезы работают и для строк, и для кортежей — это общий механизм
последовательностей.

## Идея 4. Генераторы списков/словарей/множеств (comprehensions)

Очень частый шаблон — «пройтись по коллекции и собрать новую». Можно циклом:

```python
squares = []
for n in range(5):
    squares.append(n * n)   # [0, 1, 4, 9, 16]
```

А можно короче и выразительнее — **генератором списка** (list comprehension):

```python
squares = [n * n for n in range(5)]            # [0, 1, 4, 9, 16]
evens   = [n for n in range(10) if n % 2 == 0] # [0, 2, 4, 6, 8]  — с фильтром
```

Та же идея работает для словарей и множеств:

```python
lengths = {w: len(w) for w in ["раз", "птица"]}  # {'раз': 3, 'птица': 5}  — dict comp
uniq    = {n % 3 for n in range(10)}             # {0, 1, 2}               — set comp
```

**ПОЧЕМУ это лучше, чем цикл с `.append`?** Comprehension — не просто «короче». Он
говорит читателю: «я строю новую коллекцию из существующей», и всё намерение видно в
одной строке: что берём, откуда, при каком условии. Это идиоматичный Python — junior
должен и читать его свободно, и писать. Но не переусердствуй: если внутрь лезет сложная
логика или вложенность в три уровня — обычный цикл понятнее.

## Идея 5. Распаковка последовательностей

Python позволяет «разложить» любую последовательность по переменным прямо в операторе
присваивания — это называют **распаковкой** (unpacking):

```python
pair = (10, 20)
a, b = pair          # a = 10, b = 20

# Обмен значениями — без временной переменной:
a, b = b, a          # справа Python сначала строит кортеж (b, a), потом распаковывает

# Звёздочка собирает «остаток»:
first, *rest = [1, 2, 3, 4]   # first=1, rest=[2, 3, 4]
*init, last  = [1, 2, 3, 4]   # init=[1, 2, 3], last=4
head, *_, tail = range(10)    # head=0, tail=9 — явно игнорируем середину

# Распаковка в for (кортежи из .items()):
for key, value in {"a": 1, "b": 2}.items():
    print(key, "->", value)

# Распаковка в вызове функции — оператор *:
xs = [1, 2, 3]
print(*xs)           # эквивалентно print(1, 2, 3)
```

**ПОЧЕМУ это идиоматично?** `a, b = b, a` в одну строку — это не трюк, а рекомендованный
стиль. Starred-распаковка `first, *rest = xs` заменяет `xs[0]` и `xs[1:]`, но читается
как объявление намерения: «первый элемент — голова, остальное — хвост». В функциональных
структурах данных такое разделение — основа рекурсии. В реальном коде распаковка
встречается в `for k, v in d.items()`, в возврате нескольких значений из функции и при
разборе строк через `.split()`.

> Подвох: starred-распаковка работает только в Python 3; и в одном присваивании может
> быть не более одного `*`-элемента. `a, *b, *c = xs` — это SyntaxError.

## Идея 6. Сортировка с key: sorted() vs list.sort()

Python предлагает два способа отсортировать последовательность:

| | `sorted(xs, ...)` | `xs.sort(...)` |
|---|---|---|
| Результат | **новый список** | изменяет `xs` **на месте**, возвращает `None` |
| Входной тип | любой итерируемый | только `list` |
| Когда брать | нужна копия / вход не список | нужна экономия памяти, список |

Главный параметр — `key`: функция, которую Python применяет к каждому элементу *перед*
сравнением, но **не меняет** сам элемент в результате:

```python
words = ["банан", "яблоко", "кот", "слон"]

sorted(words, key=len)                     # ['кот', 'слон', 'банан', 'яблоко']
sorted(words, key=len, reverse=True)       # ['яблоко', 'банан', 'слон', 'кот']

# key=str.lower — сортировать без учёта регистра:
sorted(["Banana", "apple", "Cherry"], key=str.lower)  # ['apple', 'Banana', 'Cherry']

# key=lambda для сложных правил:
pairs = [(1, "b"), (2, "a"), (1, "a")]
sorted(pairs, key=lambda t: (t[0], t[1]))  # [(1, 'a'), (1, 'b'), (2, 'a')]
```

**СТАБИЛЬНОСТЬ.** Python гарантирует **стабильную сортировку**: равные элементы (по
результату `key`) остаются в исходном относительном порядке. Это позволяет сортировать
по нескольким ключам за два прохода — сначала по вторичному, потом по первичному.

**ПОЧЕМУ `key`, а не `cmp`?** В Python 3 нет параметра `cmp` (как в Python 2 или
Java-компараторах). Вместо него — `key`: чище, быстрее (функция вызывается один раз на
элемент, а не при каждом сравнении), и легче составлять.

> Подвох: `xs.sort()` возвращает `None`, не список. Очень распространённая ошибка —
> написать `xs = xs.sort()` и получить `None` вместо отсортированного списка.

## Идея 7. Модуль collections: Counter, defaultdict, deque, namedtuple

Стандартная библиотека содержит модуль `collections` с «усиленными» версиями базовых
структур. Junior обязан знать четыре:

### Counter — словарь-счётчик

```python
from collections import Counter

text = "абракадабра"
c = Counter(text)          # Counter({'а': 5, 'б': 2, 'р': 2, 'к': 1, 'д': 1})
c.most_common(2)           # [('а', 5), ('б', 2)]
c['ж']                     # 0  — нет KeyError, просто ноль
c + Counter("аааббб")      # сложение счётчиков
```

**Твой `Multiset` — это упрощённый `Counter`:** та же идея «словарь element→count»,
только без арифметики над счётчиками и операций `update`/`subtract`.

### defaultdict — словарь с значением по умолчанию

```python
from collections import defaultdict

groups = defaultdict(list)
for word in ["кот", "кит", "пёс", "поле"]:
    groups[word[0]].append(word)
# defaultdict(<class 'list'>, {'к': ['кот', 'кит'], 'п': ['пёс', 'поле']})
# Ключ создаётся автоматически при первом обращении.
```

Без `defaultdict` пришлось бы писать `groups.setdefault(word[0], []).append(word)`.

### deque — двусторонняя очередь

```python
from collections import deque

d = deque([1, 2, 3], maxlen=4)   # ограниченный размер
d.appendleft(0)   # O(1) — список так не умеет эффективно!
d.popleft()       # O(1)
d.append(4)       # правый конец
# При переполнении maxlen старый элемент автоматически вытесняется с противоположного конца.
```

`list.insert(0, x)` — это O(n), а `deque.appendleft(x)` — O(1). Берёшь `deque`, когда
нужно **быстро добавлять/удалять с обоих концов** (BFS-очередь, скользящее окно, история
браузера).

### namedtuple — кортеж с именами полей

```python
from collections import namedtuple

Point = namedtuple("Point", ["x", "y"])
p = Point(3, 4)
p.x, p.y   # 3, 4  — обращение по имени
p[0]       # 3     — обращение по индексу, как у кортежа
```

`namedtuple` — это лёгкий способ дать смысловые имена полям записи, не писав полноценный
класс. Он неизменяем (как `tuple`) и хешируем.

## Идея 8. dict глубже: views, setdefault, merge, порядок вставки

### Динамические view-объекты

`d.keys()`, `d.values()`, `d.items()` возвращают **не копии**, а «живые» представления
(view), которые автоматически отражают изменения словаря:

```python
d = {"a": 1, "b": 2}
keys = d.keys()   # dict_keys(['a', 'b'])
d["c"] = 3
list(keys)        # ['a', 'b', 'c'] — обновилось само!
```

Это важно: если ты итерируешься по `d.keys()` и одновременно меняешь `d` — получишь
`RuntimeError`. Нужно явно скопировать: `for k in list(d.keys()):`.

### setdefault — добавить, если ключа нет

```python
d = {"a": [1]}
d.setdefault("a", []).append(2)   # ключ есть → просто вернул значение: [1, 2]
d.setdefault("b", []).append(9)   # ключа нет → создал [] и вернул его: {"b": [9]}
```

Это компактная альтернатива `if key not in d: d[key] = default`.

### Слияние словарей (Python 3.9+)

```python
defaults = {"debug": False, "timeout": 30}
overrides = {"timeout": 10, "retries": 3}

merged = defaults | overrides          # новый словарь; правый перезаписывает левый
defaults |= overrides                  # обновить defaults на месте (как update)
# merged == {"debug": False, "timeout": 10, "retries": 3}
```

До Python 3.9 то же делалось через `{**defaults, **overrides}`.

### Порядок вставки

С Python 3.7 `dict` **гарантированно сохраняет порядок вставки**. Итерирование по
словарю воспроизводит ключи в том порядке, в котором они были добавлены:

```python
d = {}
d["второй"] = 2
d["первый"] = 1
list(d)   # ['второй', 'первый'] — порядок вставки, не алфавитный!
```

## Идея 9. set-операции и frozenset

Множества — не только для «убрать дубликаты». У них богатая алгебра:

```python
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b    # {1, 2, 3, 4, 5, 6}  — объединение (union)
a & b    # {3, 4}              — пересечение (intersection)
a - b    # {1, 2}              — разность: в a, но не в b
b - a    # {5, 6}
a ^ b    # {1, 2, 5, 6}        — симметричная разность (XOR): только в одном
```

Отношения подмножеств:

```python
{1, 2} <= {1, 2, 3}    # True   — подмножество (нестрогое)
{1, 2} <  {1, 2, 3}    # True   — строгое подмножество
{1, 2, 3} >= {1, 2}    # True   — надмножество
```

### frozenset — хешируемое множество

`frozenset` — неизменяемый аналог `set`. Потому что `set` изменяем, его **нельзя**
положить ключом в словарь или элементом в другое множество:

```python
{frozenset({1, 2}): "пара"}   # OK  — frozenset хешируемый
{{1, 2}: "пара"}              # TypeError: unhashable type: 'set'

# Множество множеств — через frozenset:
pairs = {frozenset({1, 2}), frozenset({3, 4}), frozenset({1, 2})}
# {frozenset({1, 2}), frozenset({3, 4})} — дублей нет
```

Правило: **хешируемость = неизменяемость**. `int`, `str`, `tuple` (из хешируемых)
хешируемы; `list`, `dict`, `set` — нет. `frozenset` — хешируемый `set`.

## Идея 10. Подвохи: slice-assignment, alias vs copy, мутабельный default

### Slice-assignment — изменение части списка на месте

```python
xs = [1, 2, 3, 4, 5]
xs[1:3] = [20, 30]    # заменить элементы 1..2: [1, 20, 30, 4, 5]
xs[1:3] = [99]        # заменить двумя одним: [1, 99, 4, 5]  — список стал короче!
xs[2:2] = [10, 11]    # вставить без удаления: [1, 99, 10, 11, 4, 5]
xs[::2] = [0, 0, 0]   # extended slice: кол-во элементов должно совпадать!
```

Slice-assignment мутирует исходный список **на месте** — никакой копии не создаётся.

### alias (псевдоним) vs copy (копия)

```python
a = [1, 2, 3]
b = a         # ПСЕВДОНИМ — b и a указывают на один и тот же список
b.append(4)
print(a)      # [1, 2, 3, 4] — изменение видно через a!

c = a[:]      # ПОВЕРХНОСТНАЯ КОПИЯ — новый список, те же объекты-элементы
c.append(5)
print(a)      # [1, 2, 3, 4] — a не тронут

d = a.copy()  # то же самое — явный метод
import copy
e = copy.deepcopy(a)   # ГЛУБОКАЯ КОПИЯ — нужна для списков списков
```

**Для словарей** то же: `d2 = d` — псевдоним, `d2 = d.copy()` — мелкая копия,
`copy.deepcopy(d)` — глубокая.

### Мутабельный аргумент по умолчанию — классическая ловушка

```python
def append_to(elem, lst=[]):   # НЕПРАВИЛЬНО: [] создаётся один раз при определении функции
    lst.append(elem)
    return lst

append_to(1)  # [1]
append_to(2)  # [1, 2]  — WTF: мы не передавали список!
append_to(3)  # [1, 2, 3]

# ПРАВИЛЬНО: сигнализируем «ничего не передано» через None:
def append_to_fixed(elem, lst=None):
    if lst is None:
        lst = []
    lst.append(elem)
    return lst
```

Дефолтные значения вычисляются **один раз** — при загрузке модуля, не при каждом вызове.
Изменяемый дефолт разделяется между всеми вызовами. Правило: **никогда не используй
изменяемые объекты (`[]`, `{}`, `set()`) как значение аргумента по умолчанию**.

## Задание (существенное): Multiset — мультимножество

Обычное `set` хранит каждый элемент лишь раз: добавляешь `2` трижды — в множестве всё равно
один `2`. Но иногда нужно помнить **сколько раз** каждый элемент встретился. Например, в
частотном анализе текста: слово «кот» встречается 5 раз, «пёс» — 2 раза. В Python для этого
есть `collections.Counter` — но понять его изнутри лучше всего, **написав подобную структуру самому**.

Реализуй класс `Multiset` в файле `containers.py`. Вся сигнатура уже есть в стабе — тела
методов заменяй ты.

### API

```python
ms = Multiset()                   # пустое
ms = Multiset([1, 1, 2, 3])       # из любого итерируемого

ms.add("кот")                     # счётчик "кот" += 1
ms.add("кот", count=4)            # счётчик "кот" += 4
ms.discard("кот")                 # счётчик "кот" -= 1 (не ниже 0)
ms.discard("кот", count=10)       # обнуляет, не падает

ms.count("кот")      # -> int     сколько раз "кот" в мультимножестве
len(ms)              # -> int     суммарное число вхождений
"кот" in ms          # -> bool    True если count > 0

ms.most_common()     # -> [(elem, count), ...]  все, по убыванию count
ms.most_common(2)    # -> топ-2

a | b                # объединение: max(a.count(x), b.count(x))
a & b                # пересечение: min(a.count(x), b.count(x))
```

**Краевые случаи, которые проверяют тесты:**
- `add` / `discard` с `count <= 0` должны бросить `ValueError`.
- `discard` на отсутствующем элементе — не падает, счётчик остаётся `0`.
- `most_common(0)` → `[]`.
- `most_common(n)` при `n > числа уникальных элементов` → все уникальные.
- Элементы с нулевым счётчиком не попадают в `most_common`, `__contains__` возвращает `False`.
- `|` и `&` возвращают **новый** `Multiset`, не изменяя операнды.

### Подсказки

<details><summary>С чего начать — внутреннее хранилище</summary>
Мультимножество — это просто словарь <code>dict[element, int]</code>, где значение — счётчик.
Назови его, например, <code>self._data: dict = {}</code>. Метод <code>count(x)</code> — это
<code>self._data.get(x, 0)</code>. Всё остальное строится поверх.
</details>

<details><summary>Инициализация из iterable</summary>
Если <code>iterable</code> не <code>None</code>, пройди по нему циклом и для каждого элемента
вызови <code>self.add(element)</code>. Ты уже реализовал <code>add</code> — пусть работает!
</details>

<details><summary>discard — как не уйти ниже нуля</summary>
Уменьши счётчик, но не ниже нуля: <code>self._data[x] = max(0, self._data.get(x, 0) - count)</code>.
После этого, если значение стало <code>0</code>, можно оставить его (или удалить ключ — оба
варианта работают, но удаление чище: ключи с нулём не будут «замусоривать» словарь).
</details>

<details><summary>most_common — сортировка пар</summary>
Собери список <code>[(elem, cnt) for elem, cnt in self._data.items() if cnt > 0]</code>,
отсортируй по убыванию: <code>sorted(..., key=lambda t: t[1], reverse=True)</code>.
Потом обрежь: <code>[:n]</code> — если <code>n</code> не <code>None</code>.
</details>

<details><summary>union и intersection — алгоритм</summary>
Для объединения: собери все уникальные ключи из обоих словарей. Для каждого ключа положи
<code>max(self.count(x), other.count(x))</code> в новый <code>Multiset</code>. Можно прямо
через <code>result.add(x, count=...)</code>.<br>
Для пересечения — то же самое, но <code>min</code>; пропусти те, где <code>min == 0</code>.
</details>
