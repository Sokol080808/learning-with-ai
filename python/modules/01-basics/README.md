# Модуль 01 — Типы, переменные, числа, строки

Это фундамент. Python кажется простым (нет `int*`, нет `delete`), но у этой простоты
есть свои ловушки — и почти все они живут именно здесь, в типах и делении. Поняв этот
модуль, ты перестанешь удивляться, почему `10 / 2` вдруг `5.0`, а не `5`.

**Большая идея модуля:** в Python у каждого значения есть **тип**, и тип определяет,
что с этим значением можно делать. Переменная — это всего лишь **имя**, привязанное к
значению; сами значения бывают изменяемыми и нет. Большая часть «магии» Python — это
просто аккуратное обращение с типами и осознанный выбор операции (особенно при делении).

## Идея 1. Базовые типы: `int`, `float`, `str`, `bool`, `None` и `type()`

В Python не нужно объявлять тип заранее — он определяется значением, которое ты
присваиваешь. Но тип всё равно есть, и его видно через `type()`:

```python
x = 42          # int  — целое число
y = 3.14        # float — число с дробной частью
name = "Аня"    # str  — строка
flag = True     # bool — True или False
nothing = None  # NoneType — «значения нет»

print(type(x))     # <class 'int'>
print(type(y))     # <class 'float'>
print(type(flag))  # <class 'bool'>
```

**ПОЧЕМУ это важно.** Переменная в Python — это *ярлык*, наклеенный на значение, а не
коробочка фиксированного типа (как `int x` в C++). Поэтому одна и та же переменная
может сегодня указывать на `int`, а завтра на `str`. Это удобно, но требует
дисциплины: путаница в типах — частый источник ошибок. `type()` — твой первый
инструмент диагностики: «а что у меня вообще в руках?».

> Тонкость: `bool` в Python — это подвид `int`. `True == 1` и `False == 0`, а
> `True + True == 2`. Пока просто держи в голове — пригодится позже.

## Идея 2. `//` (целочисленное) vs `/` (дробное) — ловушка

Это **самая частая** ошибка новичка в Python. Два разных деления:

```python
7 / 2    # 3.5   — обычное деление, ВСЕГДА даёт float
7 // 2   # 3     — целочисленное деление, «отбрасывает» дробную часть (округление вниз)
10 / 2   # 5.0   — да, float! даже если делится нацело
10 // 2  # 5     — int
7 % 2    # 1     — остаток от деления
```

**ПОЧЕМУ это ловушка.** В C++ `7 / 2` для двух `int` даёт `3` (целое). Новичок,
пришедший из C++ (или просто ожидающий «целое делить на целое = целое»), пишет `/` и
получает `3.5` там, где хотел `3`. И наоборот: ждёшь точное `5`, а получаешь `5.0`, и
оно не совпадает в сравнении «как строка». Правило: нужен **целый** результат
(индексы, количества, страницы) — бери `//`. Нужно **точное** дробное — бери `/`.

> Подводный камень: `//` округляет **вниз** (к минус бесконечности), а не «к нулю».
> `-7 // 2 == -4`, а не `-3`. Для отрицательных это удивляет — запомни.

## Идея 3. f-строки и форматирование `f"{x:.2f}"`

f-строка — это строка с буквой `f` перед кавычкой, внутри которой в `{...}` можно
вставлять выражения:

```python
name = "Аня"
age = 19
print(f"Привет, {name}! Тебе {age}.")   # Привет, Аня! Тебе 19.

pi = 3.14159
print(f"{pi:.2f}")     # 3.14   — округлить до 2 знаков после точки
print(f"{pi:.0f}")     # 3      — без дробной части
print(f"{1/3:.4f}")    # 0.3333
```

**ПОЧЕМУ так удобно.** До f-строк строки склеивали через `+` и `str(...)`:
`"Привет, " + name + "!"` — длинно и легко забыть преобразовать число в строку.
f-строка читается как итоговый текст и сама зовёт `str()` для каждого `{...}`.
Спецификатор `:.2f` — это «формат: 2 знака после точки, тип float». Он не меняет само
число, только то, **как оно выглядит** в строке.

## Идея 4. Преобразования `int()`, `float()`, `str()` — и где это нужно

Типы не превращаются друг в друга сами. Преобразуешь — явно:

```python
int("42")      # 42     — строка → целое
float("3.14")  # 3.14   — строка → float
str(42)        # "42"   — число → строка
int(3.9)       # 3      — float → int ОТБРАСЫВАЕТ дробь (не округляет!)
float(7)       # 7.0    — int → float
```

**ПОЧЕМУ это нужно.** Самый частый случай — ввод: `input()` всегда возвращает **строку**,
даже если пользователь набрал «42». Чтобы посчитать, строку надо превратить в число:
`n = int(input())`. Обратное — при выводе: чтобы склеить число с текстом через `+`,
его надо сделать строкой. И ещё: нельзя складывать `str` и `int` — `"2" + 2` это
ошибка `TypeError`, Python не угадывает за тебя.

> Подводный камень: `int("3.14")` — это ошибка! `int()` от строки понимает только целые.
> Сначала `float("3.14")`, потом при желании `int(...)`.

## Идея 5. Truthiness: что такое «истинное» значение

В Python любой объект может использоваться в условии `if`. Питон тихо преобразует его
в `bool` — и называет это **truthiness** (истинностью). Вот полный список **falsy**-значений
(всё остальное — truthy):

| Falsy-значение | Тип |
|---|---|
| `0` | `int` |
| `0.0` | `float` |
| `""` | `str` (пустая строка) |
| `None` | `NoneType` |
| `[]` | `list` (пустой список) |
| `{}` | `dict` (пустой словарь) |
| `()` | `tuple` (пустой кортеж) |

```python
bool(0)     # False
bool(0.0)   # False
bool("")    # False
bool(None)  # False
bool([])    # False

bool(1)     # True
bool(-7)    # True   — любое ненулевое число
bool("hi")  # True
bool([0])   # True   — список с одним элементом (даже если он 0)
```

**Два важных идиоматических паттерна:**

```python
# Хорошо — проверить, не пустая ли строка:
if s:
    print("строка не пустая")

# Хорошо — проверить, что значение точно None:
if x is None:
    print("значение не задано")
```

**ПОЧЕМУ это разные вещи.** `if s:` ловит пустую строку, `0`, `None`, `[]` — всё
сразу. Иногда это именно то, что нужно (коротко). Но если ты хочешь отличить `None`
от `0` или от `""`, используй явное `x is None` — иначе словишь ошибку: ноль — тоже
falsy, и условие «сработает», хотя значение _есть_, просто равно нулю.

> Подводный камень: `bool(0.0)` — это `False`, поэтому `if result:` не заменяет
> проверку успешности вычисления, если результат может быть нулём. Пиши
> `if result is not None:` или проверяй явно.

## Идея 6. Знак остатка `%` — питон против C/C++

В Python оператор `%` (остаток от деления) **следует знаку делителя**, а не делимого.
Это отличается от C/C++, где остаток следует знаку делимого:

```python
-3 % 2    # 1  (Python: следует знаку делителя 2, то есть +)
3 % -2    # -1 (Python: следует знаку делителя -2, то есть -)
-3 % -2   # -1 (оба отрицательных — результат тоже отрицательный)
```

В C/C++: `-3 % 2 == -1`. В Python: `-3 % 2 == 1`. Это систематическое различие.

**Инвариант (верен всегда):** для любых чисел `a` и `b != 0`:

```python
a == (a // b) * b + a % b
```

Проверь на `-3` и `2`:
- `(-3) // 2 == -2`  (деление вниз, к минус бесконечности)
- `(-2) * 2 + 1 == -3`  ✓

Это самосогласованная система: `//` округляет вниз, а `%` «добирает» остаток
в том же направлении.

**Зачем это знать.** Если ты пишешь алгоритм с остатком от отрицательных чисел
(хеш-функция, позиция в кольцевом буфере, проверка чётности), поведение Python —
твой союзник: `n % 2` всегда `0` или `1`, никогда `0` или `-1`. Проверка чётности
`n % 2 == 0` безопасна для отрицательных чисел.

> Запомни мнемонику: в Python остаток **«дотягивается»** до знака делителя. В C —
> до знака делимого. При портировании кода это типичный источник багов.

## Идея 7. `==` vs `is` — равенство значений против идентичности

```python
a = [1, 2, 3]
b = [1, 2, 3]
c = a

a == b   # True  — значения одинаковые
a is b   # False — разные объекты в памяти
a is c   # True  — c — это тот же объект, что a
```

- `==` спрашивает: **«одинаковые ли значения?»**
- `is` спрашивает: **«это один и тот же объект в памяти?»** (та же идентичность)

**Bool-синглтоны.** `True` и `False` — это синглтоны: в Python всегда существует
ровно один объект `True` и один объект `False`. Поэтому для них `is` работает как `==`:

```python
is_even(4) is True   # True  — безопасно
is_even(4) == True   # True  — тоже работает, но стиль хуже
```

Хороший стиль: пиши `if x:` вместо `if x == True:`, и `if x is None:` вместо
`if x == None:` (для None `is` — единственно правильный вариант).

**Приоритет операторов.** В Python операторы выполняются в определённом порядке
(от высшего к низшему): `**` > унарный `-` > `* / // %` > `+ -` > сравнения
`== != < > <= >=` > `not` > `and` > `or`.

Важный сюрприз:

```python
2 ** 3 ** 2     # 512, не 64: ** правоассоциативен! читается как 2 ** (3**2)
-2 ** 2         # -4, не 4: это -(2**2), унарный минус СЛАБЕЕ **
(-2) ** 2       # 4 — скобки нужны, чтобы получить квадрат -2
```

Правило практики: если сомневаешься — ставь скобки. Код `2 ** (n - 1)` читается
лучше, чем `2 ** n - 1` (которое — `(2**n) - 1` из-за приоритета).

## Идея 8. Почему `0.1 + 0.2 != 0.3` и как с этим жить

```python
0.1 + 0.2          # 0.30000000000000004
0.1 + 0.2 == 0.3   # False!
```

Это не баг Python — это фундаментальное свойство **IEEE 754**, стандарта хранения
вещественных чисел в памяти. Большинство дробей в двоичной системе счисления
бесконечны (как `1/3` в десятичной), поэтому они округляются при сохранении в 64-битный
float. Когда ты складываешь два таких «почти-правильных» числа, ошибки накапливаются.

**Что делать при сравнении float:**

```python
import math
import pytest

# pytest.approx — для тестов:
assert 0.1 + 0.2 == pytest.approx(0.3)   # OK

# math.isclose — в обычном коде:
math.isclose(0.1 + 0.2, 0.3)   # True
math.isclose(0.1 + 0.2, 0.3, rel_tol=1e-9)   # задать точность явно
```

`math.isclose(a, b)` по умолчанию считает числа «почти равными», если относительная
погрешность `|a - b| / max(|a|, |b|)` меньше `1e-9`. Этого достаточно для большинства
физических расчётов.

**Когда float недостаточно.** Деньги, налоги, финансовые расчёты — используй
`decimal.Decimal("0.1")`, он хранит числа в точной десятичной форме. Для обычных
расчётов (физика, ML, статистика) float вполне хватает — просто не сравнивай с `==`.

> Правило: **никогда не пиши `a == b` для float**. Пиши `math.isclose(a, b)` или
> `pytest.approx` в тестах.

## Идея 9. Неизменяемость строк (immutability)  <!-- была Идея 5 -->

Строку **нельзя изменить на месте** — у неё нет «перезаписи символа»:

```python
s = "кот"
# s[0] = "р"     # TypeError! строки неизменяемы
s2 = "р" + s[1:]  # делаем НОВУЮ строку: "рот"
print(s)          # "кот" — старая строка не тронута
```

Методы строк тоже не меняют оригинал, а **возвращают новую строку**:

```python
s = "Привет"
s.upper()   # "ПРИВЕТ" — это новая строка...
print(s)    # "Привет" — ...а s осталась прежней!
big = s.upper()   # хочешь результат — сохрани его
```

**ПОЧЕМУ так сделано.** Неизменяемость даёт безопасность: если ты передал строку в
функцию, она не сможет «испортить» твою строку из-под тебя. Цена — каждое «изменение»
создаёт новую строку. Главный практический вывод: вызов вроде `s.strip()` или
`s.replace(...)` **бесполезен**, если ты не сохранил результат. `s.upper()` сам по себе
ничего не «делает» с `s`.
