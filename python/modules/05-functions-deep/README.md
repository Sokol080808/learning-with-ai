# Модуль 05 — Функции глубже: замыкания, декораторы

В первых модулях функция была для тебя «коробкой, которая что-то считает». Теперь поднимаемся
на уровень выше. **Большая идея модуля:** в Python функция — это обычный объект, который можно
положить в переменную, передать в другую функцию и вернуть из неё. Как только это укладывается в
голове, открываются три мощные техники: функции высшего порядка, замыкания и декораторы. На них
держится половина «магии» библиотек, которые ты будешь использовать как junior.

## Идея 1. Функции — объекты первого класса

«Объект первого класса» — занудный термин с простым смыслом: с функцией можно делать всё то же,
что с числом или строкой. Положить в переменную, передать аргументом, вернуть из функции, хранить
в списке.

```python
def shout(s: str) -> str:
    return s.upper() + "!"

f = shout            # НЕ вызываем (нет скобок) — берём саму функцию как значение
print(f("hi"))       # 'HI!'  — вызвали уже через новое имя

def call_on(fn, value):
    return fn(value)  # fn — это переданная функция, мы её здесь вызываем

print(call_on(shout, "hey"))   # 'HEY!'
```

**Почему это важно.** `shout` (без скобок) — это сам объект-функция; `shout("hi")` — её *вызов*.
Разница принципиальна: пока ты не поставил скобки, ты передаёшь поведение, а не результат. Именно
поэтому в `sorted(..., key=len)` мы пишем `len`, а не `len()`.

## Идея 2. `lambda` и функции высшего порядка

**Функция высшего порядка** — та, что принимает функцию аргументом или возвращает функцию.
Классика стандартной библиотеки: `map`, `filter`, `sorted(key=...)`. Когда функция-аргумент
крошечная и нужна разово, её удобно записать «на месте» через `lambda`.

```python
nums = [1, 2, 3, 4]
squares = list(map(lambda x: x * x, nums))      # [1, 4, 9, 16]
evens   = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]

words = ["banana", "kiwi", "apple"]
by_len = sorted(words, key=len)                  # ['kiwi', 'apple', 'banana']
```

**Почему `key`, а не сравнение руками.** `sorted` сам знает, как сравнивать; ему не хватает только
одного — *по какому признаку*. `key=len` говорит: «для каждого элемента возьми его длину и сравнивай
по ней». Сама функция `key` вызывается один раз на элемент, а не на каждое сравнение, — это и
быстро, и читаемо.

> Подвох: `lambda` — это просто короткая безымянная функция, тело которой обязано быть *одним
> выражением* (без `return`, без нескольких строк). Если логика сложнее — заведи обычную `def`.
> `map`/`filter` возвращают «ленивый» итератор, поэтому мы оборачиваем их в `list(...)`.

## Идея 3. Замыкания: функция помнит окружение

**Замыкание** возникает, когда одна функция определена внутри другой и использует переменные
внешней. Внутренняя функция «запоминает» это окружение и продолжает им пользоваться даже после
того, как внешняя функция завершилась.

```python
def make_adder(n: int):
    def adder(x: int) -> int:
        return x + n     # n взято из внешней функции — оно «захвачено»
    return adder         # возвращаем настроенную функцию

add5 = make_adder(5)
print(add5(10))          # 15  — add5 помнит, что n == 5
print(make_adder(100)(1))  # 101 — у каждого вызова своё окружение
```

**Почему это работает.** После `return adder` сама `make_adder` уже завершилась, но `n` не
исчезло: Python держит его живым, пока живёт `adder`. Каждый вызов `make_adder` создаёт *новое*
независимое окружение — поэтому `add5` и «add100» не мешают друг другу. Это аккуратный способ
«зашить» настройку в функцию без классов и глобальных переменных.

## Идея 4. Декораторы: обёртка над функцией

Декоратор — это функция, которая принимает функцию и возвращает её «обёрнутую» версию. Синтаксис
`@decorator` над `def` — лишь короткая запись для `func = decorator(func)`.

```python
import functools

def logged(func):
    @functools.wraps(func)            # сохранить имя/docstring исходной функции
    def wrapper(*args, **kwargs):
        print(f"вызываю {func.__name__}")
        return func(*args, **kwargs)  # дёргаем оригинал и отдаём его результат
    return wrapper

@logged
def add(a, b):
    return a + b

add(2, 3)            # печатает 'вызываю add', возвращает 5
print(add.__name__)  # 'add' — благодаря @wraps, а не 'wrapper'
```

**Зачем `@wraps`.** Без него обёртка «крадёт» личность исходной функции: `add.__name__` стал бы
`'wrapper'`, а docstring потерялся бы. `functools.wraps(func)` копирует метаданные оригинала на
обёртку — отладка и `help()` снова показывают правду. Декораторы хороши тем, что добавляют
поведение (логирование, кэш, проверки), не трогая тело самой функции.

## Идея 5. `*args` / `**kwargs`

Чтобы обёртка подходила к *любой* функции, ей нужно уметь принимать сколько угодно позиционных и
именованных аргументов и передавать их дальше без изменений. Для этого есть `*args` (кортеж
позиционных) и `**kwargs` (словарь именованных).

```python
def wrapper(*args, **kwargs):
    # args   == (2, 3)        — позиционные сложены в кортеж
    # kwargs == {'c': 4}      — именованные сложены в словарь
    return real(*args, **kwargs)   # * и ** «распаковывают» обратно при вызове

real(2, 3, c=4)
```

**Почему это ключ к декораторам.** В Идее 4 обёртка работала с `add(a, b)` и с любой другой
функцией именно потому, что объявлена как `(*args, **kwargs)` — она не знает заранее, сколько
аргументов придёт. Та же звёздочка при *вызове* делает обратное: разворачивает кортеж/словарь
обратно в отдельные аргументы. Это пригодится тебе в задании `memoize`.

## Идея 6. `nonlocal`: замыкание на запись

В Идее 3 замыкание только *читало* переменную внешней функции (`n` в `make_adder`). Попытка
*записать* в неё без специального ключевого слова бросит `UnboundLocalError` — Python решит, что ты
объявляешь *новую* локальную переменную с тем же именем, а не меняешь захваченную.

```python
def broken_counter():
    count = 0
    def inc():
        count += 1   # UnboundLocalError: Python видит «count = ...» и считает count локальным
        return count
    return inc
```

**Ловушка.** Ошибка возникает не в момент `def inc`, а при *первом вызове* `inc()` — именно тогда
Python пытается прочитать ещё не присвоенный локальный `count`. Это один из самых коварных
сюрпризов замыканий.

**Лечение — `nonlocal`.** Оно говорит Python: «`count` — это переменная из *ближайшего*
охватывающего `def`-scope, не создавай новую локальную».

```python
def make_counter(start: int = 0):
    count = start

    def counter() -> int:
        nonlocal count    # ← теперь Python знает: count из make_counter
        count += 1
        return count

    return counter

c = make_counter()
print(c())   # 1
print(c())   # 2

d = make_counter(10)
print(d())   # 11  — у d своя переменная count, независимая от c
print(c())   # 3   — c не пострадал
```

**Почему независимость.** Каждый вызов `make_counter()` создаёт *новый* фрейм со своим `count`.
`nonlocal` привязывает `counter` к конкретному фрейму-родителю — поэтому два счётчика никогда не
делят одно состояние.

> `nonlocal` работает только на один уровень вверх по `def`-вложенности.
> Для глобальной переменной используют `global` (но это почти всегда признак плохого дизайна).

## Идея 7. `functools` в реальной жизни

Модуль `functools` из стандартной библиотеки — арсенал инструментов для работы с функциями.
Три самых полезных в повседневном коде:

### `@functools.lru_cache` — штатный memoize

В задании 5 ты реализуешь `memoize` руками — это учебный упражнение, позволяющее понять механику.
На практике для кэширования используй `@functools.lru_cache`:

```python
import functools

@functools.lru_cache(maxsize=None)   # maxsize=None → неограниченный кэш (= cache_clear доступен)
def fib(n: int) -> int:
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

print(fib(50))   # мгновенно; без кэша — миллиарды вызовов
```

`lru_cache` потокобезопасен, поддерживает `cache_info()` и `cache_clear()`, умеет ограничивать
размер кэша (`maxsize=128` — вытесняет «самые давно использованные»; отсюда «LRU»). Ручной `memoize`
из этого модуля — учебный, в продакшене используй `lru_cache`.

### `functools.partial` — «заморозить» часть аргументов

```python
from functools import partial

def power(base, exp):
    return base ** exp

square = partial(power, exp=2)   # фиксируем exp=2, base остаётся свободным
cube   = partial(power, exp=3)

print(square(5))   # 25
print(cube(3))     # 27
```

`partial` возвращает новый вызываемый объект, который при вызове подставляет зафиксированные
аргументы. Удобно, когда нужно передать функцию-обратный-вызов с «уже настроенным» параметром.

### `functools.reduce` — свёртка последовательности

```python
from functools import reduce
import operator

nums = [1, 2, 3, 4, 5]
product = reduce(operator.mul, nums, 1)   # 1*2*3*4*5 = 120
total   = reduce(operator.add, nums, 0)   # аналог sum()
```

`reduce(f, seq, init)` применяет `f(acc, x)` по одному элементу, накапливая результат. Это
функциональный аналог цикла-аккумулятора. В Python `sum`, `min`, `max` покрывают большинство
случаев; `reduce` нужен для нестандартных свёрток (например, перемножить список или собрать дерево).

> **Подвох `reduce`.** Без третьего аргумента `init` он бросает `TypeError` на пустой список.
> Всегда передавай нейтральный элемент (`0` для сложения, `1` для умножения).

## Идея 8. Параметризованные декораторы (фабрика декораторов)

Обычный декоратор не принимает параметров. Что если нам нужен `@retry(times=3)` или
`@rate_limit(per_second=10)`? Решение: сделать *трёхуровневую* обёртку — функцию, которая
принимает параметры и *возвращает декоратор*.

```python
import functools
import time

def retry(times: int = 3, delay: float = 0.0):
    """Декоратор-фабрика: повторяет вызов до `times` раз при исключении."""
    def decorator(func):                   # ← второй уровень: принимает функцию
        @functools.wraps(func)
        def wrapper(*args, **kwargs):      # ← третий уровень: обёртка с нужной сигнатурой
            last_exc: Exception | None = None
            for attempt in range(times):
                try:
                    return func(*args, **kwargs)
                except Exception as exc:
                    last_exc = exc
                    if delay > 0:
                        time.sleep(delay)
            raise last_exc
        return wrapper
    return decorator


@retry(times=3)
def flaky_operation():
    ...
```

**Три уровня:**
1. `retry(times=3)` — *вызов фабрики*, возвращает `decorator`;
2. `@decorator` разворачивается в `flaky_operation = decorator(flaky_operation)`;
3. `wrapper` — финальная обёртка, которую видит пользователь.

**Почему `@wraps` обязателен.** Без него `flaky_operation.__name__` стал бы `'wrapper'` — все три
уровня оборачивают друг друга, и метаданные теряются на каждом переходе. `@wraps(func)` на
внутреннем `wrapper` копирует имя, docstring и `__wrapped__` за один раз.

> **Вперёд: модуль 14 (`retry`).** В нём ты реализуешь полноценный параметризованный декоратор
> `retry` с экспоненциальным back-off — теперь ты знаешь, как устроена трёхуровневая обёртка под
> капотом.
