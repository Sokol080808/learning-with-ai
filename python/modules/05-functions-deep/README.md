# Модуль 05 — Функции глубже: замыкания, декораторы

В первых модулях функция была для тебя «коробкой, которая что-то считает». Теперь поднимаемся
на уровень выше. **Большая идея модуля:** в Python функция — это обычный объект, который можно
положить в переменную, передать в другую функцию и вернуть из неё. Как только это укладывается в
голове, открываются три мощные техники: функции высшего порядка, замыкания и декораторы. На них
держится половина «магии» библиотек, которые ты будешь использовать как junior.

## Идея 1. Функции — объекты первого класса

«Объект первого класса» — занудный термин с простым смыслом: с функцией можно делать всё то же,
что с числом или строкой. Положить в переменную, передать аргументом, вернуть из функции, хранить
в списке.

```python
def shout(s: str) -> str:
    return s.upper() + "!"

f = shout            # НЕ вызываем (нет скобок) — берём саму функцию как значение
print(f("hi"))       # 'HI!'  — вызвали уже через новое имя

def call_on(fn, value):
    return fn(value)  # fn — это переданная функция, мы её здесь вызываем

print(call_on(shout, "hey"))   # 'HEY!'
```

**Почему это важно.** `shout` (без скобок) — это сам объект-функция; `shout("hi")` — её *вызов*.
Разница принципиальна: пока ты не поставил скобки, ты передаёшь поведение, а не результат. Именно
поэтому в `sorted(..., key=len)` мы пишем `len`, а не `len()`.

## Идея 2. `lambda` и функции высшего порядка

**Функция высшего порядка** — та, что принимает функцию аргументом или возвращает функцию.
Классика стандартной библиотеки: `map`, `filter`, `sorted(key=...)`. Когда функция-аргумент
крошечная и нужна разово, её удобно записать «на месте» через `lambda`.

```python
nums = [1, 2, 3, 4]
squares = list(map(lambda x: x * x, nums))      # [1, 4, 9, 16]
evens   = list(filter(lambda x: x % 2 == 0, nums))  # [2, 4]

words = ["banana", "kiwi", "apple"]
by_len = sorted(words, key=len)                  # ['kiwi', 'apple', 'banana']
```

**Почему `key`, а не сравнение руками.** `sorted` сам знает, как сравнивать; ему не хватает только
одного — *по какому признаку*. `key=len` говорит: «для каждого элемента возьми его длину и сравнивай
по ней». Сама функция `key` вызывается один раз на элемент, а не на каждое сравнение, — это и
быстро, и читаемо.

> Подвох: `lambda` — это просто короткая безымянная функция, тело которой обязано быть *одним
> выражением* (без `return`, без нескольких строк). Если логика сложнее — заведи обычную `def`.
> `map`/`filter` возвращают «ленивый» итератор, поэтому мы оборачиваем их в `list(...)`.

## Идея 3. Замыкания: функция помнит окружение

**Замыкание** возникает, когда одна функция определена внутри другой и использует переменные
внешней. Внутренняя функция «запоминает» это окружение и продолжает им пользоваться даже после
того, как внешняя функция завершилась.

```python
def make_adder(n: int):
    def adder(x: int) -> int:
        return x + n     # n взято из внешней функции — оно «захвачено»
    return adder         # возвращаем настроенную функцию

add5 = make_adder(5)
print(add5(10))          # 15  — add5 помнит, что n == 5
print(make_adder(100)(1))  # 101 — у каждого вызова своё окружение
```

**Почему это работает.** После `return adder` сама `make_adder` уже завершилась, но `n` не
исчезло: Python держит его живым, пока живёт `adder`. Каждый вызов `make_adder` создаёт *новое*
независимое окружение — поэтому `add5` и «add100» не мешают друг другу. Это аккуратный способ
«зашить» настройку в функцию без классов и глобальных переменных.

## Идея 4. Декораторы: обёртка над функцией

Декоратор — это функция, которая принимает функцию и возвращает её «обёрнутую» версию. Синтаксис
`@decorator` над `def` — лишь короткая запись для `func = decorator(func)`.

```python
import functools

def logged(func):
    @functools.wraps(func)            # сохранить имя/docstring исходной функции
    def wrapper(*args, **kwargs):
        print(f"вызываю {func.__name__}")
        return func(*args, **kwargs)  # дёргаем оригинал и отдаём его результат
    return wrapper

@logged
def add(a, b):
    return a + b

add(2, 3)            # печатает 'вызываю add', возвращает 5
print(add.__name__)  # 'add' — благодаря @wraps, а не 'wrapper'
```

**Зачем `@wraps`.** Без него обёртка «крадёт» личность исходной функции: `add.__name__` стал бы
`'wrapper'`, а docstring потерялся бы. `functools.wraps(func)` копирует метаданные оригинала на
обёртку — отладка и `help()` снова показывают правду. Декораторы хороши тем, что добавляют
поведение (логирование, кэш, проверки), не трогая тело самой функции.

## Идея 5. `*args` / `**kwargs`

Чтобы обёртка подходила к *любой* функции, ей нужно уметь принимать сколько угодно позиционных и
именованных аргументов и передавать их дальше без изменений. Для этого есть `*args` (кортеж
позиционных) и `**kwargs` (словарь именованных).

```python
def wrapper(*args, **kwargs):
    # args   == (2, 3)        — позиционные сложены в кортеж
    # kwargs == {'c': 4}      — именованные сложены в словарь
    return real(*args, **kwargs)   # * и ** «распаковывают» обратно при вызове

real(2, 3, c=4)
```

**Почему это ключ к декораторам.** В Идее 4 обёртка работала с `add(a, b)` и с любой другой
функцией именно потому, что объявлена как `(*args, **kwargs)` — она не знает заранее, сколько
аргументов придёт. Та же звёздочка при *вызове* делает обратное: разворачивает кортеж/словарь
обратно в отдельные аргументы. Это пригодится тебе в задании `memoize`.

</details>

<details><summary>Задание 5 — memoize (шаг 2: логика обёртки)</summary>

этого задания аргументы будут только позиционными — `*args` достаточно.
</details>
