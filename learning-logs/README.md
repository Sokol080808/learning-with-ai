# learning-logs — журнал учебных диалогов

Здесь автоматически копятся наши диалоги **во время обучения**, чтобы потом на их основе
улучшать курсы: видеть, где ты застреваешь, какие объяснения непонятны, каких подсказок не
хватает, какие задания вызывают больше всего вопросов.

## Как это работает

После каждого моего ответа срабатывает hook (`.claude/hooks/capture_dialogue.py`, подключён в
`.claude/settings.json` как `Stop`-хук). Он берёт из транскрипта сессии **последний вопрос** и
**последний ответ** и дописывает их в журнал.

**Пишется только на учебных ветках.** Хук пропускает `main` и `course/*` (там идёт авторство
курсов, а не учёба). Обычно учёба — на ветке `solutions`, её диалоги и сохраняются.

## Что появляется в этой папке

| Файл | Что это | В git? |
|------|---------|--------|
| `journal.jsonl` | канонический лог: по одной JSON-строке на обмен (`ts`, `branch`, `context`, `user`, `assistant`) | нет (локально) |
| `journal-YYYY-MM-DD.md` | человекочитаемый дневник по дням | нет (локально) |
| `.last_captured` | служебный файл (дедупликация) | нет |
| `README.md` | этот файл | да |

Сами диалоги **не коммитятся** (личные, могут быть большими) — они лежат локально. В репозитории
versionируется только сама система захвата. Если захочешь бэкап/перенос — просто скопируй
`journal*.*` куда нужно.

Поле `context` автоматически проставляется, если в диалоге упомянут модуль, например
`rust/modules/03-ownership` — это и есть привязка вопроса к месту курса.

## Как улучшать курсы по логам

Раз в какое-то время (например, закрыв часть модулей) попроси меня:

> «Разбери learning-logs и предложи улучшения курсов»

Я прочитаю `journal.jsonl`, сгруппирую по `context`, найду закономерности — где много вопросов,
где ты застревал, что в README было непонятно, каких ступенчатых подсказок не хватило — и
предложу точечные правки **материала курса** (README, подсказки, тесты) отдельным PR в `main`.
Правки идут в эталон на `main`, а ты подтянешь их в `solutions` через `git merge main`.

## Управление

- **Отключить захват:** убери второй `Stop`-хук в `.claude/settings.json` (с
  `capture_dialogue.py`).
- **Очистить журнал:** удали `journal*.*` и `.last_captured` в этой папке.
- **Проверить парсер вручную:** `python3 .claude/hooks/capture_dialogue.py --test <путь к .jsonl>`.
