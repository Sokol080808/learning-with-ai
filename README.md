# learning-with-ai

Мой персональный учебный воркспейс. Здесь я изучаю программирование (и не только)
с ИИ в роли **преподавателя**, а не решебника: материал объясняется «почему так»,
задания проверяются тестами, а готовые решения не выдаются по умолчанию — сначала
наводящие вопросы и подсказки.

## Что внутри

| Трек | Папка | Инструменты | До уровня |
|------|-------|-------------|-----------|
| **C++** | [`cpp/`](cpp/README.md) | CMake + GoogleTest | Junior |
| **Python** | [`python/`](python/README.md) | pytest | Junior |
| **Rust** | [`rust/`](rust/README.md) | cargo | Junior |
| **Deep Learning** | [`deep-learning/`](deep-learning/README.md) | numpy + PyTorch + pytest | практик junior |

> Воркспейс рассчитан на рост: новые языки и темы добавляются **папками** (каждый трек
> самодостаточен, со своим `run.sh`, теорией, заданиями и тестами). Перед стартом трека
> загляни в его `SETUP.md` — что и как установить.

## Быстрый старт

У каждого трека один и тот же ритм: одна команда собирает и гоняет тесты.

```bash
./cpp/run.sh              # все тесты курса C++
./cpp/run.sh 01           # только модуль 01

./python/run.sh           # Python (сам поднимет venv + pytest при первом запуске)
./rust/run.sh             # Rust (cargo)
./deep-learning/run.sh    # Deep Learning (сам поставит numpy + torch + pytest; первый запуск долгий)
```

В начале почти все тесты **красные** — это и есть задания. Делать их зелёными,
понимая каждую строку, — и есть учёба.

## Git-процесс

Материал курса и решения живут в одних файлах (стаб, который ты дописываешь: `src/*.cpp`
с `// TODO` в C++, `*.py` в Python, `src/lib.rs` с `todo!()` в Rust). Поэтому ветки
разведены по ролям:

- **`main` — эталонный курс без решений.** Чистые стабы, теория, граф, доки. Сюда
  прилетают только улучшения материала (через PR). Это и есть «версия курса без моих
  решений» — отдельно её хранить не нужно.
- **`solutions` — где я пишу код.** Долгоживущая ветка: решил модуль → коммит
  (`solve module 01`). Отдельная ветка под каждое задание не нужна.

Правила:

```bash
# один раз: завести рабочую ветку от эталона
git switch main && git switch -c solutions

# обновить материал курса, не потеряв свои решения (merge только в эту сторону):
git switch solutions && git merge main

# сбросить один модуль к чистому виду (если хочется переписать с нуля):
git checkout main -- cpp/modules/03-pointers-memory
```

`solutions → main` **не мержим** — иначе домашка затечёт в эталонный курс. Новые языки —
это **папки** (`cpp/`, `python/`, `rust/`, `deep-learning/`), а не ветки: одна общая
история на `main`.

## Как со мной работать (режим «преподаватель»)

- Я **не выдаю готовых решений** заданий по умолчанию — сначала наводящие вопросы.
- Спрашивай свободно: «почему здесь ссылка, а не значение?», «я думаю сделать так — ок?»,
  «не понимаю, почему тест падает».
- Показывай свой код — отвечу, в верном ли направлении ты идёшь.
- Застрял — попроси **подсказку** (hint). В README модулей они ступенчатые.

## Карта знаний (graphify)

Весь воркспейс (все треки) проиндексирован в граф знаний `graphify-out/`. Для вопросов
по содержимому используется `graphify query "<вопрос>"` — он возвращает точечный подграф
вместо чтения всех файлов. Граф перестраивается только на `main` (см. [CLAUDE.md](CLAUDE.md)).
