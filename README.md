# learning-with-ai

**Персональный учебный воркспейс: программирование с ИИ в роли преподавателя, а не решебника.**

[![Courses CI](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml/badge.svg)](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml)

Несколько самодостаточных курсов «от основ до junior». Материал объясняется «почему так»,
каждое задание проверяется тестами, а готовые решения не выдаются по умолчанию — сначала
наводящие вопросы и ступенчатые подсказки. Цель — **понимать**, как и почему работает код,
а не писать его «ради кода».

## Что внутри

| Трек | Папка | Инструменты | Финальный pet-проект |
|------|-------|-------------|----------------------|
| **C++** | [`cpp/`](cpp/README.md) | CMake + GoogleTest | **MiniDB** — Redis-lite БД (типы через `variant`, TTL, многопоточность, персистентность) |
| **Python** | [`python/`](python/README.md) | pytest | **мини-СУБД** с SQL-подобным языком запросов |
| **Rust** | [`rust/`](rust/README.md) | cargo | **конкурентное хранилище** (`Arc`/`Mutex`, типы значений) |
| **Deep Learning** | [`deep-learning/`](deep-learning/README.md) | numpy + PyTorch | **nanoGPT** — char-level трансформер с нуля |
| **CAOS** — архитектура и ОС | [`caos/`](caos/README.md) | C + CMake + GoogleTest | **Tiny VM** — байткод-интерпретатор |

Каждый трек: 15–18 модулей (одна большая идея на модуль) + капстоун. Перед стартом загляни в
его **`SETUP.md`** — что и как установить.

## Как устроен курс

Модуль — это:

```
modules/NN-name/
  README.md     теория простым языком + ПОЧЕМУ + описание заданий + ступенчатые подсказки
  <стаб>        интерфейс с `// TODO` / `todo!()` / NotImplementedError  ← ТЫ ПИШЕШЬ ЗДЕСЬ
  <тесты>       готовые тесты — критерий «сдано» (трогать не нужно)
```

Цикл один на все курсы: **тест знает, как должен вести себя код → дописываешь стаб → запускаешь →
красное становится зелёным, с пониманием.** В начале почти все тесты красные — это и есть задания.

## С чего начать (рекомендованный порядок)

Треки независимы, но некоторые опираются на другие:

1. **Python** и/или **C++** — фундамент: научиться программировать. С них и начинай.
2. **CAOS** — после C++ (он на C, очень близком): как машина и ОС реально исполняют код.
3. **Rust** — хорошо идёт после C++ (идеи про память встанут на место). С нуля — тоже ок.
4. **Deep Learning** — после Python (нужна уверенность в нём; математику курс даёт сам).

## Быстрый старт

У каждого трека один ритм — одна команда собирает и гоняет тесты:

```bash
./cpp/run.sh              # все тесты курса C++   (./cpp/run.sh 01 — только модуль 01)
./python/run.sh           # Python (сам поднимет .venv + pytest при первом запуске)
./rust/run.sh             # Rust (cargo)
./deep-learning/run.sh    # Deep Learning (сам поставит numpy + torch CPU + pytest; первый запуск долгий)
./caos/run.sh             # CAOS (C + CMake + GoogleTest)
```

`./<трек>/run.sh clean` — удалить сборку/окружение и начать с чистого листа.

## Как со мной работать (режим «преподаватель»)

- Я **не выдаю готовых решений** по умолчанию — сначала наводящие вопросы.
- Спрашивай свободно: «почему здесь ссылка, а не значение?», «я думаю сделать так — ок?»,
  «не понимаю, почему тест падает».
- Показывай свой код — отвечу, в верном ли направлении ты идёшь.
- Застрял — попроси **подсказку**. В README модулей они ступенчатые (от лёгкого намёка к сильному).

## Git-процесс

Материал курса и решения живут в одних файлах (стаб, который ты дописываешь). Поэтому ветки
разведены по ролям:

- **`main` — эталонные курсы без решений.** Сюда прилетают только улучшения материала (через PR).
- **`solutions` — где ты пишешь код.** Долгоживущая ветка: решил модуль → коммит.

```bash
git switch main && git switch -c solutions    # один раз: завести рабочую ветку
git switch solutions && git merge main         # подтянуть обновления материала (merge только сюда)
git checkout main -- cpp/modules/03-pointers-memory   # сбросить модуль к чистому виду
```

`solutions → main` **не мержим** — иначе домашка затечёт в эталон. Новые языки — это **папки**,
а не ветки: одна общая история на `main`.

## Карта знаний (graphify)

Весь воркспейс проиндексирован в граф знаний `graphify-out/`. Для вопросов по содержимому —
`graphify query "<вопрос>"` (возвращает точечный подграф вместо чтения всех файлов),
`graphify path "<A>" "<B>"`, `graphify explain "<концепт>"`. Граф перестраивается только на
`main`. Подробнее — [CLAUDE.md](CLAUDE.md).

## Журнал диалогов (learning-logs)

Во время учёбы (на ветке `solutions`) наши диалоги «вопрос ↔ ответ» автоматически копятся в
[`learning-logs/`](learning-logs/README.md). Потом по ним можно улучшать курсы: видно, где ты
застревал и что было непонятно. Данные локальные (в git не идут).

## Проверка (CI)

[GitHub Actions](.github/workflows/courses-ci.yml) на каждый PR проверяет, что **скелеты курсов
остаются валидными на Linux**: C++/Rust/CAOS компилируются, Python/DL-тесты импортируются и
собираются. Задания намеренно красные, поэтому CI проверяет «компилируется / тесты собираются»,
а не «всё зелёное».
