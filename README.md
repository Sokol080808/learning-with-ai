# learning-with-ai

[![Courses CI](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml/badge.svg)](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml)

Набор практических курсов программирования: **C++, Python, Rust, Deep Learning** и **CAOS**
(устройство компьютера и операционных систем). Каждый курс доводит как минимум до уровня junior и проверяется тестами. Курсы рассчитаны на прохождение вместе с ИИ-ассистентом,
который работает как преподаватель: не выдаёт готовых решений, а задаёт наводящие вопросы и даёт подсказки.

## Как устроен курс

Каждый курс лежит в своей папке и состоит из модулей. В модуле три части:

- `README.md` — теория и описание заданий;
- **стаб** — заготовка кода с пометками `// TODO` (`todo!()` в Rust, `NotImplementedError` в
  Python), которую дописываешь ты;
- **тесты** — готовые, менять их не нужно. Они задают ожидаемое поведение и служат критерием «сдано».
  В Python и C++ к разобранным примерам добавлены рандомизированные (property-based) тесты — они
  проверяют твою реализацию на сотнях случайных входов и краевых случаях.

Рабочий цикл одинаков для всех курсов: запустить тесты → увидеть красное (это и есть задания) →
дописать код → добиться зелёного.

Например, дан стаб:

```python
def is_even(n: int) -> bool:
    raise NotImplementedError("TODO: реализуй is_even")
```

и тест к нему:

```python
def test_is_even():
    assert is_even(4) is True
    assert is_even(7) is False
```

Тест падает, пока функция не реализована. Дописываешь тело (`return n % 2 == 0`) — тест проходит.

## Установка и запуск

Перед началом курса открой его `SETUP.md` — там перечислено, что установить (компилятор или
интерпретатор и пара инструментов). Дальше всё делается одной командой на курс:

```bash
./cpp/run.sh            # собрать и прогнать все тесты курса C++
./cpp/run.sh 01         # прогнать только модуль 01
./python/run.sh         # Python (виртуальное окружение и pytest поднимаются сами)
./rust/run.sh           # Rust (cargo)
./deep-learning/run.sh  # Deep Learning (поставит numpy и torch; первый запуск долгий)
./caos/run.sh           # CAOS (C, CMake, GoogleTest)

./cpp/run.sh clean      # удалить сборку и окружение
```

В начале большинство тестов красные — это нормально, такими они и задуманы.

## Работа с ИИ

Курсы рассчитаны на прохождение с ассистентом в роли преподавателя. Покажи ему свой код или задай
вопрос — в ответ будет разбор и подсказка, а не готовый фрагмент. Например:

```
— Почему is_even(-3) возвращает не то?
— Проверь, что выдаёт -3 % 2 в Python. В этом и причина.
```

Подсказки в README модулей устроены ступенчато — от лёгкого намёка к сильному. Полное решение
ассистент даёт только по явной просьбе и с объяснением.

## Ветки

Материал курса и решения лежат в одних и тех же файлах (дописываются те самые стабы), поэтому
ветки разделены по назначению:

- **`main`** — эталонные курсы без решений; сюда попадают только правки материала;
- **`solutions`** — рабочая ветка, где пишется код заданий;
- **`reference`** — «ключ с ответами»: заполненные стабы, на которых все тесты зелёные. Нужен, чтобы
  доказать, что задания решаемы, и проверять тесты на CI. Это не для прохождения курса — подглядывать туда смысла нет.

Чтобы подтянуть обновления материала, не теряя своих решений, делается `git merge main` в
`solutions`. Обратно (`solutions → main`) ветку не сливают. Новые языки добавляются папками, а не
ветками.

## Дополнительно

- `graphify-out/` — граф знаний по всему репозиторию. Запрос по содержимому:
  `graphify query "<вопрос>"`.
- [`learning-logs/`](learning-logs/README.md) — журнал учебных диалогов (хранится локально),
  по которому можно дорабатывать курсы.
- [`.github/workflows/`](.github/workflows/courses-ci.yml) — CI: проверяет, что курсы компилируются
  и тесты собираются (сами задания при этом намеренно красные). Отдельная проверка `reference-green`
  гоняет эталонные решения с ветки `reference` и требует, чтобы на них всё было зелёным — гарантия,
  что задания действительно решаемы.
- [`CLAUDE.md`](CLAUDE.md) — правила работы ИИ-ассистента в этом репозитории.
