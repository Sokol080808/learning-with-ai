# learning-with-ai

[![Courses CI](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml/badge.svg)](https://github.com/Sokol080808/learning-with-ai/actions/workflows/courses-ci.yml)

Это мой личный «тренажёрный зал» для программирования. Я учусь не по видеокурсам, где показывают
готовое и говорят «повторяй», а вместе с ИИ, который ведёт себя как **преподаватель**: объясняет,
почему всё устроено именно так, задаёт наводящие вопросы и не выкладывает решение, пока я сам не
попробую до него дойти.

Я не профи — я учусь, местами с нуля (на Rust раньше не писал, машинным обучением не занимался
вообще). И ровно поэтому всё здесь построено вокруг одной мысли: **понимать, а не зубрить.**
Скопировать код или прорешать задачку «лишь бы загорелось зелёным» бесполезно — оно выветривается
за неделю. А вот когда понимаешь, *почему* так, знание остаётся.

## Как это работает — на конкретном примере

Каждый курс — это модули, и все они сделаны по одному шаблону. В модуле три вещи: **README** с
теорией, **стаб** (заготовка, куда я пишу) и **готовые тесты** (их не трогаю — они и есть точное
определение «сделано»).

Выглядит это так. Вот стаб, который мне дают:

```python
# basics.py — сюда пишу я
def is_even(n: int) -> bool:
    """Вернуть True, если число чётное."""
    raise NotImplementedError("TODO: реализуй is_even")
```

А вот тест рядом, который уже написан и говорит, чего от меня хотят:

```python
# test_basics.py — трогать не нужно, это критерий «сдано»
def test_is_even():
    assert is_even(4) is True
    assert is_even(7) is False
```

Запускаю `./python/run.sh 01` — тест **красный** (я же ещё ничего не написал). Это и есть задание.
Дописываю тело: `return n % 2 == 0`, запускаю снова — **зелёный**. Готово, иду дальше.

Вот и весь цикл, один на все пять курсов: **красное → пишу код → зелёное**, маленькими шагами, с
постоянной проверкой. Я не пишу код «в пустоту» — сразу вижу, работает он или нет, и где именно
ошибся.

## Как я работаю с ИИ

ИИ тут — не кнопка «реши за меня», а собеседник. Я не получаю готовый кусок кода; я получаю разбор.
На практике это выглядит примерно так:

> — Не понимаю, почему мой `is_even(-3)` возвращает не то.
>
> — А что у тебя выдаёт `-3 % 2` в Python? Проверь в интерпретаторе — результат тебя удивит, и в
>   нём вся причина. Подумай, как это влияет на твою проверку.

То есть меня подталкивают к ответу, а не выдают его. Застрял по-серьёзному — прошу намёк посильнее;
в README модулей подсказки и так идут ступенчато, от лёгкого к сильному. Полное решение мне дадут,
только если прямо попрошу, и всё равно с объяснением «почему именно так». Смысл — чтобы знание
оставалось в голове, а не в буфере обмена.

## Курсы

Пять курсов, каждый ведёт от самых основ до уровня junior. Они независимы, лежат каждый в своей
папке (со своей теорией, заданиями, тестами и общей командой `run.sh`) и заканчиваются
pet-проектом, в котором всё пройденное сходится воедино.

### C++ — [`cpp/`](cpp/README.md)
Самый «фундаментальный» язык в наборе. Курс ведёт от того, как программа вообще превращается в
исполняемый файл, через указатели и модель памяти, классы и RAII, копирование/перемещение, умные
указатели, шаблоны, STL и обработку ошибок — до многопоточности и оценки сложности (Big-O).
**К концу** я не «знаю синтаксис C++», а собираю на нём настоящую вещь: капстоун **MiniDB** —
маленькую базу в духе Redis с несколькими типами значений (через `std::variant`), временем жизни
ключей, потокобезопасным доступом из нескольких потоков и сохранением на диск.

### Python — [`python/`](python/README.md)
Язык, на котором быстро превращаешь идею в работающую программу. Путь: типы и коллекции, функции и
замыкания, декораторы, ООП, генераторы, обработка ошибок, файлы, стандартная библиотека,
аннотации типов, инструменты. Капстоун — **мини-СУБД со своим SQL-подобным языком**: я пишу
токенизатор, парсер и движок, который понимает `CREATE / INSERT / SELECT … WHERE / ORDER BY /
DELETE` над таблицами в памяти.

### Rust — [`rust/`](rust/README.md)
Язык, который ловит ошибки памяти ещё при компиляции — без сборщика мусора и без потери скорости.
Его главная идея (владение, заимствование, времена жизни) — это и есть сердце курса. Иду через
`enum` и сопоставление с образцом, `Option`/`Result`, коллекции, дженерики и трейты, замыкания и
итераторы, до умных указателей. Капстоун — **конкурентное хранилище** с потокобезопасной обёрткой
`Arc<Mutex<…>>`. Я на Rust раньше не писал, поэтому курс честно начинается с «зачем он вообще нужен
и где силён/слаб», а уже потом — синтаксис. (Первые недели борьбы с компилятором будут бесить — это
нормально, так и задумано: он ловит за руку баги, которые в C жили бы незаметно.)

### Deep Learning — [`deep-learning/`](deep-learning/README.md)
Флагман набора — самый полный и подробный. ML я раньше не трогал, поэтому курс стартует с самого
начала: что это вообще такое, зачем и какая математика реально нужна (её он объясняет сам). Дальше —
тензоры, градиенты, регрессия, нейросеть с нуля на numpy, **обратное распространение руками**,
собственный autograd-движок, оптимизаторы — и только *потом* PyTorch, свёрточные и рекуррентные
сети, внимание и трансформеры. Капстоун — **nanoGPT**: крошечная языковая модель, собранная из
кубиков, которые я написал сам. После него «языковая модель» перестаёт быть магией.

### CAOS — [`caos/`](caos/README.md)
Computer Architecture & Operating Systems: как код на самом деле исполняется железом и как этим
управляет ОС. Курс на C — языке систем. От представления чисел и битовых операций через память,
выравнивание и чтение ассемблера x86-64 — к кэшам, аллокаторам, процессам и потокам, планировщику,
виртуальной памяти и синхронизации. Капстоун — **Tiny VM**, своя стековая виртуальная машина:
байткод-интерпретатор с циклом «выборка — декодирование — исполнение».

## Как начать

```bash
git clone https://github.com/Sokol080808/learning-with-ai.git
cd learning-with-ai
```

1. **Выбери курс** (если только начинаешь — бери Python или C++, см. порядок ниже) и открой его
   `SETUP.md` — там сказано, что поставить. Обычно это компилятор/интерпретатор и пара инструментов;
   окружение питоновских курсов `run.sh` поднимает сам при первом запуске.
2. **Запусти курс** — одна команда собирает всё и гоняет тесты:
   ```bash
   ./cpp/run.sh              # все тесты C++   ( ./cpp/run.sh 01 — только модуль 01 )
   ./python/run.sh           # Python — сам поднимет окружение и pytest
   ./rust/run.sh             # Rust — через cargo
   ./deep-learning/run.sh    # Deep Learning — сам поставит numpy + torch; первый запуск долгий
   ./caos/run.sh             # CAOS — C, CMake, GoogleTest
   ```
   Почти всё красное — это нормально, это и есть план работы. (`run.sh clean` сотрёт сборку, если
   захочется с чистого листа.)
3. **Открой README модуля `00`** в выбранном курсе и иди по нему. Дописывай стабы, гоняй тесты,
   спрашивай меня, когда непонятно.

**Рекомендованный порядок**, если идёшь с нуля: **Python и/или C++** (научиться программировать) →
**CAOS** (после C++ — что под капотом) → **Rust** (после C++ — идеи про память встанут на место) →
**Deep Learning** (после Python).

## Что под капотом (и зачем)

Несколько вещей, которые делают репозиторий удобным именно для учёбы.

**Две ветки git вместо одной.** Материал курса и мои решения лежат в одних и тех же файлах (я
дописываю те самые стабы), поэтому ветки разведены по ролям. На **`main`** — *эталонный* курс без
решений (чистые стабы и теория); туда идут только улучшения материала. На **`solutions`** — мой код.
Когда `main` обновляется, я подтягиваю его к себе через `git merge main`, не теряя решений. Обратно
не сливаю, чтобы моя «домашка» не затекла в эталон. Поэтому и новые языки добавляются **папками**, а
не ветками.

**Граф знаний.** Весь воркспейс проиндексирован в граф (`graphify-out/`): концепции, зависимости
между модулями и связи между курсами (например, RAII в C++ перекликается с владением в Rust и
контекст-менеджерами в Python). Чтобы сориентироваться, вместо чтения десятков файлов я спрашиваю
`graphify query "<вопрос>"` и получаю точечный кусочек графа.

**Журнал диалогов.** Пока я учусь (на ветке `solutions`), наши разговоры складываются в
[`learning-logs/`](learning-logs/README.md) — потом по ним можно улучшать сами курсы: видно, где я
застревал и что было непонятно. Записи остаются локально, в git не идут.

**CI.** На каждый PR [GitHub Actions](.github/workflows/courses-ci.yml) проверяет, что скелеты курсов
не разваливаются: C++, Rust и CAOS компилируются, а тесты Python и Deep Learning собираются. Задания
*намеренно* красные, поэтому проверка — «компилируется и собирается», а не «всё зелёное».

Правила, по которым ИИ работает в этом репозитории (как преподаёт, когда трогает граф), — в
[CLAUDE.md](CLAUDE.md).
